## 1. What Line 12e is

Form 1040 line 12e is labeled **"Standard deduction or itemized deductions (from Schedule A)"**. It is the dollar amount that gets subtracted from adjusted gross income (line 11b) — together with line 13a (qualified business income deduction) and line 13b (additional deductions on Schedule 1-A) — to arrive at taxable income (line 15).

```
Form1040.line12e = one of three values, chosen by the rule in section 8 below
```

The three possible values come from three completely different IRS paths:
- **path A** — the computed standard deduction (an amount derived from filing status, age, blindness, and dependency status — no Schedule A involved);
- **path B** — the total of itemized deductions from Schedule A line 17;
- **path C** — the special increased-standard-deduction-for-net-qualified-disaster-loss path, where line 12e takes Schedule A line 16 combined amount (not line 17).

The label of the chosen path also feeds an output field called `deductionType` ("Standard" or "Itemized") for user-interface display.

## 2. Plain-English computation per ingredient (the IRS rules — independent of our code)

The following four checkboxes on line 12 of the printed Form 1040 do **not** themselves carry a dollar amount, but they each modify the standard deduction computation in section 5 below. They are inputs to line 12e, not outputs.

### 2.1 Line 12a — "Someone can claim:"

Two sub-boxes:
- **You** as a dependent — checked when the taxpayer (or, on Married Filing Jointly, the family head) can be claimed as a dependent by somebody else. Input field on the taxpayer-side standard-deductions form: `someoneCanClaimYou` (Boolean).
- **Your spouse** as a dependent — applies only on a Married Filing Jointly return. Input field on the spouse-side standard-deductions form: `someoneCanClaimSpouse` (Boolean).

If either sub-box is true, the taxpayer must use the **Standard Deduction Worksheet for Dependents** (see section 5.3) instead of the regular standard-deduction amount.

A subtle Married Filing Jointly nuance from the 2025 instructions: a person can still be claimed on someone else's return when the joint return is filed only to claim a refund of withheld tax or estimated tax. The user attests this via the same `someoneCanClaimSpouse` Boolean.

### 2.2 Line 12b — "Spouse itemizes on a separate return"

Checked only when:
- filing status is **Married Filing Separately**, and
- the spouse itemizes deductions on the spouse's own separate return.

Input field on the spouse-side standard-deductions form: `spouseItemizesSeparateReturn` (Boolean).

Effect (per Internal Revenue Code section 63(c)(6)(A)):

```
if line 12b is checked:
    standard_deduction = 0
```

This zero hard-floor cannot be undone by checking age or blindness boxes on line 12d.

### 2.3 Line 12c — "You were a dual-status alien"

Checked when the taxpayer was a dual-status alien (an individual who was both a resident alien and a nonresident alien during the same tax year — typically because of a mid-year status change). Underlying rule: Internal Revenue Code section 7701(b).

Input field on the taxpayer-side standard-deductions form: `youWereDualStatusAlien` (Boolean).

Effect (per Internal Revenue Code section 63(c)(6)(B)):

```
if line 12c is checked:
    standard_deduction = 0
```

The 2025 instructions add an explicit exception: if the taxpayer and spouse validly elect to be taxed on combined worldwide income as United States residents for the full year, line 12c should not be checked (the user is expected to report this via the same Boolean — by leaving it false when the election applies).

### 2.4 Line 12d — Age and blindness boxes

Four sub-boxes (each contributes one to a running count):
- The taxpayer was born before **January 2, 1961** — Input field: `youBornBeforeThreshold` (Boolean).
- The taxpayer is blind — Input field: `youAreBlind` (Boolean).
- The spouse was born before **January 2, 1961** — Input field on spouse form: `spouseBornBeforeThreshold` (Boolean).
- The spouse is blind — Input field on spouse form: `spouseIsBlind` (Boolean).

Rules for which sub-boxes are allowed to count:
- For **Head of Household**, spouse sub-boxes are never allowed.
- For **Married Filing Separately**, spouse sub-boxes are allowed only when the spouse had no gross income, is not filing a return, and cannot be claimed as a dependent — captured via the Boolean `spouseMeetsAgeBlindnessMfsRequirements` on the spouse form.
- For **Qualifying Surviving Spouse**, at most 2 boxes contribute (one for age and one for blindness — there is no separate spouse).
- A person is treated as reaching age 65 on the **day before** the 65th birthday (so a birthday of January 1, 1961 still counts as age 65 by the end of 2025).

The count produced by line 12d is between 0 and 4 and feeds two different downstream computations:
- the per-box addition in the Dependent Standard Deduction Worksheet (section 5.3) when line 12a is checked;
- the lookup in the Age-and-Blindness Standard Deduction Chart (section 5.4) when line 12a is not checked.

## 3. Reference data needed to compute line 12e (2025 amounts — these change every tax year)

**Standard deduction base amounts** (these are the amounts printed on the Form 1040 next to line 12e for each filing status):

```
STANDARD_DEDUCTION_BASE_SINGLE             = $15,750
STANDARD_DEDUCTION_BASE_MARRIED_FILING_SEPARATELY = $15,750
STANDARD_DEDUCTION_BASE_MARRIED_FILING_JOINTLY    = $31,500
STANDARD_DEDUCTION_BASE_QUALIFYING_SURVIVING_SPOUSE = $31,500
STANDARD_DEDUCTION_BASE_HEAD_OF_HOUSEHOLD  = $23,625
```

**Additional standard deduction per age-or-blindness box** (added on top of the base for each of the four line 12d boxes):

```
ADDITIONAL_STANDARD_DEDUCTION_PER_BOX_SINGLE_OR_HEAD_OF_HOUSEHOLD = $2,000
ADDITIONAL_STANDARD_DEDUCTION_PER_BOX_MARRIED_OR_SURVIVING_SPOUSE = $1,600
```

**Dependent Standard Deduction Worksheet constants** (used when line 12a is checked):

```
DEPENDENT_EARNED_INCOME_ADDON_FLOOR_THRESHOLD = $900
DEPENDENT_EARNED_INCOME_ADDON                  = $450
DEPENDENT_STANDARD_DEDUCTION_MINIMUM           = $1,350
```

**Age threshold for line 12d age boxes** (born before this date is the trigger):

```
STANDARD_DEDUCTION_BORN_BEFORE_DATE = January 2, 1961
```

**Schedule A medical-expense floor rate**:

```
SCHEDULE_A_MEDICAL_FLOOR_RATE = 7.5% of adjusted gross income (line 11b)
```

**Schedule A state-and-local-tax (SALT) cap, OBBBA-2025 rules** (this is the largest 2025 change — the prior flat $10,000 / $5,000 rule is no longer correct):

```
SCHEDULE_A_SALT_LIMIT_SINGLE                    = $40,000
SCHEDULE_A_SALT_LIMIT_MARRIED_FILING_JOINTLY    = $40,000
SCHEDULE_A_SALT_LIMIT_QUALIFYING_SURVIVING_SPOUSE = $40,000
SCHEDULE_A_SALT_LIMIT_HEAD_OF_HOUSEHOLD         = $40,000
SCHEDULE_A_SALT_LIMIT_MARRIED_FILING_SEPARATELY = $20,000

SCHEDULE_A_SALT_PHASEOUT_THRESHOLD_DEFAULT = $500,000  (modified adjusted gross income)
SCHEDULE_A_SALT_PHASEOUT_THRESHOLD_MARRIED_FILING_SEPARATELY = $250,000

SCHEDULE_A_SALT_FLOOR_DEFAULT = $10,000
SCHEDULE_A_SALT_FLOOR_MARRIED_FILING_SEPARATELY = $5,000
```

The phasedown reduces the cap one dollar for each dollar of modified adjusted gross income above the threshold, but never below the floor.

## 4. Standard deduction computation (IRS canonical — plain English)

These four cases are checked in order. The first matching case wins.

### 4.1 Hard-zero case (line 12b or line 12c is checked)

```
if line12b is true or line12c is true:
    standard_deduction = 0
```

This zero is final for the standard deduction. It cannot be increased by age, blindness, or dependent-worksheet logic. If the taxpayer also wants to itemize, the itemized path in section 6 below still works.

### 4.2 Dependent worksheet case (line 12a is checked)

When somebody can claim the taxpayer (or, on Married Filing Jointly, the spouse), the IRS replaces the regular standard deduction with the *smaller* of two things: a number based on the dependent's earned income, or the regular base for the filing status. The age and blindness boxes still add on top.

```
earned_income = the dependent's earned income (see section 4.2.1 below for the IRS definition)

if earned_income > $900:
    worksheet_line_2 = earned_income + $450
else:
    worksheet_line_2 = $1,350

worksheet_line_3 = base_standard_deduction_for_filing_status   (from section 3)

worksheet_line_4_a = the smaller of worksheet_line_2 and worksheet_line_3

per_box_addon = $2,000 if filing status is Single or Head of Household
              = $1,600 otherwise

worksheet_line_4_b = line_12d_count * per_box_addon

standard_deduction = worksheet_line_4_a + worksheet_line_4_b
```

#### 4.2.1 Earned income for the dependent worksheet (IRS rule)

Per the 2025 Form 1040 instructions for the Dependent Worksheet, earned income is:

```
earned_income = Form1040.line1z
              + Schedule1.line3      (business income from Schedule C)
              + Schedule1.line6      (farm income from Schedule F)
              + Schedule1.line8r     (scholarship and fellowship grants not on a W-2)
              + Schedule1.line8t     (pension or annuity from a nonqualified deferred-compensation
                                       plan or a section 457 plan included in earned-income computations)
              + Schedule1.line8u     (wages earned while incarcerated)
              − Schedule1.line15     (deductible part of self-employment tax)
```

The dependent's standard-deductions personal form does have a `dependentStandardDeductionEarnedIncome` (a `number` field) — the user can override the auto-computed value or supply it directly.

### 4.3 Age-and-blindness chart case (line 12a is not checked, line 12d count is at least 1)

When the taxpayer is not a dependent but is over 65 or blind (or has a qualifying spouse who is), the IRS adds a fixed dollar amount per box checked. The chart for 2025 is:

| Filing status | 1 box | 2 boxes | 3 boxes | 4 boxes |
|---|---|---|---|---|
| Single | $17,750 | $19,750 | — | — |
| Married Filing Separately | $17,350 | $18,950 | $20,550 | $22,150 |
| Married Filing Jointly | $33,100 | $34,700 | $36,300 | $37,900 |
| Qualifying Surviving Spouse | $33,100 | $34,700 | — | — |
| Head of Household | $25,625 | $27,625 | — | — |

The amounts are mechanically equal to the base for the filing status plus the per-box addon times the number of boxes — see section 3.

### 4.4 Plain base case (no exceptions apply)

```
standard_deduction = base_standard_deduction_for_filing_status   (from section 3)
```

## 5. Itemized deduction computation (Schedule A — plain English)

When the taxpayer chooses to itemize, line 12e takes the total from Schedule A line 17. The full structure of Schedule A:

### 5.1 Schedule A lines 1 through 4 — Medical and dental expenses

```
schedule_a_line_1 = medicalDentalExpensesPaid     (input from the deductions form)
schedule_a_line_2 = Form1040.line11b              (adjusted gross income)
schedule_a_line_3 = schedule_a_line_2 * 7.5%      (the medical floor)
schedule_a_line_4 = max(0, schedule_a_line_1 - schedule_a_line_3)
```

### 5.2 Schedule A lines 5 through 7 — Taxes paid

The taxpayer chooses one of the two: state and local income taxes, or state and local general sales taxes (not both). If the taxpayer elects general sales taxes, the box next to line 5a must be checked.

```
selected_state_local_tax = stateLocalIncomeTaxesPaid    if stateLocalTaxChoice == "Income"
                         = stateLocalSalesTaxesPaid     if stateLocalTaxChoice == "Sales"

schedule_a_line_5_d = selected_state_local_tax + realEstateTaxesPaid + personalPropertyTaxesPaid

# OBBBA-2025 SALT cap with phasedown — see section 3 for constants.
salt_cap = SCHEDULE_A_SALT_LIMIT_for_filing_status

if modified_adjusted_gross_income > SCHEDULE_A_SALT_PHASEOUT_THRESHOLD:
    excess = modified_adjusted_gross_income - SCHEDULE_A_SALT_PHASEOUT_THRESHOLD
    salt_cap = max(SCHEDULE_A_SALT_FLOOR, salt_cap - excess)

schedule_a_line_5_e = min(schedule_a_line_5_d, salt_cap)

schedule_a_line_6 = foreignTaxesPaid             (state and local foreign taxes; not subject to the SALT cap)

schedule_a_line_7 = schedule_a_line_5_e + schedule_a_line_6
```

Important nuance: per the spec, modified adjusted gross income is the basis for the phasedown. The 2025 instructions use Form 1040 line 11b as the modified adjusted gross income proxy in the absence of other modifications.

### 5.3 Schedule A lines 8 through 10 — Interest paid

```
deductible_investment_interest = min(investmentInterestPaid, netInvestmentIncome)
schedule_a_line_8_e = homeMortgageInterestPaid + homeMortgagePointsPaid
schedule_a_line_9   = deductible_investment_interest
schedule_a_line_10  = schedule_a_line_8_e + schedule_a_line_9
```

(There is also Form 4952 logic for carrying forward disallowed investment interest from a prior year and for electing to treat qualified dividends or net capital gain as investment income — those are not currently exposed via the personal form. See section 9.)

### 5.4 Schedule A lines 11 through 14 — Gifts to charity

```
schedule_a_line_11 = charitableCashContributions
schedule_a_line_12 = charitableNonCashContributions
schedule_a_line_13 = 0                                  (carryover from prior year — not currently captured)
schedule_a_line_14 = schedule_a_line_11 + schedule_a_line_12 + schedule_a_line_13
```

### 5.5 Schedule A line 15 — Casualty and theft losses

This is the personal-casualty-and-theft-loss path. The loss must be attributable to a **federally declared disaster** to be deductible. The user enters the deductible amount directly via `personalCasualtyAndTheftLoss`. The disaster-loss-increased-standard path uses a separate field (`netQualifiedDisasterLoss`) — see section 7.

```
schedule_a_line_15 = personalCasualtyAndTheftLoss
```

### 5.6 Schedule A line 16 — Other itemized deductions

This line is not a free-form bucket. It is limited to specific items listed in the Schedule A instructions: attorney fees from unlawful discrimination claims, impairment-related work expenses, repayments of section 1341 claim of right amounts above $3,000, certain estate-tax deductions, gambling losses up to gambling winnings, and a few others.

```
schedule_a_line_16 = otherAllowedItemizedDeductions
                   + (excess_ssa_rrb_repayment_auto_addon if no section 1341 credit was claimed)
                   + (netQualifiedDisasterLoss if electDisasterLossStandardDeductionIncrease is true and not itemizing — see section 7)
```

The excess Social Security or Railroad Retirement repayment auto-addon comes from line 6a/6b (when the prior-year Social Security benefit overpayment is being repaid more than the gross benefits this year, the excess is deductible on Schedule A line 16 instead of being a § 1341 credit, per IRS Publication 915 "Repayments More Than Gross Benefits").

### 5.7 Schedule A line 17 — Total itemized deductions

```
schedule_a_line_17 = schedule_a_line_4              (medical)
                   + schedule_a_line_7              (taxes)
                   + schedule_a_line_10             (interest)
                   + schedule_a_line_14             (charity)
                   + schedule_a_line_15             (casualty/theft)
                   + schedule_a_line_16             (other)
```

### 5.8 Schedule A line 18 — Elect to itemize even though less than standard

A box; checked when the taxpayer explicitly elects to itemize even though itemized deductions are less than the standard deduction. The user signals this election by choosing "Itemize deductions" on the deductions form even when standard would be larger.

## 6. Net qualified disaster loss path (Schedule A line 16 → line 12e)

This path is **separate** from normal itemizing. It bumps the standard deduction up by the net qualified disaster loss amount; the user does **not** also claim Schedule A medical / taxes / interest / charity.

Inputs:
- `netQualifiedDisasterLoss` (number) — the amount from Form 4684 line 15.
- `electDisasterLossStandardDeductionIncrease` (Boolean) — the user explicit-elects this path.
- The taxpayer must **not** be otherwise itemizing.

When the election is in effect and the taxpayer is not itemizing:

```
schedule_a_line_16_combined = base_standard_deduction_amount + netQualifiedDisasterLoss
Form1040.line12e            = schedule_a_line_16_combined
```

Other Schedule A lines are left blank in this path. Form 4684 must be attached.

If the taxpayer is **itemizing** (i.e. taking the regular Schedule A path), the net qualified disaster loss is included on Schedule A line 16 alongside the other line-16 items, and the final amount still flows through Schedule A line 17. There is no separate handling — the disaster loss simply becomes one term inside line 16.

---

## 7. Section 962 election / section 250 deduction — special exclusion

If the taxpayer made a **section 962 election** and is taking a **section 250 deduction** with respect to section 951A inclusions (these are GILTI — Global Intangible Low-Taxed Income — inclusions for shareholders of controlled foreign corporations):

```
do NOT report the section 250 deduction on line 12e.
Instead, report the related tax on line 16 and include the required explanatory statement.
```

The user is expected to manage this manually — there is no automatic routing for the section 962 / 250 path.

## 8. Final mapping to line 12e (the IRS rule)

```
if user has electDisasterLossStandardDeductionIncrease == true AND the user is not itemizing:
    Form1040.line12e = schedule_a_line_16_combined_amount        (path C, from section 6)
elif the user elects to itemize:
    Form1040.line12e = schedule_a_line_17                         (path B)
else:
    Form1040.line12e = standard_deduction_from_section_4         (path A)
```

The taxpayer **may elect to itemize even when the itemized total is smaller than the standard deduction** — this is the Schedule A line 18 election. So line 12e is **not** simply `max(standard, itemized)`. It is one of the three values above, and the choice is driven by the explicit `deductionElection` field on the standard-deductions form (`Take the standard deduction` / `Itemize deductions` / `Auto choose the larger deduction`).

`deductionType` ("Standard" or "Itemized") is an output label that records which path won.

## 11. Computation as implemented by the code (TaxReturnComputeService.java)

The implementation is split across several private methods:

### 11.1 Dispatch in `computeLine12()` (TaxReturnComputeService line 5018)

```
status                = normalize(filingStatus.filingStatus)

line12a               = indicators.someoneCanClaimYou
                       OR (status == "Married filing jointly"
                            AND indicators.someoneCanClaimSpouse)

line12b               = (status == "Married filing separately")
                       AND indicators.spouseItemizesSeparateReturn

line12c               = indicators.youWereDualStatusAlien

spouseMeetsReqs       = indicators.spouseMeetsAgeBlindnessMfsRequirements
mfsSpouseBoxesAllowed = (status != "Married filing separately") OR spouseMeetsReqs

line12dCount          = countAgeBlindnessBoxes(status,
                            indicators.youBornBeforeThreshold,
                            indicators.youAreBlind,
                            (mfsSpouseBoxesAllowed ? indicators.spouseBornBeforeThreshold : null),
                            (mfsSpouseBoxesAllowed ? indicators.spouseIsBlind : null))
```

`countAgeBlindnessBoxes` itself also drops the spouse arguments for Head-of-Household (per the IRS rule that Head-of-Household has no spouse), and clamps the Qualifying Surviving Spouse count at 2.

### 11.2 Earned income for the dependent worksheet — the code's fallback

```
if line12a is true AND user did not enter dependentStandardDeductionEarnedIncome AND income.totalWages is positive:
    dependentStandardDeductionEarnedIncome = income.totalWages   (the line 1z value)
```

That fallback uses only line 1z. The IRS rule (see section 4.2.1) adds Schedule 1 lines 3, 6, 8r, 8t, 8u and subtracts Schedule 1 line 15. **This is a known gap** — see section 12.

### 11.3 Standard deduction — `computeStandardDeduction()` (TaxReturnComputeService line 5431)

```
if status is null:
    return null
if line12b OR line12c:
    return 0
if line12a:
    earnedIncome = deductionsTaxpayer.dependentStandardDeductionEarnedIncome
    if earnedIncome > 900:
        line2 = earnedIncome + 450
    else:
        line2 = 1350
    limitedBase = min(line2, base_standard_deduction_for_status(status))
    addOn       = additional_standard_deduction_for_boxes(status, line12dCount)
    return roundMoney(limitedBase + addOn)
if line12dCount > 0:
    return roundMoney(lookup_age_blindness_standard_deduction(status, line12dCount))
return roundMoney(base_standard_deduction_for_status(status))
```

The base, addon, and chart lookups are simple table lookups against the constants in section 3.

### 11.4 Schedule A construction — `buildScheduleA()` (TaxReturnComputeService line 5494)

The Schedule A object is built unconditionally whenever the user has any Schedule A input (or the auto-addon for excess Social Security repayment is positive). Highlights:

```
deductibleMedical = max(0, medicalDentalExpensesPaid - 7.5% * adjustedGrossIncome)

selectedStateTax  = stateLocalSalesTaxesPaid  if stateLocalTaxChoice == "Sales"
                  = stateLocalIncomeTaxesPaid otherwise

taxesBeforeLimit  = selectedStateTax + realEstateTaxesPaid + personalPropertyTaxesPaid

# OBBBA-2025 SALT cap with phasedown via saltLimitForStatusAndMagi(status, agi)
deductibleTaxes   = min(taxesBeforeLimit, saltLimitForStatusAndMagi(status, agi))

deductibleInvestmentInterest = min(investmentInterestPaid, netInvestmentIncome)
totalInterest                = homeMortgageInterestPaid + homeMortgagePointsPaid + deductibleInvestmentInterest

totalCharity        = charitableCashContributions + charitableNonCashContributions

deductibleForeignTaxes = foreignTaxesPaid     (no cap, no Form 1116 reduction wired in)

otherAllowedItemized = userEnteredOtherItemized + (autoOtherItemizedAddon if positive)

totalItemized       = deductibleMedical
                    + deductibleTaxes
                    + totalInterest
                    + totalCharity
                    + deductibleForeignTaxes
                    + personalCasualtyAndTheftLoss
                    + otherAllowedItemized
                    + netQualifiedDisasterLoss
```

### 11.5 Disaster-loss path — augmentation of standard deduction

```
standardDeduction = standardDeductionBase   (from computeStandardDeduction)

if scheduleA != null AND scheduleA.usedForStandardDeductionIncrease == true:
    standardDeduction = standardDeduction + scheduleA.netQualifiedDisasterLoss
```

The code's approach is *augmentation*: it adds the disaster loss to the standard deduction, then routes through the normal AUTO or STANDARD branch in `chooseLine12e`. The IRS recipe routes line 12e directly to the Schedule A line 16 combined amount. The two are functionally equivalent — `base + disaster_loss == schedule_a_line_16_combined` — but they differ in what gets printed on Schedule A:
- IRS recipe: Schedule A line 16 prints the combined amount; other Schedule A lines are blank.
- Code path: Schedule A line 16 contains only the disaster loss; the rest of Schedule A still contains the medical / taxes / interest / charity numbers (if any were entered) — but those numbers are never *used* on line 12e because the augmented standard deduction is larger.

### 11.6 Final 12e selection — `chooseLine12e()` (TaxReturnComputeService line 5626)

```
if deductionElection == "ITEMIZED":
    return roundMoney(itemizedDeductions)         (Schedule A line 17)
if deductionElection == "STANDARD":
    return roundMoney(standardDeduction)
return roundMoney(max(standardDeduction, itemizedDeductions))   (AUTO branch — tax-optimization default)
```

`deductionElection` is normalized to `AUTO`, `STANDARD`, or `ITEMIZED`. When the user has not made an explicit choice, the code falls into the AUTO branch and picks the larger of the two.

### 11.7 Output field assembly

```
Deductions.deductionAmount               = line12e (from chooseLine12e)
Deductions.deductionType                 = chooseDeductionType(deductionElection, standardDeduction, itemizedDeductions)
Deductions.standardDeductionComputed     = standardDeduction (post-augmentation)
Deductions.itemizedDeductionsFromScheduleA = scheduleA.totalItemizedDeductions
Deductions.deductionElection             = the normalized AUTO / STANDARD / ITEMIZED string
Deductions.line12aChecked                = line12a
Deductions.line12bChecked                = line12b
Deductions.line12cChecked                = line12c
Deductions.line12dBoxesCheckedCount      = line12dCount
```

Schedule A line 18 (the elect-to-itemize-although-less-than-standard flag) is set when:

```
deductionElection == "ITEMIZED" AND standardDeduction > 0 AND itemizedDeductions > 0 AND itemizedDeductions < standardDeduction
    => scheduleA.electsToItemizeAlthoughLessThanStandard = true
```
