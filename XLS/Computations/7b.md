## 1) What Line 7b is

Line 7b on the 2025 Form 1040 is NOT a money amount in its own right. It is a small disclosure strip that sits directly under Line 7a and consists of **three separate outputs**:

1. **Checkbox c1_43[0]** — "If you do not need to file Schedule D, check this box and enter the amount, if any, from Schedule D, line 13."
   In plain English: the taxpayer is telling the IRS "I did not need to file Schedule D this year, so the amount on Line 7a is just the capital gain distributions reported to me on Form 1099-DIV box 2a (and any Form 8814 line 10 the parent is reporting for a child)."

2. **Checkbox c1_44[0]** — "Includes child's capital gain or (loss)" — the second checkbox under the shared "Check if:" label on the actual 2025 Form 1040.
   In plain English: this box tells the IRS "Part of the amount on Line 7a is my child's capital gain distributions that I am reporting on my return using Form 8814 instead of making my child file." Per the 2025 Form 1040 PDF (verified 2026-06-06) the two Line 7b checkboxes appear as two **independent** boxes under a single "Check if:" label, with no text gating one on the other. The application correctly treats Output B as TRUE whenever Form 8814 line 10 > 0, regardless of the Schedule D path.

3. **Text entry-space f1_71[0]** — the dollar figure that prints on the same line as the second checkbox, carrying the Form 8814 line 10 amount. When the parent has more than one child for whom Form 8814 is filed (Form 8814 box C is checked indicating "more than one Form 8814 is attached"), this entry-space prints the **sum** of Form 8814 line 10 across all the children.

The three outputs together let the IRS know how Line 7a was computed without making the taxpayer attach Schedule D for the simple cases.

## 3) IRS computation for each Line 7b output (plain English)

### 3.1 Output A — Checkbox "Schedule D not required" (PDF field `line7_schedule_d_not_required`)

**Equation in plain English:**

```
line7b_schedule_d_not_required_checkbox = TRUE
    if and only if  ALL of the following hold for the taxpayer's return:
       (a) The taxpayer received no Form 1099-B sales / no Form 1099-DA sales
           and made no other sales or exchanges of capital assets during the year;
       (b) The taxpayer has no capital losses for the tax year and no prior-year
           capital loss carryover;
       (c) The taxpayer received no Form 2439 from a regulated investment
           company or real estate investment trust;
       (d) The taxpayer has no income items reportable through Form 6252, Form
           4797 Part I, Form 4684, Form 6781, Form 8824, or a Schedule K-1
           (Forms 1041 / 1065 / 1120-S) carrying capital gain or loss amounts;
       (e) The taxpayer has no Qualified Opportunity Fund deferral or
           termination activity (these always require Schedule D and Form
           8949 / Form 8997);
       (f) The taxpayer's Form 1099-DIV statements have no amounts in box 2b
           (unrecaptured §1250 gain), box 2c (§1202 qualifying gain), or
           box 2d (collectibles 28% gain). Box 2a alone is fine; the other
           boxes always force Schedule D so the worksheets for the 28%
           Rate Gain Worksheet and Unrecaptured §1250 Gain Worksheet can
           be applied.
       (g) The taxpayer's Form 1040 line 7a amount comes only from capital
           gain distributions (Form 1099-DIV box 2a totals, possibly with
           Form 8814 line 10 amounts added directly).
```

**Where the rule comes from:** 2025 Instructions for Form 1040, line 7 caption "If you do not need to file Schedule D, check this box and enter the amount..." together with the 2025 Schedule D Instructions list of conditions that require Schedule D to be filed.

**What the box prints:** a check mark when TRUE; nothing when FALSE.

### 3.2 Output B — Checkbox "Includes child's capital gain or (loss)" (PDF field `line7_includes_child_capital_gain_or_loss`)

**Equation in plain English (strict IRS reading):**

```
line7b_includes_child_cap_gain_checkbox = TRUE
    if and only if  BOTH:
       (a) the "Schedule D not required" checkbox above is TRUE
           (when Schedule D IS required, the child amount flows through
            Schedule D line 13 instead and the parent does not also check
            this box on Form 1040 line 7b); AND
       (b) the parent is filing one or more Forms 8814 for which the
           child's portion of capital gain distributions (Form 8814 line 10)
           is greater than zero.
```

**Application implementation reads this more liberally:** TRUE whenever Form 8814 line 10 > 0 regardless of whether Schedule D is required. The extra checkbox is harmless disclosure (it tells the IRS the parent is reporting a child amount on the return) but is broader than the strict IRS reading. See §6 M2.

### 3.3 Output C — Entry-space amount (PDF field `line7_schedule_d_note_text`)

**Equation in plain English:**

```
line7b_child_amount_from_form_8814_line_10 =
    sum over every Form 8814 the parent files of (Form 8814 line 10)

where for each individual Form 8814:

    line_4_total_child_income = child_taxable_interest (line 1a)
                              + child_ordinary_dividends (line 2a)
                              + child_capital_gain_distributions (line 3)

    if line_4_total_child_income >= 13,500 :
        do NOT file Form 8814 for this child  (child must file own return)
        contribute NOTHING to the line 7b entry-space

    if line_4_total_child_income == 0 :
        Form 8814 is not built for this child
        contribute NOTHING to the line 7b entry-space

    if line_4_total_child_income > 0 AND line_4_total_child_income <= 2,700 :
        skip Form 8814 lines 5-12 entirely → line 10 = 0
        contribute NOTHING to the line 7b entry-space

    if line_4_total_child_income > 2,700 :
        line_6 = line_4_total_child_income - 2,700
        line_8_ratio = child_capital_gain_distributions (line 3) / line_4_total_child_income
        line_10 = line_6 * line_8_ratio  (rounded to whole dollars)
        contribute line_10 to the line 7b entry-space
```

**Form-level format:** the PDF field is a free-text slot. The amount is printed as plain dollars (e.g. `30` or `1,250`), not enclosed in parentheses since this is always a positive amount on Form 8814.

**Where the rule comes from:** 2025 Instructions for Form 8814, Part I lines 4-10 plus the printed form caption on line 10: "Multiply line 6 by line 8. Enter the result here. See the instructions for where to report this amount on your return."

## 6) Computation the code actually implements

Reference: `C:\us-tax\us-tax-be\src\main\java\com\ustax\microservices\TaxReturnComputeService.java`.

### 6.1 Output A — `line7bScheduleDNotRequired`

The code computes a single boolean called `exception1` near line 10764:

```
exception1 = (NOT hasQofDeferral)
          AND (NOT hasSalesOrExchanges)
          AND (NOT hasCapitalLosses)
          AND (NOT hasCarryover)
          AND (NOT hasForm2439)
          AND (NOT hasOtherSources)
          AND (NOT hasSpecial1099DivBoxes)
          AND (NOT hasNonZeroAmount(legacyNetOtherAdjustments))
```

This is an 8-condition AND-gate. Each input has multiple supporting sources:

- `hasQofDeferral` is TRUE if the personal form indicates a Qualified Opportunity Fund deferral, OR any transaction is marked as a QOF transaction, OR any statement has a QOF indicator.
- `hasSalesOrExchanges` is TRUE if the personal form `hadCapitalAssetSalesOrExchanges` is TRUE, OR any Form 1099-B / 1099-DA entry exists, OR the Schedule D direct-aggregation rows (line 1a or line 8a) have non-zero values, OR there is at least one Form 8949 transaction.
- `hasCapitalLosses` is TRUE if the personal form indicates capital losses for the year, OR Schedule D net short-term line 7 is negative, OR net long-term line 15 is negative, OR net total line 16 is negative.
- `hasCarryover` is TRUE if the personal form indicates a prior-year carryover, OR any short / long / fallback carryover amount is non-zero.
- `hasForm2439` is TRUE if the personal form indicates Form 2439 was received, OR any Form 2439 statement entry exists.
- `hasOtherSources` is TRUE if the personal form indicates other Schedule D source forms, OR any 6252 / 4797 / 4684 / 6781 / 8824 / K-1 statement entry exists, OR any of the supplemental line 4 / 5 / 11 / 12 amounts is non-zero.
- `hasSpecial1099DivBoxes` is TRUE if any per-person aggregation shows non-zero box 2b / 2c / 2d on Form 1099-DIV.
- `legacyNetOtherAdjustments` is a legacy intake field for manual adjustments; when non-zero, Exception 1 is blocked.

The boolean `exception1` is then written into the Line 7b output via the orchestrator return value:

```
line7bScheduleDNotRequired = exception1 ? TRUE : FALSE
```

Implementation site for the assignment: `TaxReturnComputeService.java:10982` (the CapitalGainLossComputation record constructor, second argument) and the downstream mapping at `TaxReturnComputeService.java:7012-7013`.

### 6.2 Output B — `line7bIncludesChildCapitalGainLoss`

The code computes a single boolean called `includesChildCapitalGainLoss` that is TRUE whenever **any** Form 8814 with a positive line 10 amount is built, regardless of whether `exception1` is TRUE or FALSE:

```
includesChildCapitalGainLoss = (form8814Line10Total > 0)
```

The boolean is then written into the Line 7b output:

```
line7bIncludesChildCapitalGainLoss = includesChildCapitalGainLoss ? TRUE : FALSE
```

Implementation site: `TaxReturnComputeService.java:10983` (third constructor argument) and `TaxReturnComputeService.java:7014`.

### 6.3 Output C — `line7bChildAmountFromForm8814Line10`

The code first builds zero or more Form 8814 objects via `buildForm8814()` at `TaxReturnComputeService.java:11093`. The relevant arithmetic is:

```
line_4 = (line_1a or 0) + (line_2a or 0) + (line_3 or 0)

if line_4 is null or zero :
    return no Form 8814 for this child

if line_4 >= 13,500 :
    emit blocking flag FORM8814_CHILD_GROSS_INCOME_TOO_HIGH
    return no Form 8814 for this child

if line_4 > 2,700 :
    line_6 = line_4 - 2,700
    if line_2b > 0 OR line_3 > 0 :
        line_7_ratio = (line_2b or 0) / line_4    (10 decimal places)
        line_8_ratio = (line_3  or 0) / line_4    (10 decimal places)
        line_9  = round_money(line_6 * line_7_ratio)
        line_10 = round_money(line_6 * line_8_ratio)
    else :
        line_9 = 0
        line_10 = 0
```

The application then sums `line_10` across every Form 8814 it built:

```
form8814Line10Total = sum over all built Form 8814 of getLine10CapGainDistributions()
```

That sum is rounded once more and written into the Line 7b entry-space:

```
line7bChildAmountFromForm8814Line10 = round_money(form8814Line10Total)
```

Implementation sites: `buildForm8814()` at `TaxReturnComputeService.java:11093-11163`, the cross-child sum at the orchestrator, and the mapping at `TaxReturnComputeService.java:7015`.

### 6.4 Downstream coupling

Output C is more than just a display field. It also participates in Line 7a:

```
if exception1 == TRUE AND form8814Line10Total > 0 :
    line7a = line7aBase + form8814Line10Total           // direct addition

if exception1 == FALSE AND form8814Line10Total > 0 :
    Schedule_D_line_13 = Schedule_D_line_13 + form8814Line10Total
    Schedule_D_line_15 = recompute including the new line 13
    Schedule_D_line_16 = recompute
    line7a = line7aBase (the recomputed line 16)        // indirect via Schedule D
```

In either path the entry-space disclosure on Line 7b is the same number. This is the Phase 1 fix that landed 2026-04-16 — before the fix the code added the child amount directly to Line 7a even on the Schedule D path, which double-counted (once via direct add, once because Schedule D line 13 already included it).

## 7) Code-vs-IRS gap analysis

### 7.1 Mismatches against the IRS rule

**M1 — None for Output A.** The 8-condition AND-gate matches the IRS Line 7 caption and the Schedule D Instructions list of triggers exactly. The eighth condition (`legacyNetOtherAdjustments`) is defensive — there is no IRS-prescribed trigger named "legacy net other adjustments"; it exists to catch any non-zero manual entry that would invalidate the simple direct-entry path.

**M2 — RECLASSIFIED 2026-06-06 as NOT A MISMATCH.** The original draft of this file claimed Output B was overly broad because the application sets the "Includes child's capital gain or (loss)" checkbox TRUE regardless of whether Schedule D was required. After verifying against the actual 2025 Form 1040 PDF (rendered image checked 2026-06-06), Line 7b shows **two independent checkboxes** under a single `Check if:` label, with no language gating one on the other. The application's path-independent behavior is therefore correct per the literal form layout. No code change.

**M3 — RECLASSIFIED 2026-06-06 as NOT A MISMATCH.** Same root cause as M2: the original draft assumed the entry-space was tied to the no-Schedule-D path, but the actual 2025 Form 1040 PDF shows the entry-space sits next to the "Includes child's capital gain or (loss)" checkbox, which is independent of the "Schedule D not required" checkbox. Whenever the child amount is being included on the return, the entry-space should print the included amount — regardless of which path the amount took. The closely-related S3 below DOES still apply (write `null` to the entry-space when no child amount is being included).

### 7.2 Missing inputs / fields

**G1 — CLOSED 2026-06-06.** Added `boxCMultipleFormsAttached` Boolean field to `com.ustax.model.output.Form8814` (with getter / setter). After the Form 8814 list is built, the orchestrator now sets the flag TRUE on every entry when `form8814List.size() > 1`. - Relationship must be in `EIC_QUALIFYING_RELATIONSHIPS` (already covers son / daughter / stepchild / sibling / half-sibling / step-sibling / grandchild / niece / nephew) OR start with "foster child".
- Age at year end must be under 19, OR under 24 with `isFullTimeStudent == true`, OR `isPermanentlyAndTotallyDisabled == true`.

### 7.3 Shortcomings

**S1 — CLOSED 2026-06-06.** Added the new advisory flag `FORM8814_MFS_ELECTION_REVIEW` (non-blocking) at the orchestrator immediately after `computeChildInterestDividends` returns. Fires when `isMfsReturn` is true AND the Form 8814 list is non-empty. The flag message reminds the user that only one parent may elect Form 8814 per child per year, and asks them to confirm with the other spouse.

**S2 — CLOSED 2026-06-06.** Added a defensive non-negative clamp inside `buildForm8814` immediately before the per-line setters: `line9 = line9.max(BigDecimal.ZERO); line10 = line10.max(BigDecimal.ZERO); line12 = line12.max(BigDecimal.ZERO);`. By construction these values are non-negative (line 6 ≥ 0, ratios ≥ 0), but the clamp hardens the invariant against future refactors and against rounding edge cases.

**S3 — CLOSED 2026-06-06.** The orchestrator's CapitalGainLossComputation constructor call now writes `null` (not `roundMoney(childAmount)`) to the line 7b entry-space when `includesChildCapitalGainLoss == false`. New helper variable `BigDecimal line7bChildAmount = includesChildCapitalGainLoss ? roundMoney(childAmount) : null;` makes the dual-path explicit. The PDF entry-space now stays blank when the checkbox is unchecked, matching the printed 2025 Form 1040 layout.

**S4 — CLOSED 2026-06-06.** Added an internal-consistency invariant check at the bottom of the orchestrator: if `exception1 == (scheduleD != null)` the application emits a new blocking flag `LINE7B_INTERNAL_INCONSISTENCY` containing the observed values of both sides. By construction `exception1 == TRUE` should always coincide with `scheduleD == null` (no Schedule D built) and vice versa; the invariant catches a future refactor that breaks this coupling rather than letting a misleading PDF ship.
