# Form 1040 — Line 1b

**UI label:** *Household employee wages not reported on Form(s) W-2*
**Semantic 1040 field:** `line1b_household_employee_wages`
**Output field path:** `form1040.income.householdEmployeeWages`
**Output PDF field:** `topmostSubform[0].Page1[0].f1_48[0]`
**Category:** earned income → wages received as a household employee with no W-2
**Tax year:** 2025

---

## 1) Authoritative IRS definition

**2025 Form 1040 Instructions, line 1b:**

> Enter the total of your wages received as a household employee that was not reported on Form(s) W-2. ([IRS][1])

Line 1b is **not** a "threshold line." It is a **reporting location**: household-employee wages **with no W-2**.

---

## 2) Why a W-2 may not exist (employer reporting threshold)

The employer-side W-2 obligation is triggered at:

* **Tax year 2024:** wages < $2,700 → employer is not required to issue W-2. ([IRS][2])
* **Tax year 2025:** wages < $2,800 → employer is not required to issue W-2. ([IRS][1])

### Critical clarification — the threshold is NOT an eligibility gate

| Misconception | Truth |
|---|---|
| "Wages below $2,800 are reported on line 1b" | WRONG. Threshold has nothing to do with eligibility. |
| "Wages at or above $2,800 must come from a W-2" | PARTIAL. Above $2,800, the EMPLOYER is required to issue a W-2; if they did not, the worker still reports the wages on line 1b and is offered "Missing W-2" guidance. |
| "Line 1b only applies if no W-2 was received" | CORRECT. Regardless of amount: household-employee wages with no W-2 belong on line 1b. |

**Implementable rule:** If it's household employee wages and **no W-2 reports them**, they belong on **Line 1b — regardless of amount**. ([IRS][1])

---

## 3) Plain-English formula

```text
Line 1b = sum of household-employee wages received by taxpayer (and spouse on MFJ),
          where (a) the worker is a household employee (passes the IRS control test, NOT self-employed),
                (b) NO Form W-2 was received for those wages, AND
                (c) the employer did NOT treat the worker as a contractor
                    (those wages route to Form 8919 → line 1g instead).
```

---

## 4) Step-by-step computation (per person)

(See `XLS/Computations/1b.xlsx` "Main Computation" sheet for the canonical map.)

| Step | Operation | Formula | Notes |
|---|---|---|---|
| 1 | Outer gate: household work answered Yes | `if employmentForm.householdWork ≠ true → return null` | UI also gates on `hasEmploymentIncome=true` upstream |
| 2 | Control-test gate | `if employmentForm.householdEmployeeUnderControlTest ≠ true → return null` | When explicitly false, `validateHouseholdEmployeeControlTest()` ALSO emits `HOUSEHOLD_WORK_SELF_EMPLOYMENT_{TAXPAYER,SPOUSE}` blocking flag; wages route to Schedule C (out of scope) |
| 3 | W-2 gate: must be explicitly "no W-2" | `if employmentForm.householdReceivedW2 ≠ false → return null` | true → wages belong on line 1a via W-2 entry |
| 4 | Per-employer loop: collision guard | `for each employer in householdEmployers[]: if employer.employerTreatedAsContractor == true → SKIP` | Wages route to Form 8919 → line 1g |
| 5 | Per-employer loop: accumulate wages | `total = Σ employer.wages` (for non-contractor entries only) | Absent/null = $0 |
| 6 | MFS guard for spouse | `if filingStatus == "Married filing separately" → spouse contribution forced to null` | Each MFS filer reports only their own household wages |
| 7 | Aggregate taxpayer + (non-MFS) spouse | `householdWagesRaw = addNonNull(taxpayerAmount, spouseAmount)` | Null + null = null |
| 8 | Round to whole dollars | `line1b = roundMoney(householdWagesRaw)` | HALF_UP |
| 9 | Persist on Form 1040 | `income.setHouseholdEmployeeWages(line1b)` only when non-null | Field omitted when null |

## 5) Routing decision tree

| Worker situation | Answer pattern | Routes to |
|---|---|---|
| Household employee, received W-2 | `householdWork=Y, controlTest=Y, receivedW2=Y` | **Line 1a** (via W-2 statement entry) |
| Household employee, NO W-2 | `householdWork=Y, controlTest=Y, receivedW2=N, employerTreatedAsContractor=N` | **Line 1b** ← THIS COMPUTATION |
| Household worker, employer issued 1099 instead | `householdWork=Y, controlTest=Y, receivedW2=N, employerTreatedAsContractor=Y` | **Line 1g** (via Form 8919) |
| Independent domestic-services business (self-employed) | `householdWork=Y, controlTest=N` | **Schedule C** (OUT OF SCOPE — blocking flag) |
| Not household work | `householdWork=N` | Line 1a (W-2 path) |
| User hasn't answered | any null in chain | Line 1b returns null |

## 6) Classification gate: household employee vs self-employed

**Schedule H instructions (employer-side, but defines relationship):**

> Self-employed workers are also not your employees. A worker is self-employed if only the worker can control how the work is done. ([IRS][3])

If the worker operates an independent domestic-services business (sets methods, offers services publicly, provides tools, etc.), route to **Schedule C**, not Line 1b.

When `householdEmployeeUnderControlTest = false`, `validateHouseholdEmployeeControlTest()` emits `HOUSEHOLD_WORK_SELF_EMPLOYMENT_{TAXPAYER,SPOUSE}` blocking flag and the wages are excluded from line 1b.

## 9) Side-effect outputs / cross-line interactions

Line 1b is a leaf computation — it produces no Schedule 1 carve-outs. But it INTERACTS with adjacent lines via gates and collision guards.

| Interaction | Direction | Mechanism |
|---|---|---|
| Line 1a missing-W-2 flag suppression | line 1b → line 1a | When `householdWork = true`, `validateEmploymentIncomeW2ForPerson()` suppresses `MISSING_W2_EMPLOYMENT_INCOME_*` |
| Line 1g (Form 8919) collision guard | line 1b → line 1g | Per-employer `employerTreatedAsContractor = true` skips entry in line 1b loop; wages flow to Form 8919 line 6 → line 1g |
| Schedule C out-of-scope routing | line 1b → Schedule C | `HOUSEHOLD_WORK_SELF_EMPLOYMENT_*` blocking flag; wages excluded from line 1b |
| Form 1040 line 1z aggregation | line 1b → line 1z | `line1z = 1a + 1b + 1c + 1d + 1e + 1f + 1g + 1h` (addNonNull) |

| Pub 915 SS Worksheet line 3 | line 1b → SS taxability | Worksheet line 3 references line 1z; `computeSocialSecurityBenefits` adds household wages via `sumHouseholdEmployeeWages()` with the same MFS guard |

## 12) "Missing W-2" workflow

If wages are high or the user expected a W-2:

* Allow reporting (so the return can be prepared);
* Surface the 1040 instructions "Missing or Incorrect Form W-2?" guidance. ([IRS][1])

### Federal-withholding strong signal

Schedule H instructions indicate that if **federal income tax was withheld** from household employee wages, the employer has filing obligations (including W-2 reporting) **regardless of the $2,800 threshold**. ([IRS][3])

**Implemented per-employer (2026-05-05):** each entry in `householdEmployers[]` carries a `federalTaxWithheld` boolean. When `federalTaxWithheld === true` for a specific employer (and that employer is NOT contractor-treated), a strengthened warning appears citing the employer by name:

> "{Employer name} was required to issue a W-2 regardless of the $2,800 threshold because they withheld federal income tax (per IRS Schedule H rules). Contact them to request a W-2. If they refuse or cannot be reached, file Form 4852 (Substitute W-2)."

This field is UX-only — line 1b compute does not read it.

## 14) Test scenarios that should exist

1. Single household-employee row, no W-2 → line 1b = that amount.
2. Three employers, one contractor-treated, two normal → line 1b = sum of two normal rows.
3. `controlTest=false` → returns null AND emits `HOUSEHOLD_WORK_SELF_EMPLOYMENT_*` blocking flag.
4. `receivedW2=true` → returns null (wages belong on 1a).
5. MFJ both spouses with household employers → aggregated.
6. MFS with spouse stale household data → spouse contribution dropped.
7. `householdWork=null` (legacy data) → no flag, contributes $0.
8. `federalTaxWithheld=true` on an employer → UI warning rendered; compute unaffected.

## 17) Final condensed rule map

1. Include wages paid to the taxpayer as a household employee (work in/around a home where payer controls what/how work is done). ([IRS][3])
2. Only include amounts not reported on any Form W-2. ([IRS][1])
3. Compute as the sum of all such wage amounts across all household employers. ([IRS][1])
4. Do not use the $2,700 / $2,800 threshold as eligibility. ([IRS][1], [2])
5. Exclude / redirect:
   * If reported on W-2 → Line 1a.
   * If using Form 8919 → wages go to Line 1g (collision guard).
   * If self-employed (independent business) → Schedule C, not Line 1b.
6. Apply MFS guard so spouse household wages do not aggregate on MFS.
7. Federal-withholding "strong signal" — warn the user (employer was obligated to file W-2 regardless of threshold). ([IRS][3])
