# Form 1040 Line 1g - Wages from Form 8919

## 1. Line identity

- Form: `1040 / 1040-SR / 1040-NR`
- Line: `1g`
- Label: `Wages from Form 8919, line 6`
- Source of truth: `Form 8919 (2025), line 6`
- Direct mapping: `Form1040.Line1g = addNonNull(taxpayer.Form8919.Line6, spouse.Form8919.Line6)`
- Semantic PDF field: `topmostSubform[0].Page1[0].f1_53[0]` (key `line1g_wages_form8919_line6`)
- Output field path: `form1040.income.uncollectedSocialSecurityMedicareWages`
- Backend orchestrator: `TaxReturnComputeService.computeUncollectedSSTax(...)` declared at line 19422
- Per-spouse helper: `TaxReturnComputeService.computeForm8919ForPerson(...)` declared at line 19454

This is a 2025 line spec. Use 2025 Form 8919 line numbers and the 2025 Social Security wage base (`$176,100`).

## 2. What line 1g is

Line 1g is for wages that should have been treated as employee wages, but the employer did not withhold the employee share of Social Security and Medicare tax. The amounts are computed on `Form 8919` and carried to:

- `Form 1040 line 1g` from `Form 8919 line 6` (the WAGE portion), and
- `Schedule 2 line 6` from `Form 8919 line 13` (the TAX portion — uncollected employee FICA share).

If Additional Medicare Tax applies, `Form 8919 line 6` also flows to `Form 8959 line 3` (Medicare wages).

Conceptually: the worker pays BOTH the income tax (via line 1g → AGI) AND the employee FICA share (via Schedule 2 line 6) that the firm should have withheld but didn't.

## 3. When Form 8919 is required

A person must file Form 8919 if all of the following are true:

- the person performed services for a firm,
- the person believes the pay should have been treated as employee wages,
- the firm did not withhold the employee share of Social Security and Medicare tax, and
- one of the IRS reason codes `A`, `C`, `G`, or `H` applies.

## 4. What Form 8919 is not for

Do not use Form 8919 for:

- true independent-contractor income (`Schedule C / Schedule SE` — IRC §1402)
- unreported tip income (`Form 4137` — IRC §6053(a))
- W-2 box 12 codes `M` and `N` (group-term life insurance uncollected taxes for former employees — Schedule 2 line 5)

## 5. Reason codes

Each firm row on Form 8919 requires one of the following reason codes (Form 8919 instructions):

| Code | Meaning | IRS date required? | Notes |
|---|---|---|---|
| `A` | IRS issued a determination letter saying the worker is an employee | YES (column d) | |
| `C` | IRS issued other correspondence saying the worker is an employee | YES (column d) | |
| `G` | Worker filed Form SS-8 and has not yet received a reply | n/a | SS-8 must be filed SEPARATELY (out of scope for this app) |
| `H` | Worker received both a W-2 and a 1099-MISC/NEC from the same firm for the same services, and the 1099 amount should have been W-2 wages | n/a | SS-8 should NOT be filed |

Validation behavior in `computeForm8919ForPerson`:

- code must be in `{A, C, G, H}` after normalize (trim + upper); otherwise `FORM8919_REASON_CODE_INVALID_*` (blocking)
- codes `A` and `C` require `irsDeterminationDate`; otherwise `FORM8919_IRS_DATE_REQUIRED_*` (blocking)
- code `G` emits non-blocking advisory `FORM8919_SS8_REQUIRED_*`
- code `H` without `received1099MiscOrNec=true` emits non-blocking soft warning `FORM8919_CODE_H_1099_*`

## 6. Per-spouse multiplicity

Form 8919 is person-specific.

- Non-joint return: 0 or 1 Form 8919
- MFJ return: 0, 1, or 2 Forms 8919

If both spouses qualify, each completes a separate Form 8919:

- `line 1g = addNonNull(taxpayer.Form8919.line6, spouse.Form8919.line6)`
- `Schedule 2 line 6 = addNonNull(taxpayer.Form8919.line13, spouse.Form8919.line13)`

On MFS, spouse files separately; the spouse contribution is gated by `isMfsReturn` at the orchestrator (line 19442, passing `null` for spouse data so `computeForm8919ForPerson` early-returns an empty result).

## 7. Inputs needed for line 1g

### 7.1 Per-firm user inputs (form `uncollected-ss-medicare-{taxpayer,spouse}`)

For each firm row:

- `firmName` (column a)
- `firmFederalIdNumber` (column b; "unknown" acceptable)
- `reasonCode` (column c; one of `A`, `C`, `G`, `H`)
- `irsDeterminationDate` (column d; required for codes A and C)
- `received1099MiscOrNec` (column e)
- `wagesNoFicaNotW2` (column f — the wage amount)

### 7.2 Imported / computed inputs (feed Form 8919 line 8)

- `W-2 box 3` Social Security wages (SSN-filtered via `sumW2SocialSecurityWagesAndTipsForSsn`)
- `W-2 box 7` Social Security tips (same helper)
- `W-2 box 14` RRTA tier-1 compensation (`sumW2RrtaCompensationForSsn`)
- `Form 4137 line 10` unreported tips subject to Social Security tax — cross-form dependency: `computeTips()` MUST run BEFORE `computeUncollectedSSTax()` (execution order enforced in `prepare()`)

### 7.3 Identity / status inputs

- `you.ssn` / `spouse.ssn` (drives W-2 SSN filtering)
- `filingStatus` → `isMfsReturn` → spouse contribution gate

## 8. Core 2025 computation per spouse

### 8.1 Line 6 — wages flowing to Form 1040 line 1g

`Form 8919 line 6` is the total of all column (f) wage amounts for that person.

`line6 = Σ parseAmount(firm.wagesNoFicaNotW2)` (via `addNonNull`, skipping null rows)

### 8.2 Form 8919 tax computation feeding Schedule 2

| Line | Formula | Source |
|---|---|---|
| 7 | `$176,100` | `ReferenceData.SOCIAL_SECURITY_WAGE_BASE` (2025 SSA wage base) |
| 8 | `sumW2BoxSSWagesAndTipsForSsn + sumW2RrtaForSsn + Form4137.line10` | W-2 box 3 + box 7 + RRTA + Form 4137 line 10 |
| 9 | `max(0, line7 − line8)` | Remaining SS wage base |
| 10 | `min(line6, line9)` | Wages subject to SS tax (uncollected) |
| 11 | `line10 × 0.062` | 6.2% employee SS tax (IRC §3101(a)) |
| 12 | `line6 × 0.0145` | 1.45% employee Medicare tax (IRC §3101(b)(1)); no SS wage-base cap |
| 13 | `line11 + line12` | Per-person total uncollected tax |

All lines are whole-dollar HALF_UP rounded via `roundMoney` on each output line. The 6.2% and 1.45% rates are hard-coded inline (lines ~20351-20352), not in `ReferenceData`.

### 8.3 Return-level aggregation

```
form1040.income.uncollectedSocialSecurityMedicareWages
    = addNonNull(taxpayer.line6, spouse.line6)

schedule2.otherTaxes.uncollectedSocialSecurityMedicareTaxOnWages
    = addNonNull(taxpayer.line13, spouse.line13)
```

### 8.4 Form 8959 integration

When Additional Medicare Tax applies (Medicare wages > applicable threshold):

```
Form8959.Line3 = mfj
    ? addNonNull(taxpayer.line6, spouse.line6)
    : taxpayer.line6
```

2025 Additional Medicare thresholds (`ReferenceData.ADDITIONAL_MEDICARE_THRESHOLD_*`):

| Status | Threshold | Constant |
|---|---|---|
| Single | `$200,000` | `_SINGLE` |
| MFJ | `$250,000` | `_MFJ` |
| MFS | `$125,000` | `_MFS` |
| HOH | `$200,000` | `_HOH` |
| QSS | `$200,000` | `_QSS` |

Rate: `0.9%` (`ReferenceData.ADDITIONAL_MEDICARE_RATE`; IRC §3101(b)(2)).

## 9. Multi-page rule for more than 5 firms

A single Form 8919 PDF holds up to 5 firm rows.

If one person has more than 5 firms:

- attach additional Forms 8919 for the extra rows (lines 1–5 only on overflow pages),
- complete lines 6–13 on only one main Form 8919 using combined totals.

Engine rule: line 6 must equal the combined total of all column (f) rows across all pages for that person; lines 7–13 are computed once per person. UI multi-page handled via `buildMultiPagePdf` in `form-tax-return-8919-*.component.ts`.

## 10. Validation flags

| Flag code | Trigger | Blocking? |
|---|---|---|
| `FORM8919_REASON_CODE_INVALID_TAXPAYER` / `_SPOUSE` | Reason code missing or not in `{A, C, G, H}` | YES |
| `FORM8919_IRS_DATE_REQUIRED_TAXPAYER` / `_SPOUSE` | Code A or C with missing IRS date | YES |
| `FORM8919_SS8_REQUIRED_TAXPAYER` / `_SPOUSE` | Reason code G (informational) | NO |
| `FORM8919_CODE_H_1099_TAXPAYER` / `_SPOUSE` | Code H with `received1099MiscOrNec != true` | NO |

Non-flag defensive behaviors:

- `data == null` → `computeForm8919ForPerson` returns `Form8919Result(null, null, null)` (silent)
- `firmEntries` empty → same silent return
- All rows empty (no `hasAnyForm8919RowInput`) → silent return
- `wagesNoFicaNotW2` null for a firm → 0 contribution
- `W-2 box 3/7` absent for spouse SSN → 0 line 8 contribution

## 11. Cross-line collision protection

The implementation must prevent the same wages from being counted again in another line-of-return path:

- **Line 1b household-employee guard**: `householdEmployeeAmount` skips employer rows where `reportedOnForm8919 == true` (prevents double-counting on lines 1b and 1g)
- **Line 1c Form 4137 tip collision**: NOT currently guarded — same wage row entered as both an unreported tip on Form 4137 AND a Form 8919 firm row would double-count (Code Validation #3; deferred advisory)
- **W-2 box 12 codes M/N (group-term life)**: NOT routed via Form 8919; belong on Schedule 2 line 5 (out of scope for line 1g)

## 12. Compute order

Sequence in `prepare()`:

1. `computeTips()` → produces `TipComputation` with `form4137Taxpayer.line10UnreportedTipsSubjectSocialSecurity` and spouse counterpart
2. `computeUncollectedSSTax(uncollectedTaxpayer, uncollectedSpouse, you, spouse, w2Entries, tips, isMfsReturn, flags)` at line 413 — consumes `tips` for Form 8919 line 8
3. `buildIncome()` at line ~7361 aggregates `line1g` into `line1z` via `addNonNull`
4. `line9 = addNonNull(line1z, line2b, ...)` at line 7440
5. Form 8959 (Additional Medicare Tax) computation reads `Form 8919 line 6` into `line3`
6. Schedule 2 line 6 and line 7 subtotals are emitted

## 14. Downstream impact

- Line 1g → line 1z → line 9 → AGI → line 11b → line 15
- Form 8919 line 13 → Schedule 2 line 6 → Schedule 2 total → Form 1040 line 23
- Form 8919 line 6 → Form 8959 line 3 → 0.9% Additional Medicare Tax when total Medicare wages > threshold
- Line 1g → EIC earned income (line 27a; IRC §32 — Form 8919 wages qualify as wages per IRC §3121)
- Line 1g → Schedule 8812 ACTC earned income (line 18a)
- Line 1g → Form 2441 earned income (IRC §21)

## 15. Negative / null contract

| Value | Meaning |
|---|---|
| `null` | No Form 8919 generated for either spouse |
| `BigDecimal.ZERO` | Edge case: Form 8919 generated but all `wagesNoFicaNotW2 = 0` |
| Positive | Standard case — at least one firm with positive wages |
| Negative | IMPOSSIBLE — wages are sums of user-entered amounts (each ≥ 0 by UI validation) |

## 16. Out of scope

- W-2 box 12 codes M/N (group-term life insurance uncollected taxes) — Schedule 2 line 5, not Form 8919
- Form SS-8 worker classification filing (separate IRS process; gated as code G advisory)
- True independent-contractor income (Schedule C / Schedule SE)
- Unreported tip income other than the Form 4137 line 10 contribution to Form 8919 line 8
