# Form 1040 Line 1h - Other earned income

## 1. Line identity

- Form: `1040 / 1040-SR`
- Line: `1h`
- Label: `Other earned income (see instructions)`
- Concept: catch-all wages slot for earned-income items that belong on the wages section of Form 1040 but do not fit lines 1aâ€“1g
- Semantic PDF fields:
  - Amount: `topmostSubform[0].Page1[0].f1_55[0]` (key `line1h_other_earned_income`)
  - Write-in: `topmostSubform[0].Page1[0].f1_54[0]` (key `line1h_statement_text`)
- Output field paths: `form1040.income.otherEarnedIncome` + `form1040.income.otherEarnedIncomeStatements` (List<String>)
- Backend method: `TaxReturnComputeService.computeOtherEarnedIncome(...)` declared at line 19100
- Call site: line 425 in `prepare()`

This is a 2025 line spec.

## 2. What belongs on line 1h for 2025

Per the 2025 Form 1040 instructions, line 1h includes four categories (mutually non-overlapping in this implementation):

| # | Category | Source | Backend variable |
|---|---|---|---|
| Cat 1 | Strike or lockout benefits (other than bona fide gifts) | Manual entry + 1099-MISC box 3 with `box3IsStrikeBenefits=true` | `strikeTotal` |
| Cat 2 | Excess elective deferrals (pre-tax Â§402(g) / SIMPLE / Â§457(b)) | W-2 box 12 codes `D`, `E`, `F`, `S`, `G` | `excessDeferrals` |
| Cat 3 | Disability pension wages â€” 1099-R code `3` recipient BELOW plan minimum retirement age | 1099-R box 2a (preferred), box 1 fallback | `disabilityTotal` |
| Cat 4 | Corrective distributions â€” 1099-R code `8` (current year only) | 1099-R box 2a; non-IRA only | `correctiveTotal` |

`line1h = roundMoney(addNonNull(addNonNull(addNonNull(disabilityTotal, excessDeferrals), correctiveTotal), strikeTotal))`

## 3. What does not belong on line 1h

The 2025 Form 1040 instructions explicitly route these elsewhere:

- Scholarship / fellowship grants not on W-2 â†’ `Schedule 1 line 8r`
- IRA distributions â†’ `Form 1040 lines 4a / 4b` (gated via `iraSepSimple` flag on 1099-R)
- Disability pension payments received AT or AFTER plan minimum retirement age â†’ `Form 1040 lines 5a / 5b`
- Other pension/annuity payments on 1099-R â†’ `lines 5a / 5b`
- Inmate wages â†’ `Schedule 1 line 8u`
- NQDC (W-2 box 11) â†’ `Schedule 1 line 8t`
- Prior-year corrective distributions (1099-R code `P`) â†’ prior-year return (amend if missed)
- Designated Roth excess (W-2 box 12 codes `AA`, `BB`, `EE`) â†’ already in W-2 box 1; adding to line 1h would double-count

## 4. Top-level formula

```
line1hRaw = addNonNull(addNonNull(addNonNull(disabilityTotal, excessDeferrals), correctiveTotal), strikeTotal)
line1h    = roundMoney(line1hRaw)   // whole-dollar HALF_UP
```

`addNonNull` semantics: `null + null = null`; `null + X = X`; `X + Y = X+Y`. So when no category contributes, `line1h = null` and the framework omits the field.

## 6. Retired Public Safety Officer (PSO) exclusion

The 2025 Form 1040 instructions for lines 5a/5b state: if a taxpayer is retired on disability and reports the disability pension on line 1h, they include only the taxable amount on line 1h and enter `PSO` plus the excluded amount next to line 1h.

**Implementation (2026-06-01).**

- `ReferenceData.PSO_PREMIUM_EXCLUSION_CAP = $3,000` (statutory; not annually-indexed; IRC Â§402(l))
- `computeOtherEarnedIncome` reads per-person PSO sub-objects from `data.disabilityPensions.psoExclusionTaxpayer` and `data.disabilityPensions.psoExclusionSpouse`
- Helper `computeLine1hPsoExclusion` enforces the `$3,000` cap and clamps to the person's disability total via `subtractNonNegative` (line 1h never goes negative from PSO)
- **Double-gate**: requires BOTH `isEligibleRetiredPublicSafetyOfficer == true` AND `electsPsoPremiumExclusion == true`
- Statement label switches to `"Disability pension wages (PSO)"` when any exclusion is applied
- MFS guard: spouse PSO sub-object skipped on MFS

PSO interaction with line 5b: in the common case a single 1099-R goes to line 1h XOR line 5b based on the below-min-age gate, so independent `$3,000` caps produce the correct answer. Cross-line shared-cap with line 5b PSO is a documented future enhancement.

## 7. Category 1 â€” Strike/lockout benefits

### Rule

IRC Â§61 + Pub. 525: strike pay from a union is taxable wages. Bona fide gifts are excludable, but the typical treatment is wages on line 1h.

### Inputs

- Manual entry: `strikeBenefits.manualStrikeBenefitsAmount` (form `other-earned-income`)
- Imported: 1099-MISC entries where `box3IsStrikeBenefits == true` (user-flagged checkbox on the 1099-MISC statement form)

### Computation


## 9. Category 3 â€” Disability pension wages

### Rule (IRC Â§105(c); Reg Â§1.105-1; Pub. 525)

Periodic disability pension payments received BEFORE the plan's minimum retirement age are taxable wages for income tax purposes (line 1h). AT or AFTER the minimum retirement age, they convert to pension income (lines 5a/5b).

### Inputs

From Form 1099-R:

- `grossDistributionAmount` (box 1)
- `taxableAmountAmount` (box 2a) â€” preferred
- `distributionCodes` (box 7) â€” must contain `"3"`

## 10. Category 4 â€” Corrective distributions

### Rule (Reg Â§1.402(g)-1(e)(8); Pub. 525)

Excess deferrals returned with allocable income BEFORE April 15 of the following year are reported as taxable wages on the current-year return (code `8` path). After April 15, the excess is taxable in the deferral year (code `P`) plus when finally distributed.

### 1099-R code timing

| Code | Meaning | Line 1h impact |
|---|---|---|
| `8` | Excess contributions / deferrals taxable in CURRENT year | INCLUDED in line 1h (non-IRA only) |
| `P` | Excess taxable in PRIOR year | EXCLUDED â€” belongs on prior-year return |

### Inputs

From Form 1099-R: `taxableAmountAmount` (box 2a), `distributionCodes`, `iraSepSimple`, `recipientTIN`.

Per-1099-R override:

- `correctiveDistributions.entries[].source1099RId`
- `correctiveDistributions.entries[].owner` (required when TIN doesn't match either SSN)

### Computation

```
hasCurrentCorrective = codes.contains("8")
hasPriorCorrective    = codes.contains("P")
iraSepSimple = (rEntry.iraSepSimple == true)
if (hasCurrentCorrective || hasPriorCorrective) AND !iraSepSimple AND !codes.contains("3"):
  // "!codes.contains(\"3\")" guard: combined code 3+8 routes ONLY through Cat 3 (disability wins)
  if hasCurrentCorrective:
    if owner == null:
      owner = normalize(selection.owner)
      if empty â†’ missingCorrectiveOwner = true
    correctiveTotal = addNonNull(correctiveTotal, parseAmount(rEntry.taxableAmountAmount))
  // code-P-only entry: iterated, but NO amount added to line 1h (prior-year correction)
```

### Cat 3 vs Cat 4 asymmetry (deliberate)

- Cat 3 (disability) falls back to box 1 (gross) when box 2a is null
- Cat 4 (corrective) does NOT fall back â€” per IRS 1099-R instructions, issuers MUST populate box 2a for code-8 corrective distributions (corrective amount must be split from any non-taxable basis)

Lock-in test `computesLine1hCorrectiveDistribution_box2aEmpty_excludedNoFallback` prevents accidental mirroring. (Code Validation #9, 2026-05-10.)

## 11. Mutual exclusion rules (enforced by code)

| Rule | Implementation | Rationale |
|---|---|---|
| Disability + corrective on same 1099-R | `!codes.contains("3")` guard in Cat 4 | Prevents double-counting when 1099-R box 7 contains both `3` and `8`; disability wins |

## 13. MFJ attribution rules

Line 1h is return-level on Form 1040, but several source records are person-specific. The implementation attributes via SSN normalization (digits-only):

- W-2 box 12 deferrals â†’ `sumW2Box12ByCodes(w2Entries, codes, ssn)` filters by `employeeSSN`

On MFS, the orchestrator passes `isMfsReturn=true` to `computeOtherEarnedIncome`; the helper nulls spouseSsn + spouseAge and adds an explicit `continue` filter at the top of the 1099-R loop when `recipientTIN == spouseSsnRaw` (three-point gate per Code Validation #1, 2026-05-10).
