# Form 1040 (2025) - Line 22: Tax After Nonrefundable Credits

> Source of truth: 2025 Form 1040, 2025 Schedule 8812, 2025 Schedule 3, and the 2025 Instructions for Form 1040.

---

## 1. Line Identity

**Form 1040 line 22** is:

```text
Subtract line 21 from line 18. If zero or less, enter -0-
```

Exact mapping:

```text
Form1040.line22 = max(0, Form1040.line18 - Form1040.line21)
```

This is the return's **tax after page-2 nonrefundable credits** and before adding `line 23` other taxes. It is **not** the final total tax â€” that role belongs to `line 24`.

---

## 2. Core 2025 Formula

```text
Form1040.line18 = Form1040.line16 + Form1040.line17
Form1040.line21 = Form1040.line19 + Form1040.line20
Form1040.line22 = max(0, Form1040.line18 - Form1040.line21)
```

Expanded view:

```text
Form1040.line22 = max(0, (Form1040.line16 + Form1040.line17) - (Form1040.line19 + Form1040.line20))
```

Implementation convention:

```text
Form1040.line22 = max(0, nz(Form1040.line18) - nz(Form1040.line21))
```

where `nz(x) = safeAmount(x)` treats null as zero.

Reference implementation (`TaxReturnComputeService.java:26963-26966`):

```java
// Line 22 = max(0, line18 - line21).
BigDecimal line18 = safeAmount(tac.getTotalTaxBeforeCredits());
BigDecimal line22 = roundMoney(line18.subtract(line21).max(BigDecimal.ZERO));
tac.setTaxAfterCredits(line22);
```

The `.max(BigDecimal.ZERO)` clamp is the structural enforcement of the IRS "If zero or less, enter -0-" rule.

---

## 3. What Line 22 Means

Line 22 is the **remaining tax after subtracting Form 1040 lines 19 through 21 credits from the line-18 tax subtotal**.

It is not:

- final total tax (that is `line 24`),
- a payment line (those start at `line 25`),
- a refundable-credit line (those land on `lines 27a-31`), or
- a line that can go negative.

The floor belongs here. If credits fully wipe out line 18, line 22 is zero.

---

## 4. Upstream Dependencies

### 4a. Tax subtotal input

```text
Form1040.line18 = line16 + line17
```

`line 18` is the tax-before-credits subtotal (regular tax + AMT).

### 4b. Credit subtotal input

```text
Form1040.line21 = line19 + line20
```

`line 21` is the total of the page-2 nonrefundable credits (child tax credit + Schedule 3 line 8 other nonrefundable credits).

So line 22 is simply:

```text
tax subtotal - nonrefundable credits subtotal
```

with a zero floor.

---

## 5. Storage Convention - ALWAYS-SET

Line 22 uses the **ALWAYS-SET** persistence convention. This is distinct from the neighboring lines:

| Line | Field | Storage convention when zero |
|---|---|---|
| 20 | `otherCreditsSchedule3` | null-when-zero (PDF cell blank) |
| 21 | `totalCredits` | null-when-zero (PDF cell blank) |
| **22** | **`taxAfterCredits`** | **ALWAYS-SET (PDF cell shows "0")** |
| 23 | `otherTaxes` | null-when-zero (only set by `finalizeSchedule2OtherTaxes` when grand total > 0) |
| 24 | `totalTax` | ZERO-when-zero (always non-null; PDF cell shows "0" or positive) |

The setter call is unconditional:

```java
tac.setTaxAfterCredits(line22);   // no `(line22 > 0) ? line22 : null` guard
```

Rationale: the IRS instruction explicitly says **"enter -0-"** when the subtraction would be zero or negative. A blank PDF cell would violate that instruction. ALWAYS-SET keeps the PDF cell populated with the literal "0".

## 6. Downstream Effect

Line 22 feeds line 24 through line 23:

```text
Form1040.line23 = Schedule2.line21
Form1040.line24 = Form1040.line22 + Form1040.line23
```

So the surrounding page-2 flow is:

```text
Form1040.line18 = line16 + line17
Form1040.line19 = Schedule8812.line14
Form1040.line20 = Schedule3.line8
Form1040.line21 = line19 + line20
Form1040.line22 = max(0, line18 - line21)
Form1040.line23 = Schedule2.line21
Form1040.line24 = line22 + line23
```

All four of lines 20, 21, 22, 24 are written by the single method `computeLine20ThroughLine24` at `TaxReturnComputeService.java:26942-26975`. Line 23 is written earlier by `finalizeSchedule2OtherTaxes` at `TaxReturnComputeService.java:22005-22070`.

### Form 2210 reads line 22 (not line 18)

Form 2210 Part I line 1 reads `tac.getTaxAfterCredits()` (line 22), **not** `getTotalTaxBeforeCredits()` (line 18):


This is the G3 fix (2026-04-19). Prior to the fix Form 2210 read line 18 directly, overstating the underpayment penalty base.

## 7. 2025 Guardrails

### 7a. The zero floor is mandatory on line 22

The 2025 form explicitly says:

```text
If zero or less, enter -0-
```

So line 22 must never be negative. Enforced by `.max(BigDecimal.ZERO)` at `TaxReturnComputeService.java:26965`.

### 7b. The floor belongs on line 22, not on line 21

Do not truncate `line 21` itself at zero. The arithmetic is:

```text
line21 = line19 + line20             (no floor; raw sum)
line22 = max(0, line18 - line21)     (floor at the subtraction)
```

If line 21 were truncated at zero independently, the model would lose the ability to track "credits exceed tax" as a soft state.

### 7c. Line 22 excludes later taxes

`line 22` does **not** include `line 23` other taxes. Those are added afterward on `line 24`.

### 7d. Line 22 excludes refundable credits and payments

Do not subtract or include:

- `line 27a` Earned Income Credit,
- `line 28` Additional Child Tax Credit,
- `line 29` American opportunity credit refundable portion,
- `line 30` refundable adoption credit,
- `line 31` Schedule 3 Part II amounts,
- withholding,
- estimated tax payments.

Those occur later in the payments section (`lines 25-33`).

### 7e. Margin-note boxes for line 12a/b/c/d are NO-OP for line 22

The 2025 form shows a margin note beside line 22 for taxpayers who checked a box on `line 12a`, `12b`, `12c`, or `12d` (extra standard deduction for age/blindness on the dependent worksheet path).

That note does **not** change the line-22 formula itself. Line 22 remains:

```text
max(0, line18 - line21)
```

Any special handling tied to those boxes belongs to the upstream deduction / tax instructions, not to different line-22 arithmetic. The code at `TaxReturnComputeService.java:26963-26966` contains no special-case branching for line 12 box flags.

## 9. Validation Rules

```text
Form1040.line22 >= 0
Form1040.line22 = max(0, nz(Form1040.line18) - nz(Form1040.line21))
Form1040.line22 <= Form1040.line18
Form1040.line22 is always non-null (ALWAYS-SET)
```

Practical checks:

- if `line21 = 0`, then `line22 = line18`
- if `line21 >= line18`, then `line22 = 0` (PDF still shows "0", not blank)
- if `line18 > line21`, then `line22 = line18 - line21`

## 12. Scope Note

`line 22` is a mechanical arithmetic line. The substantive tax-law complexity lives in the feeder lines:

- `line 18` for tax before credits,
- `line 19` for Schedule 8812 nonrefundable child-related credits,
