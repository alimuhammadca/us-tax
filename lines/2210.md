# Form 2210 — Underpayment of Estimated Tax by Individuals, Estates, and Trusts

> Form 2210 determines whether the taxpayer owes an **underpayment of estimated tax penalty**
> and, if so, computes the penalty. In most cases the IRS can figure the penalty itself; the
> 1040 instructions say not to file Form 2210 unless the form indicates it is required. When a
> penalty is computed, it flows to **Form 2210 line 19** and then to the **"Estimated tax
> penalty"** line of the return — **Form 1040 line 38** (`AmountOwed.estimatedTaxPenalty`).
>
> **Primary implementation blocker**: Part I requires the taxpayer's **prior-year total tax**
> (2024 Form 1040 line 22). The application has no prior-year return artifact. Until a
> `prior-year-tax-taxpayer` personal form is added, Form 2210 cannot produce a correct result
> for taxpayers who qualify for the prior-year safe harbor.

**IRS authority:**
- 2025 Form 2210 (`f2210.pdf` — 3 pages, 199 fields)
- 2025 Instructions for Form 2210 (`i2210.pdf`)
- 2025 Form 1040 Instructions (`i1040gi_2025.pdf`), Payments section
- IRS Publication 505 (2025), *Tax Withholding and Estimated Tax*
- J.K. Lasser's *Your Income Tax 2025*, Chapter on Estimated Tax Penalties

## 2. What Form 2210 Is For

Use Form 2210 to test whether the taxpayer paid enough tax during 2025 through:

- withholding, and
- estimated tax payments.

The form computes:

- the **required annual payment**,
- the **required installment** for each of the four payment periods,
- the **underpayment** by period, and
- the **penalty** if applicable.

If a penalty is due it goes on **Form 2210 line 19** and on the **"Estimated tax penalty"** line
of the return (Form 1040 line 38). The penalty is an interest-style charge — it can apply even
when the taxpayer is owed a refund on the return overall.

---

## 3. When a Penalty Does NOT Apply — No-Penalty Test

The taxpayer will **not** owe a penalty (and generally need not file Form 2210) if **either**
of the following is true:

```text
EXCEPTION 1 — Small balance:
  balance_due_after_withholding_and_refundable_credits < $1,000

EXCEPTION 2 — Safe-harbor payments:
  total_payments >= required_annual_payment

where:
  total_payments          = line25d_withholding + line26_estimatedTax
  required_annual_payment = min(0.90 × current_year_net_tax, prior_year_safe_harbor)

  prior_year_safe_harbor  =
    1.10 × 2024_total_tax   if 2024_AGI > $150,000 (or $75,000 if MFS)
    1.00 × 2024_total_tax   otherwise

  current_year_net_tax    = Form1040.line22  (total tax after nonrefundable credits,
                                               before refundable credits, per Form 2210 Part I)
```

Additional automatic waivers:

| Condition | Rule |
|---|---|
| Zero prior-year tax | 2024 total tax was zero OR no 2024 return was required (full 12-month U.S. resident year) |
| First-year filer | Never filed before; no prior-year tax exists |
| Farmers/fishermen | Use Form 2210-F instead; 66⅔% gross income from farming/fishing |

## 4. When to File Form 2210 — Part II Boxes A–E

The IRS prefers to compute the penalty itself when none of the special circumstances below apply.
The **Part II check boxes** on the form identify when filing is required or where only page 1
must be filed:

| Box | Condition | Filing requirement |
|---|---|---|
| **A** | Requesting waiver of the **entire** penalty | File **page 1 only**; do not compute penalty |
| **B** | Requesting waiver of **part** of the penalty | **Must file full Form 2210** and compute penalty |
| **C** | Using the **annualized income installment method** (Schedule AI) | **Must file full Form 2210** |
| **D** | Electing to treat withholding as paid on the **actual dates withheld** (vs. equal four-period allocation) | **Must file full Form 2210** |
| **E** | Filed or filing a joint return for **either** 2024 or 2025 but not both, and line 8 < line 5 | File **page 1 only** unless Box B, C, or D also applies |

**Filing decision tree:**

```text
if Box A checked:
    file page 1 only; IRS waives entire penalty

elif Box B or Box C or Box D checked:
    file Form 2210 (all pages) and compute penalty

elif Box E applies (joint/separate filing-status mismatch):
    file page 1 only unless Box B/C/D also applies

else:
    generally do NOT file; leave Form 1040 line 38 blank;
    IRS will compute and bill if a penalty applies
```

## 5. Part I — Required Annual Payment (Lines 1–9, Page 1)

### 5a. Line-by-Line Formula

```text
Line 1:  currentYearTax        = Form1040.line22
                                  (total tax after nonrefundable credits, before refundable credits)
                                  = TaxAndCredits.totalTaxBeforeCredits
                                    - nonrefundableCreditsSchedule3
                                  Note: excludes SE tax (out of scope)

Line 2:  otherTaxes            = supported Schedule 2 taxes included in the Form 2210 scope
                                  (excludes SE-related components)
                                  In this app: Form8959 AMT, Form5329 penalties, household
                                  employment tax — all already in TaxAndCredits.totalTax

Line 3:  combinedTax           = line1 + line2

Line 4:  refundableCredits     = credits used in the Form 2210 formula
                                  (typically EIC, ACTC, AOTC refundable — Payments lines 27a/28/29)

Line 5:  netTaxBasis           = line3 − line4
                                  (this is the "current-year net tax" used for the 90% test)

Line 6:  ninetyPctCurrentYear  = roundMoney(line5 × 0.90)

Line 7:  priorYearSafeHarbor   = if (priorYear_AGI > $150,000, or $75,000 if MFS_2025)
                                      roundMoney(priorYearTax × 1.10)
                                  else
                                      roundMoney(priorYearTax × 1.00)
                                  (null if priorYearTax not supplied)

Line 8:  requiredAnnualPayment = min(line6, line7)
                                  if line7 is null: requiredAnnualPayment = line6

Line 9:  underpaymentExposure  = max(0, line8 − totalPayments)
                                  where totalPayments = line25d + line26
```

- If `line5 ≤ 0` → no penalty; stop.
- If `totalTax − totalWithholding < $1,000` → no penalty; stop (Exception 1).
- If `underpaymentExposure ≤ 0` → no penalty; stop (safe harbor met).
- Otherwise: proceed to Part III (regular method) or Schedule AI.

### 5b. Form 1040 Line 22 vs. Line 24

Form 2210 Part I uses **Form 1040 line 22** (not line 24) as its starting point. In the 2025
Form 1040 structure:

| Line | Description | Relation to Form 2210 |
|---|---|---|
| Line 18 | `line16 + line17` (tax + AMT) | Tax before all credits |
| Line 21 | `line18 − line19 − line20` | After CTC and Schedule 3 nonrefundable credits |
| **Line 22** | `line21 + Schedule 2 line 10` | **Form 2210 Part I line 1 input** |
| Line 24 | Total tax (after everything, before withholding) | Different — includes refundable credit offsets |

Using line 22 means: regular tax + AMT + other Schedule 2 taxes, **after nonrefundable credits
but before refundable credits**. The Form 2210 instructions then separately subtract refundable
credits in Part I to derive the net tax basis for the 90% calculation.

For the application's purposes, `TaxAndCredits.totalTax` (line 24) already nets everything.
A precise implementation must back out refundable credits from line 24 to arrive at the correct
Part I line 5 basis. **Simplification allowed**: for most wage/investment filers without
large refundable credits, `line 24 ≈ line 22` and the difference is immaterial.

## 6. Computation Paths

### Path A — IRS Computes (Most Common)

When no Part II box applies, leave Form 1040 line 38 blank. The IRS calculates the penalty
from return data and sends a bill if one is owed. **No Form 2210 generated by the application.**

### Path B — Regular Method (Four Equal Installments)

Used when filing is required and the taxpayer is not using Schedule AI:

1. Compute required annual payment (Part I).
2. Divide into four equal required installments (25% each).
3. For each period, compare required installment to payments credited (withholding + estimated).
4. Compute per-period underpayments.
5. Apply penalty rate × days underpaid.
6. Sum all four period penalties → **Form 2210 line 19** → **Form 1040 line 38**.

```text
required_installment[i] = required_annual_payment / 4

underpayment[i] = max(0, required_installment[i]
                        − payments_credited_to_period[i]
                        − overpayment_carry_from_prior_period)

penalty[i] = underpayment[i] × (annual_rate / 365) × days_underpaid[i]

Form2210.line19 = sum(penalty[1..4])
```

### Path C — Annualized Income Installment Method (Schedule AI)

Used when income was **uneven** during the year (e.g., large capital gain in Q4, seasonal
business). Schedule AI reduces required installments for early periods:

```text
required_installment[i] = annualized_income_method_output[i]   (from Schedule AI line 27)
```

If Schedule AI is used for **any** installment period it must be used for **all** periods.
Annualized income inputs (period-by-period income and deductions) are not currently collected
by the application — this path is **out of scope** for the initial implementation.

### Path D — Withholding on Actual Dates Withheld

Normally, total withholding is allocated **equally** over the four installment periods
(25% per period), which allows large year-end withholding to retroactively cover early
underpayments. When Box D is elected, withholding is instead credited to the exact period in
which it was withheld. This requires per-paycheck or per-period withholding data not currently
collected — **out of scope** for initial implementation.

## 7. Part III — Regular Method Penalty Table (Page 2, Lines 10–18 × 4 Columns)

### 7a. Four 2025 Installment Due Dates

| Column | Installment | Due date | Period |
|---|---|---|---|
| A | 1st | **04/15/25** | Jan 1 – Apr 15, 2025 |
| B | 2nd | **06/15/25** | Apr 16 – Jun 15, 2025 |
| C | 3rd | **09/15/25** | Jun 16 – Sep 14, 2025 |
| D | 4th | **01/15/26** | Sep 15, 2025 – Jan 14, 2026 |

*If the return is filed and any balance paid by February 2, 2026, the 4th installment is not
required and that period's penalty accrual stops at the filing date.*

### 7b. Per-Period Computation (Lines 10–18)

```text
For each period i ∈ {Apr 15, Jun 15, Sep 15, Jan 15}:

  Line 10[i]: required_installment[i]  = required_annual_payment / 4

  Line 11[i]: estimated_payment[i]     = installment payment made for that period
                                         (installment1..4 from personal form)

  Line 12[i]: withholding[i]           = total_withholding / 4
                                         (equal allocation; or actual if Box D elected)

  Line 13[i]: overpayment_carry[i]     = excess from prior period
                                         = max(0, line14[i-1] − line10[i-1])

  Line 14[i]: total_available[i]       = line11[i] + line12[i] + line13[i]

  Line 15[i]: underpayment[i]          = max(0, line10[i] − line14[i])

  Line 16[i]: days_underpaid[i]        = days from installment due date until:
                                          ● the underpayment is fully covered by a later payment, OR
                                          ● April 15, 2026 (standard return due date), OR
                                          ● February 2, 2026 if return filed and balance paid by then

  Line 17[i]: penalty_rate[i]          = IRS underpayment rate for the period
                                         = (federal_short_term_rate + 3%) ÷ 365  (daily)

  Line 18[i]: penalty[i]               = roundMoney(underpayment[i] × penalty_rate[i]
                                                     × days_underpaid[i])
```

**Line 37**: `sum(penalty[A..D])` → `Form2210.line19` → `AmountOwed.estimatedTaxPenalty`

### 7c. Withholding Allocation — Default Equal Distribution

By default: `withholding[i] = totalWithholding / 4` for each period.

This means end-of-year W-2 withholding retroactively covers early-period underpayments under
the equal-allocation rule, which is favorable for taxpayers who have heavy Q4 withholding.

## 8. Schedule AI — Annualized Income Installment Method (Page 3, Lines 1–27)

### 8a. Purpose and Eligibility

Schedule AI is for taxpayers whose income was **substantially uneven** during the year (seasonal
business, large mid-year bonus, lump-sum capital gain). Instead of 25%-per-period installments,
each period's required payment is based on income actually earned through that date, annualized
to a full year — reducing required payments for early periods.

If Schedule AI is used for **any** period, it must be used for **all four** periods.

### 8b. Annualization Periods and Multipliers

| Period | Income measured through | Multiplier |
|---|---|---|
| 1st (Apr 15) | Mar 31, 2025 | 4.0 |
| 2nd (Jun 15) | May 31, 2025 | 2.4 |
| 3rd (Sep 15) | Aug 31, 2025 | 1.5 |
| 4th (Jan 15) | Dec 31, 2025 | 1.0 (no annualization) |

### 8c. Schedule AI Conceptual Formula

```text
For each period i:
  annualized_income[i] = income_through_date[i] × multiplier[i]
  annualized_tax[i]    = tax_on(annualized_income[i]) × 0.90
  required_installment[i] = annualized_tax[i] × 0.25
                             (adjusted for prior-period cumulative test)
  → flows to Part III Line 10[i]
```

### 8d. Implementation Status

**Out of scope** for initial implementation. Requires period-by-period income and deduction
breakdowns not currently collected by the application. Intake form design is deferred.

## 10. 2025 Underpayment Penalty Rate

The IRS underpayment rate = **federal short-term rate (adjusted quarterly) + 3 percentage
points**, compounded daily. The daily rate = `annual_rate ÷ 365`.

## 11. Waiver Rules

### 11a. Retirement or Disability Waiver

Applies when the taxpayer **retired** (age 62+) or **became disabled** in 2024 or 2025, and
the underpayment was due to **reasonable cause** (not willful neglect).

- Check Part II **Box B** (partial waiver) or **Box A** (full waiver).
- Attach written explanation: retirement/disability date, relation to underpayment.
- Backend intake: `retiredOrDisabledWaiver: boolean` on `prior-year-tax-taxpayer` form.

### 11b. Casualty, Disaster, or Unusual Circumstances Waiver

Applies when the underpayment resulted from a federally declared disaster, casualty, or
extraordinary event beyond the taxpayer's control.

- Federally declared disasters: IRS often grants automatic postponement — no Form 2210 needed.
- Other: Check Box B; attach documentation (police reports, insurance claims, written statement).
- Backend intake: `casualtyOrDisasterWaiver: boolean`.

### 11c. Full vs. Partial Waiver

- **Full waiver** (Box A): entire penalty waived; file **page 1 only**; no penalty computation.
- **Partial waiver** (Box B): part of penalty waived; must compute full penalty and submit the
  waiver amount; file **complete Form 2210**.
- Boxes A and B are mutually exclusive.

## 12. Special Cases

### 12a. First-Year Filers (No Prior-Year Tax)

If the taxpayer was not required to file a 2024 return **or** 2024 total tax was zero:
- Prior-year safe harbor = $0 (not applicable).
- `requiredAnnualPayment = ninetyPctCurrentYear` (only the 90% current-year test applies).
- If 2024 was a full 12-month U.S. resident year with zero tax → **no penalty** regardless of
  current-year underpayment.

### 12b. High-AGI Taxpayers — 110% Rule

When 2024 AGI > $150,000 ($75,000 if filing MFS for 2025):
```text
priorYearSafeHarbor = roundMoney(priorYearTax × 1.10)
```
The AGI threshold is based on **2024 AGI** (from the prior-year return), which must be manually
entered via the `prior-year-tax-taxpayer` form.

### 12c. Joint/Separate Filing-Status Change (Box E)

- **Joint 2024 → Separate 2025**: Each spouse must compute their proportional share of the 2024
  joint tax and use that as their individual prior-year safe harbor.
- **Separate 2024 → Joint 2025**: Both spouses' 2024 separate-return taxes are combined.

Box E must be checked on Form 2210 when this situation applies. The application should flag this
case and prompt the taxpayer to manually enter their share of the 2024 joint tax.

### 12d. Withholding Equal-Allocation Benefit

Because withholding is allocated equally over four periods by default, large Q4 withholding
(e.g., employer withholding $20,000 in December) retroactively reduces underpayments for all
four periods. This is a significant benefit for taxpayers who rely on year-end bonuses.
Only Box D election removes this advantage.

## 15. Minimal Implementation Summary

```text
# Step 1: Safe-harbor / no-penalty test
current_year_net_tax    = TaxAndCredits.totalTax  (approximation for line 22)
required_annual_payment = min(0.90 × current_year_net_tax, prior_year_safe_harbor)
                          (prior_year_safe_harbor = null if priorYearTax not supplied
                           → use only 90% test)

balance_due = max(0, totalTax − totalWithholding)

if balance_due < 1000:
    no penalty; stop
elif total_payments >= required_annual_payment:
    no penalty; stop
else:
    penalty may apply

# Step 2: Determine filing requirement (Part II boxes)
if Box A (full waiver elected):
    file page 1 only; estimatedTaxPenalty = null (IRS decides on waiver)

elif Box B or Box C or Box D:
    must file Form 2210 and compute penalty

elif Box E (joint/separate mismatch):
    file page 1 only unless also Box B/C/D

else:
    do not generate Form 2210; leave line 38 blank

# Step 3: Compute penalty (regular method — Paths B)
for each period i in [Apr15, Jun15, Sep15, Jan15]:
    required_installment[i]  = required_annual_payment / 4
    withholding[i]           = totalWithholding / 4
    estimated_payment[i]     = installment[i] from personal form
    total_available[i]       = withholding[i] + estimated_payment[i] + carry_from[i-1]
    underpayment[i]          = max(0, required_installment[i] − total_available[i])
    penalty[i]               = underpayment[i] × (rate / 365) × days_underpaid[i]

total_penalty = sum(penalty[Apr15..Jan15])

# Step 4: Wire output
Form2210.line19                  = total_penalty
AmountOwed.estimatedTaxPenalty   = total_penalty   → Form 1040 line 38
```

## 18. Validations

| Validation | Rule |
|---|---|
| No-penalty threshold | `totalTax − totalWithholding < $1,000` → `estimatedTaxPenalty = null` |
| Safe harbor met | `totalPayments ≥ requiredAnnualPayment` → `estimatedTaxPenalty = null` |
| Prior-year tax non-negative | `priorYearTax ≥ 0` |
| 110% AGI gate | `priorYearAgi > $150,000` ($75,000 if MFS) → use 1.10 multiplier |
| Period payments non-negative | All installment amounts ≥ 0 |
| Total penalty non-negative | `estimatedTaxPenalty ≥ 0` |
| Box A / Box B mutual exclusion | Cannot check both full and partial waiver |
| SE exclusion | Do not include SE tax in line 2 (SE out of scope); log if any SE-related amount would otherwise appear |

## 19. Out of Scope

| Item | Reason |
|---|---|
| Schedule AI (annualized income) | Requires period-by-period income/deduction breakdowns not collected |
| Box D — actual withholding dates | Requires per-paycheck W-2 withholding detail not collected |
| Form 2210-F (farmers/fishermen) | Separate form; 66⅔% farming income threshold not modeled |
| Self-employment tax in Part I line 2 | SE is out of scope for this application; explicitly excluded |
| Waiver documentation generation | Application does not generate or store written waiver attachments |
| Fiscal-year taxpayers | All installment dates assume calendar year |
| Community property MFS payment split | Proportional allocation of joint estimated payments deferred |
| Precise daily compounding | Simplification: flat `annualRate ÷ 365` per day (not compound) |
| Prior-year auto-import | 2024 return artifact not available; `priorYearTotalTax` is manual entry |
