# Form 1040 (2025) - Line 23: Other Taxes From Schedule 2, Line 21

> Source of truth: 2025 Form 1040, 2025 Schedule 2 (Form 1040), and the 2025 Instructions for Form 1040.

---

## 1. Line Identity

**Form 1040 line 23** is:

```text
Other taxes, including self-employment tax, from Schedule 2, line 21
```

Exact mapping:

```text
Form1040.line23 = Schedule2.line21
```

This line brings the return's **other taxes** onto Form 1040 page 2 after `line 22` and before `line 24` total tax. It is implemented as a pure pass-through read of the finalized `TaxAndCredits.otherTaxes` field; the substantive aggregation lives upstream in `finalizeSchedule2OtherTaxes`.

---

## 2. Core 2025 Formula

```text
Form1040.line23 = Schedule2.line21
```

For 2025, Schedule 2 line 21 is:

```text
Schedule2.line21 = line4 + line7 + line8 + line9 + line11 + line12
                 + line13 + line14 + line15 + line16 + line18 + line19
```

This follows the exact 2025 form instruction:

```text
Add lines 4, 7 through 16, 18, and 19.
```

**Line 10 is reserved for future use in 2025** (treated as zero structurally — it is not even in the `sumAmounts(...)` argument list). **Line 20 (Section 965 net tax liability installment) is excluded** per the printed 2025 rule.

Reference implementation — `finalizeSchedule2OtherTaxes` at `TaxReturnComputeService.java:22005-22070`:

```java
BigDecimal grandTotal = sumAmounts(
    ot.getSelfEmploymentTax(),                                  // line 4   (out of scope today)
    ot.getTotalAdditionalSocialSecurityMedicareTax(),           // line 7   (lines 5 + 6)
    ot.getAdditionalTaxOnIras(),                                // line 8
    ot.getHouseholdEmploymentTaxes(),                           // line 9
    // line 10 reserved for future use → always 0
    ot.getAdditionalMedicareTax(),                              // line 11
    ot.getNetInvestmentIncomeTax(),                             // line 12
    ot.getUncollectedSocialSecurityMedicareRrtaTax(),           // line 13
    ot.getInterestOnInstallmentSalesResidentialLots(),          // line 14
    ot.getInterestOnDeferredTaxInstallmentSalesOver150k(),      // line 15
    ot.getRecaptureLowIncomeHousingCredit(),                    // line 16
    line18,                                                     // line 18 (just computed)
    ot.getRecaptureNetEpeForm4255Line1dColL()                   // line 19
);

if (grandTotal != null && grandTotal.compareTo(BigDecimal.ZERO) > 0) {
    ot.setTotalOtherTaxes(roundMoney(grandTotal));
    tac.setOtherTaxes(roundMoney(grandTotal));
}
```

Note: the `tac.setOtherTaxes(...)` call is **null-when-zero** — only invoked when `grandTotal > 0`. This is the convention that `computeLine20ThroughLine24` then reads via `safeAmount(tac.getOtherTaxes())` at line 26969 (null → zero) before forming line 24.

## 3. 2025 Schedule 2 Structure Relevant To Line 23

### Part I - Tax -> Form 1040 line 17, not line 23

These Schedule 2 items do **not** feed Form 1040 line 23:

- `1a` through `1z` additions to tax (excess advance PTC, etc.)
- `2` alternative minimum tax (Form 6251 line 11)
- `3` total Part I

Those feed:

```text
Form1040.line17 = Schedule2.line3
```

### Part II - Other Taxes -> Form 1040 line 23

The 2025 line-23 feeder is only **Schedule 2 line 21** from Part II.

## 4. Important 2025 Implementation Facts

### 4a. Form 8960 IS implemented (line 12 — NIIT)

The earlier project note that Form 8960 was unimplemented is stale. `Schedule2OtherTaxes.netInvestmentIncomeTax` is wired by the Form 8960 build path and flows through `finalizeSchedule2OtherTaxes` line 12 as `ot.getNetInvestmentIncomeTax()` at `TaxReturnComputeService.java:22050`.

### 4b. Form 8959 Parts I + III + IV + V drive line 11 (G3 fix)

Form 8959 line 18 (`getLine18TotalAdditionalMedicareTax`) — the sum of Part I (Medicare on wages) + Part II (SE; OOS) + Part III (RRTA) — is written to `schedule2.otherTaxes.additionalMedicareTax` at `TaxReturnComputeService.java:991`. This is the **G3 fix (2026-04-19)** that supersedes the earlier Part-I-only estimate.

Form 8959 Part V line 24 (Additional Medicare Tax **withheld**) is separately routed to Form 1040 line 25c (withholdingOther), not to Schedule 2 line 11.

### 4c. `finalizeSchedule2OtherTaxes` is the single aggregator (G5 fix)

Individual writer methods (`applyAdditionalSocialSecurityMedicareTaxes`, `applyForm5329TaxToSchedule2`, the Form 8960 builder, the Form 8959 G3 setter, etc.) set their individual fields on `Schedule2OtherTaxes` but **do not** maintain a running grand total. Only `finalizeSchedule2OtherTaxes` writes `tac.otherTaxes`. The G5 fix removed double-accumulation from individual writers.

### 4d. Form 8978 negative line 14 does NOT go on line 23

Form 8978 (BBA partnership pushout) routes:

- **positive** line 14 → Form 1040 line 16 box 3 ("Form 8978" write-in), affecting tax-before-credits
- **negative** line 14 → Schedule 3 line 6l (reducing nonrefundable credits)

Neither path touches Schedule 2 line 21 / Form 1040 line 23.

### 4e. Form 2210 reads `tac.getOtherTaxes()` (G7 fix)

```java
// TaxReturnComputeService.java:27570-27573
BigDecimal l2 = form1040.getTaxAndCredits() != null
    ? safeAmount(form1040.getTaxAndCredits().getOtherTaxes())
    : BigDecimal.ZERO;
f.setOtherTaxes(l2);
```

This is the **G7 fix**. Prior to the fix Form 2210 line 2 was hardcoded to zero, understating the penalty base.

## 5. Critical 2025 Guardrails

### 5a. Line 23 is Schedule 2 line 21 only

Do not add Part I tax items here. Line 23 is not a generic Schedule 2 total.

```text
Form1040.line23 = Schedule2.line21
```

### 5b. Schedule 2 line 21 excludes line 20 on the 2025 form

This is the main 2025 implementation trap.

The 2025 Schedule 2 form explicitly says:

```text
Add lines 4, 7 through 16, 18, and 19.
```

So the current 2025 Schedule 2 total **does not include line 20**. Structurally enforced by the absence of `getSection965NetTaxLiabilityInstallment` (or equivalent) from the `sumAmounts(...)` argument list in `finalizeSchedule2OtherTaxes`.

### 5c. Lines 5 and 6 do not flow separately

Because line 21 uses `line 7`, not lines `5` and `6` individually:

```text
Schedule2.line7 = line5 + line6
Schedule2.line21 includes line7   (NOT line5 and NOT line6 directly)
```

Do not double count `5` and `6` by adding them again after line 7 is computed.

### 5d. Line 17 detail lines only flow through line 18

`17a` through `17z` are detail lines. They reach line 23 only through:

```text
Schedule2.line18 = sum(line17a through line17z)
Schedule2.line21 includes line18   (NOT 17a-17z directly)
```

Do not add `17a-17z` directly to `line21` in addition to `line18`.

### 5e. Line 10 is reserved

`Schedule 2 line 10` is reserved for future use in 2025. The implementation omits it entirely from the `sumAmounts(...)` list rather than passing a zero — same numeric effect, but the omission documents the reservation.

## 6. What Line 23 Means On Form 1040

Line 23 is the **other-taxes subtotal** added after the line-22 tax-after-credits amount.

It is not:

- a payment,
- a withholding line,
- a refundable-credit line, or
- the final total tax.

The page-2 flow is:

```text
Form1040.line22 = max(0, Form1040.line18 - Form1040.line21)
Form1040.line23 = Schedule2.line21
Form1040.line24 = Form1040.line22 + Form1040.line23
```

`computeLine20ThroughLine24` at `TaxReturnComputeService.java:26942-26975` reads `tac.getOtherTaxes()` via `safeAmount(...)` so a null line-23 is silently treated as zero when forming line 24.

## 7. Inputs and Outputs

### Inputs

```text
Depends on:
  Schedule2.line21  (= TaxAndCredits.otherTaxes, written by finalizeSchedule2OtherTaxes)
```

### Output

```text
Produces:
  Form1040.line23   →   PDF field line23_other_taxes (consumed by line 24)
```

Note: line 23 is **not** stored as a separate field. The same `TaxAndCredits.otherTaxes` value is read both for PDF export (line 23 cell) and for the line 24 sum.

### Attachments

Line 23 itself creates no attachment, but a nonzero line 23 generally implies **Schedule 2** is attached, plus the source forms for the Schedule 2 items:

- Form 4137 (line 5)
- Form 8919 (line 6)
- Form 5329 (line 8)
- Schedule H (line 9)
- Form 8959 (line 11)
- Form 8960 (line 12)
- Form 8611 (line 16)
- Form 4255 (line 19)
- source forms behind `17a-17z`

## 8. Validation Rules

```text
Form1040.line23 >= 0
Form1040.line23 = Schedule2.line21
Schedule2.line7  = Schedule2.line5 + Schedule2.line6
Schedule2.line18 = sum(Schedule2.line17a through Schedule2.line17z)
Schedule2.line21 = line4 + line7 + line8 + line9 + line11 + line12 + line13 + line14 + line15 + line16 + line18 + line19
```

Practical validation notes:

- if all included Schedule 2 Part II lines are zero, then `tac.otherTaxes` is null (`finalizeSchedule2OtherTaxes` skips the setter when `grandTotal <= 0`); line 23 PDF cell is blank.
- if `line5` or `line6` is present, verify they were first rolled into `line7`.
- if any `17*` detail line is present, verify `line18` is populated before `line21`.
- do not include `line20` in `line21` unless future IRS guidance changes the printed 2025 form rule.
- do not include `line 11` Additional Medicare Tax `withheld` (that is Form 8959 Part V line 24, which goes to Form 1040 line 25c).

## 9. Implementation Notes For This Project

Recommended order (mostly enforced by `prepare()` at `TaxReturnComputeService.java:~2055-2075`):

1. compute all Schedule 2 Part I items -> `Schedule2.line3` -> `Form1040.line17`
2. compute all Schedule 2 Part II sub-items via their individual writers (each sets one field on `Schedule2OtherTaxes`; none accumulates the grand total — G5 fix)
3. compute `Schedule2.line7` via `applyAdditionalSocialSecurityMedicareTaxes`
4. compute `Schedule2.line18` inside `finalizeSchedule2OtherTaxes` from the 17a-17z fields
5. compute `Schedule2.line21` inside `finalizeSchedule2OtherTaxes`; write to `tac.otherTaxes` only when > 0
6. `computeLine20ThroughLine24` reads `safeAmount(tac.getOtherTaxes())` as line 23 for line 24 sum
7. `computeForm2210` reads `safeAmount(tac.getOtherTaxes())` as Part I line 2 (G7 fix)
