# Form 1040 (2025) - Line 24: Total Tax

> Source of truth: 2025 Form 1040, 2025 Schedule 2 (Form 1040), and the 2025 Instructions for Form 1040.

---

## 1. Line Identity

**Form 1040 line 24** is:

```text
Add lines 22 and 23. This is your total tax
```

Exact mapping:

```text
Form1040.line24 = Form1040.line22 + Form1040.line23
```

For 2025, line 24 is the return's **total tax before payments and refundable credits are applied**. It is the most consequential single value on the return — every downstream payment, refund, amount-owed, and underpayment-penalty calculation reads from it.

---

## 2. Core 2025 Formula

```text
Form1040.line22 = max(0, Form1040.line18 - Form1040.line21)
Form1040.line23 = Schedule2.line21
Form1040.line24 = Form1040.line22 + Form1040.line23
```

Expanded view:

```text
Form1040.line24 = max(0, Form1040.line18 - Form1040.line21) + Schedule2.line21
```

Implementation convention:

```text
Form1040.line24 = nz(Form1040.line22) + nz(Form1040.line23)
```

Reference implementation (`TaxReturnComputeService.java:26968-26971`):

```java
// Line 24 = line22 + line23 (other taxes from Schedule 2).
BigDecimal line23 = safeAmount(tac.getOtherTaxes());
BigDecimal line24 = roundMoney(line22.add(line23));
tac.setTotalTax(line24.compareTo(BigDecimal.ZERO) > 0 ? line24 : BigDecimal.ZERO);
```

No floor is applied on line 24 beyond the floor already applied on line 22 (and the fact that line 23 is non-negative by IRS construction).

---

## 3. Storage Convention - ZERO-when-zero

Line 24 uses the **ZERO-when-zero** persistence convention (distinct from both ALWAYS-SET on line 22 and null-when-zero on lines 20/21/23):

```java
tac.setTotalTax(line24.compareTo(BigDecimal.ZERO) > 0 ? line24 : BigDecimal.ZERO);
```

So `totalTax` is **never null** — it is either a positive value or `BigDecimal.ZERO`. The PDF cell shows "0" or the positive amount, never blank. Three null-handling conventions coexist in the single method `computeLine20ThroughLine24`:

| Line | Field | Setter |
|---|---|---|
| 20 | `otherCreditsSchedule3` | `tac.setOtherCreditsSchedule3(line20);` where `line20` is null when totalCredits == 0 |
| 21 | `totalCredits` | `tac.setTotalCredits(line21.compareTo(ZERO) > 0 ? line21 : null);` |
| 22 | `taxAfterCredits` | `tac.setTaxAfterCredits(line22);` (unconditional — ALWAYS-SET) |
| 24 | `totalTax` | `tac.setTotalTax(line24.compareTo(ZERO) > 0 ? line24 : BigDecimal.ZERO);` (ZERO-when-zero) |

Rationale: total tax is the return's marquee value. A blank cell would be ambiguous (was zero computed? was the line skipped?). The literal "0" makes the answer explicit.

---

## 4. What Line 24 Means

Line 24 is the return's **total tax** on Form 1040 page 2.

It includes:

- tax remaining after nonrefundable credits (`line 22`), and
- other taxes from Schedule 2 Part II (`line 23`).

It does **not** include:

- withholding,
- estimated tax payments,
- refundable credits,
- Schedule 3 Part II payments / refundable credits,
- Earned Income Credit,
- Additional Child Tax Credit,
- American opportunity credit refundable portion,
- refundable adoption credit (line 30, new for 2025).

Those are handled later in the payments section (`lines 25-33`).

---

## 5. Upstream Dependencies

### 5a. Line 22 input

```text
Form1040.line22 = max(0, Form1040.line18 - Form1040.line21)
```

This is the tax after page-2 nonrefundable credits. Always non-null (ALWAYS-SET).

### 5b. Line 23 input

```text
Form1040.line23 = Schedule2.line21 (= TaxAndCredits.otherTaxes)
```

This is the total of 2025 Schedule 2 Part II other taxes. Null-when-zero. `computeLine20ThroughLine24` reads it via `safeAmount(...)` so a null line-23 is treated as zero when forming line 24.

So the line-24 computation is a simple subtotal of:

```text
remaining tax after credits + other taxes
```

---

## 6. Downstream Effect

Line 24 starts the payments and refund / balance-due section.

Immediately after line 24, the 2025 Form 1040 moves into:

- `line 25a-25d` federal income tax withheld,
- `line 26` estimated tax payments,
- `line 27a` Earned Income Credit,
- `line 28` Additional Child Tax Credit,
- `line 29` refundable American Opportunity Credit,
- `line 30` refundable adoption credit (new for 2025),
- `line 31` Schedule 3 Part II amounts,
- `line 32` total other payments and refundable credits = line 27a + 28 + 29 + 30 + 31,
- `line 33` total payments = line 25d + line 26 + line 32,
- `line 34` overpayment if line 33 > line 24,
- `line 37` amount owed if line 24 > line 33.

So the surrounding page-2 flow is:

```text
Form1040.line22 = max(0, line18 - line21)
Form1040.line23 = Schedule2.line21
Form1040.line24 = line22 + line23
Form1040.line25d = line25a + line25b + line25c
Form1040.line33 = line25d + line26 + line32
Form1040.line34 = max(0, line33 - line24)    (overpayment / refund pivot)
Form1040.line37 = max(0, line24 - line33)    (amount-owed pivot)
```

### Form 2210 reads line 22 and line 23 separately (not line 24)

Form 2210 Part I reconstructs the current-year tax from its components:

```text
Form2210.line1 = Form1040.line22    (tax after credits — G3 fix)
Form2210.line2 = Form1040.line23    (other taxes, excluding SE tax — G7 fix)
Form2210.line3 = Form2210.line1 + Form2210.line2
```

Form 2210 does **not** read `Form1040.line24` directly — it reads the same two source values:

```java
// TaxReturnComputeService.java:27562-27576
BigDecimal l1 = safeAmount(form1040.getTaxAndCredits().getTaxAfterCredits());   // line 22
f.setCurrentYearTax(l1);
BigDecimal l2 = safeAmount(form1040.getTaxAndCredits().getOtherTaxes());        // line 23
f.setOtherTaxes(l2);
BigDecimal l3 = l1.add(l2);                                                     // structurally = line 24
f.setCombinedTax(l3);
```

The G3 fix (2026-04-19) ensures `Form 2210` line 1 reads `taxAfterCredits` (line 22), not `totalTaxBeforeCredits` (line 18). The G7 fix removed a hardcoded zero on line 2.

## 7. 2025 Guardrails

### 7a. Line 24 is a pure addition line

The form instruction is exact:

```text
Add lines 22 and 23.
```

Do not introduce any extra adjustments here.

### 7b. Line 24 is not reduced by payments

Payments and refundable credits begin **after** line 24. So line 24 must be computed before any of these are applied:

- withholding on lines 25a-25d,
- estimated tax on line 26,
- earned income credit on line 27a,
- ACTC on line 28,
- refundable AOTC on line 29,
- refundable adoption credit on line 30,
- Schedule 3 Part II on line 31,
- total payments on lines 32 and 33.

### 7c. Line 24 can equal line 22

If `Form1040.line23 = 0` (or null, treated as zero), then:

```text
Form1040.line24 = Form1040.line22
```

This is the common case for returns with no Schedule 2 Part II activity.

### 7d. Line 24 can be positive even when line 22 is zero

Because line 23 contains Schedule 2 other taxes:

```text
if Form1040.line22 = 0 and Form1040.line23 > 0,
then Form1040.line24 = Form1040.line23
```

This is a common implementation edge case. Credits can eliminate regular tax while leaving Additional Medicare Tax, NIIT, household employment tax, or other Schedule 2 taxes still due.

### 7e. Line 24 ≥ line 22 and line 24 ≥ line 23

Both addends are non-negative by construction (line 22 floored at zero by `.max(BigDecimal.ZERO)`; line 23 non-negative by IRS Schedule 2 form rules), so the sum dominates each addend.

## 9. Validation Rules

```text
Form1040.line24 >= 0
Form1040.line24 = nz(Form1040.line22) + nz(Form1040.line23)
Form1040.line24 >= Form1040.line22
Form1040.line24 >= Form1040.line23
Form1040.line24 is always non-null (ZERO-when-zero)
```

Practical checks:

- if both inputs are zero, `line24 = 0` (PDF cell shows "0")
- if `line23 = 0` (or null), `line24 = line22`
- if `line22 = 0`, `line24 = line23`
- if both are positive, `line24` must equal their exact whole-dollar sum after `roundMoney`

## 10. Implementation Notes For This Project

The line-24 computation is small — three lines inside `computeLine20ThroughLine24` at `TaxReturnComputeService.java:26968-26971`:

```java
BigDecimal line23 = safeAmount(tac.getOtherTaxes());
BigDecimal line24 = roundMoney(line22.add(line23));
tac.setTotalTax(line24.compareTo(BigDecimal.ZERO) > 0 ? line24 : BigDecimal.ZERO);
```

Call order (handled by `prepare()` at `TaxReturnComputeService.java:~2055-2075`):

1. compute `line 22` (inside `computeLine20ThroughLine24`)
2. `line 23` already set by `finalizeSchedule2OtherTaxes` (ran earlier at line 2058)
3. compute `line 24` (inside `computeLine20ThroughLine24`)
4. compute payments beginning at `line 25a` via `computeLine31ThroughLine38` (despite the name, this method handles lines 25-38)

PDF export reads `form.taxAndCredits?.totalTax` and renders the value in field `line24_total_tax` on Form 1040 page 2.

## 12. Scope Note

`line 24` is a simple arithmetic total. The substantive tax-law complexity lives upstream in:

- `line 22` for tax after nonrefundable credits, and
- `line 23` for Schedule 2 other taxes (Form 8959 NIIT, Form 5329 IRA penalty, Form 8960 NIIT, household employment tax, etc.).

This spec stays focused on the 2025 line-24 total-tax mapping and the ZERO-when-zero persistence convention.
