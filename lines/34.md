# Form 1040 (2025) — Line 34: Amount Overpaid

> Source of truth: 2025 Form 1040, 2025 Form 1040 Instructions, J.K. Lasser 2025.
> Verified 2026-06-14.

---

## 1. Line Identity

**Form 1040 line 34** label (2025):

```text
If line 33 is more than line 24, subtract line 24 from line 33. This is the amount you OVERPAID.
```

Exact formula:

```text
line34 = line33 − line24    (computed only when line33 > line24)
```

Line 34 is the **overpayment pivot** — the dollar amount by which total payments (line 33) exceed total tax (line 24). It triggers the refund cascade (lines 35a–35d, line 36). It is mutually exclusive with line 37 (amount owed): exactly one of the two lines is populated per return (neither when line 33 == line 24).

---

## 2. Top-Level Formula

```text
line34 = roundMoney(line33 − totalTax)    when line33 > totalTax (strict inequality)
line34 = null                              otherwise
```

Where:

```text
line33    = Payments.totalPayments        (set at line 27313)
totalTax  = TaxAndCredits.totalTax        (Form 1040 line 24)
```

The strict inequality `line33 > totalTax` is what enforces the mutual exclusion with line 37. When `line33 == totalTax` (balanced return), neither line 34 nor line 37 is set; the `Refund` and `AmountOwed` output objects are both left null.

---

## 3. Compute Order and Wiring

`TaxReturnComputeService.computeLine31ThroughLine38()` at lines 27328-27349:

```text
1. Compute line 33 (line 27293-27313).
2. Read line 36 raw user election (line 27315-27326, Stage 1).
3. Load totalTax from TaxAndCredits at line 27329-27330.
4. Gate at line 27332: if (line33 > totalTax) → refund branch.
5. Inside the refund branch (line 27333-27349):
   a. overpaid = roundMoney(line33 − totalTax)     [pure line 34]
   b. Lazy-init Refund object if absent (line 27335-27339).
   c. refund.setOverpaid(overpaid)                  [line 27340 — stores line 34]
   d. line36Capped = line36.min(overpaid)           [line 27343 — cap line 36 at line 34]
   e. if line36Capped > 0: refund.setAmountAppliedToNextYear(line36Capped)
   f. line35a = overpaid − line36Capped              [line 27349 — refund amount]
   g. refund.setRefundAmount(line35a > 0 ? line35a : null)
   h. lines 35b/c/d direct-deposit cascade (if !hasSplitRefund && wantsDirectDeposit)
6. else-if branch at line 27440: line37 (amount owed) — mutually exclusive.
```

Pure line 34 (overpayment) is stored at `Refund.overpaid`. The refund branch is a single if-block; lines 35a/35b–d/36 all cascade off the same gate.

## 4. Output Target

| Field | Type | Where wired | When populated |
|---|---|---|---|
| `form1040.refund.overpaid` | BigDecimal | `computeLine31ThroughLine38` line 27340 | When `line33 > totalTax` |
| `form1040.refund` | Refund object | Lazy-init at line 27335-27339 | Same condition (only inside refund branch) |
| PDF field `line34_overpaid` | — | Frontend export | From `refund?.overpaid` |

The `Refund` object is **lazy-initialized** — it only exists on `Form1040` when the taxpayer overpaid. On a balanced or owed return, `form1040.getRefund() == null`.

---

## 5. Null Behavior — GATED-NOT-SET (Convention 1)

Line 34 uses the **GATED-NOT-SET** Convention 1 mechanism: when `line33 ≤ totalTax`, the setter is never called and `Refund.overpaid` stays at its default `null`. There is no explicit `setOverpaid(null)` call; the entire if-block (line 27332-27439) is skipped on the else path.

This is structurally cleaner than ternary-at-setter (used by line 32/33/35a) because the lazy-init Refund object is also skipped — no empty Refund container is created on owed returns.

---

## 6. Mutual Exclusivity with Line 37

```text
if (line33 > totalTax) {
    // Refund branch — line 34 set, line 35a/35b-d/36 cascade
} else if (totalTax > line33) {
    // Owed branch — line 37 set
}
// Implicit balanced branch: line33 == totalTax → NEITHER set
```

Both branches use strict inequalities. When `line33 == totalTax` exactly (balanced return), neither line 34 nor line 37 is set, and both `Refund` and `AmountOwed` remain null. This satisfies the IRS rule that lines 34 and 37 cannot both be populated on the same return.

The mutual exclusivity is **structurally** enforced by the if/else-if branch; no validation flag is needed.

## 7. Refund Cascade (Same If-Block)

When line 34 is set, four downstream fields are computed inside the same if-block:

| Sub-line | Field | Formula | Notes |
|---|---|---|---|
| 35a | `refund.refundAmount` | `overpaid − line36Capped` | Refund check/DD amount |
| 35b | `refund.routingNumber` | String passthrough | Only if wantsDirectDeposit && !hasSplitRefund |
| 35c | `refund.accountType` | "Checking" or "Savings" | Same gating |
| 35d | `refund.accountNumber` | String passthrough (17-char truncate) | Same gating + IRS Doc 1346 validation |
| 36 | `refund.amountAppliedToNextYear` | `min(line36 raw, overpaid)` | Capped at line 34 |

See `lines/35.md` for 35a–35d details and `lines/36.md` for the apply-to-2026 election.

## 8. MFS Inheritance via M2 (Transitive)

Line 34 has no per-spouse code or MFS-specific logic. Inputs are:

- `line33` — M2 transitive via 25d + 26 + 32 sub-line MFS protection
- `totalTax` (line 24) — per-return; each MFS spouse files separately with their own totalTax

Line 34 is a pure subtraction of two MFS-clean upstream values. No in-helper MFS check is needed.

---

## 9. Reference Data

Line 34 wiring uses **zero** tax-year-specific constants. It is a pure conditional subtraction. Reference data relevant to the refund branch (NOT used at line 34 wiring) includes the `$1` IRS minimum-refund rule — refunds under $1 are not auto-issued; the taxpayer must request them in writing. This rule is NOT enforced at line 34 (it would apply at line 35a).

## 10. Validations and Invariants

| Invariant | Rationale / How Enforced |
|---|---|
| `line34 > 0` when set | Gate requires strict inequality `line33 > totalTax` |
| `line34` null when `line33 ≤ totalTax` | STRUCTURALLY enforced via GATED-NOT-SET |
| Lines 34 and 37 mutually exclusive | STRUCTURALLY enforced via if/else-if at line 27332/27440 |
| Line 34 not set when `line33 == totalTax` | STRUCTURALLY enforced — strict inequalities on both branches |
| 35a, 35b-d, 36 cascade off line 34 | All four computed inside the same if-block at line 27340-27437 |
| `Refund` lazy-initialized | `new Refund()` at line 27337 if not already present |

Line 34 emits NO blocking flags. The gate prevents a negative result by construction.

---

## 11. Out of Scope

- Refund offsets (past-due child support, federal debt) — IRS Treasury Offset Program handled outside the return
- Refund interest computations (IRC § 6611)
- Frozen / withheld refunds (identity-theft holds)

These are downstream IRS processes; they do not alter the line 34 arithmetic on the filed return.

---

## 12. Summary Rule

For tax year 2025:

```text
line34 = roundMoney(line33 − totalTax)    when line33 > totalTax
line34 = null                              otherwise
```

Stored at `Refund.overpaid`. Triggers the refund cascade (35a/35b-d/36). Mutually exclusive with line 37 (amount owed).
