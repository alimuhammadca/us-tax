# Form 1040 (2025) — Line 35: Refund (35a/35b/35c/35d)

> Source of truth: 2025 Form 1040, 2025 Form 1040 Instructions, IRS Document 1346.
> Verified 2026-06-14.

---

## 1. Line Identity

**Form 1040 line 35** covers four sub-lines on page 2 (2025):

```text
35a — Amount of line 34 you want refunded to you.
      If Form 8888 is attached, check here  □
35b — Routing number
35c — Type: □ Checking   □ Savings
35d — Account number
```

Sub-line summary:

| Sub-line | Purpose | Type |
|---|---|---|
| 35a | Refund amount (line 34 − line 36) | BigDecimal |
| 35b | Routing number (9-digit ABA, Mod-10) | String |
| 35c | Account type — "Checking" or "Savings" | String → two PDF checkboxes |
| 35d | Account number (≤ 17 chars, IRS Doc 1346 class) | String |

---

## 2. Top-Level Formulae

### 2a. Line 35a — Refund Amount

```text
line35a = roundMoney(overpaid − line36Capped)        // when overpayment branch active
        stored null when ≤ 0 (ternary-at-setter)
```

Where:

```text
overpaid       = line 34 (Refund.overpaid)
line36Capped   = min(line 36 raw, overpaid)
```

### 2b. Lines 35b/35c/35d — Direct Deposit

Pure string passthrough from the `35-direct-deposit` personal form, gated by:

1. Outer overpayment if-block (`line33 > totalTax`)
2. `!hasSplitRefund` — Form 8888 NOT elected
3. `directDepositData != null` — personal form present
4. `wantsDirectDeposit == true`
5. Per-field non-blank check (each of routing / type / account number gated individually)

When Form 8888 is attached (taxpayer wants to split the refund among up to three accounts), the IRS instructions require leaving 35b/35c/35d **blank** on Form 1040; Form 8888 carries the routing/account triples instead.

## 3. Compute Order and Wiring

`TaxReturnComputeService.computeLine31ThroughLine38()` at lines 27348-27439 (inside the `if (line33 > totalTax)` refund branch):

```text
Line 35a (line 27348-27350):
   line35a = roundMoney(overpaid.subtract(line36Capped))
   refund.setRefundAmount(line35a > 0 ? line35a : null)

Lines 35b/c/d (line 27352-27438):
   hasSplitRefund = refundAllocationData && wantsRefundAllocation == TRUE
   if (!hasSplitRefund && directDepositData != null) {
      wantsDd = getBoolean(directDepositData, "wantsDirectDeposit")
      if (wantsDd == TRUE) {
          refund.setDirectDeposit(true)
          routing = trim(directDepositData.routingNumber)
          if (routing != null && !routing.isEmpty())
              refund.setRoutingNumber(routing)
          acctType = trim(directDepositData.accountType)
          if (acctType != null && !acctType.isEmpty())
              refund.setAccountType(acctType)
          acctNum = trim(directDepositData.accountNumber)
          if (acctNum != null && !acctNum.isEmpty()) {
              // 35d.md Gap D — 17-char truncate (IRS Doc 1346)
              stored = acctNum.length > 17 ? acctNum.substring(0,17) : acctNum
              if (acctNum.length > 17)
                  emit LINE_35D_ACCOUNT_NUMBER_TOO_LONG advisory
              // 35d.md Gap E — class [0-9A-Z-] check
              if (!stored.toUpperCase().matches("^[0-9A-Z\\-]+$"))
                  emit LINE_35D_ACCOUNT_NUMBER_INVALID_CHARACTERS advisory
              refund.setAccountNumber(stored)
          }
          // 35d.md Gap F — all-three-or-none triple check
          if (any of routing/type/number missing while DD elected)
              emit LINE_35BCD_INCOMPLETE_TRIPLE advisory
      }
   }
```

## 5. Two Gating Layers

Lines 35b/35c/35d are gated by two independent layers:

### Layer 1 — Form 8888 split refund (trumps DD)

```text
hasSplitRefund = refundAllocationData != null
              && wantsRefundAllocation == TRUE
```

Source: the `refund-allocation-taxpayer` personal form's `wantsRefundAllocation` boolean. When TRUE, Form 8888 is attached and 35b/c/d on Form 1040 are blank — Form 8888 carries the routing/account/amount triples (up to 3 accounts).

### Layer 2 — Direct deposit election

```text
wantsDirectDeposit = directDepositData.wantsDirectDeposit == TRUE
```

Source: the `35-direct-deposit` personal form. When FALSE (or form absent), the taxpayer receives a paper check — 35b/c/d remain blank.

Both layers must permit before any 35b/c/d field is written: Form 8888 has priority over direct-deposit (Form 8888 trumps the Form 1040 DD fields).

## 6. Line 35d Validation — IRS Document 1346

Line 35d (account number) has three closure-grade validations from IRS Doc 1346 (the e-file business rules manual). All three emit **non-blocking advisory** flags so the taxpayer can correct before signing:

### 6a. 17-character truncate (35d.md Gap D — LINE_35D_ACCOUNT_NUMBER_TOO_LONG)

Browser `maxlength=17` stops typical-user input but not paste / programmatic submit / non-browser clients. Backend truncates to first 17 chars at line 27379-27380 and emits advisory describing the overrun + truncated value. Stored value is the truncated form.

### 6b. Allowed character class (35d.md Gap E — LINE_35D_ACCOUNT_NUMBER_INVALID_CHARACTERS)

IRS Doc 1346 allowed class: digits `0-9`, letters `A-Z`, hyphen `-`. No spaces. Value is uppercased before regex check at line 27397 — lower-case letters pass without flagging. Common cause for failure: spaces between digit groups copied from a paper check. Advisory describes the cause and remedy.

### 6c. All-three-or-none triple rule (35d.md Gap F — LINE_35BCD_INCOMPLETE_TRIPLE)

IRS rejects a return where the taxpayer elected direct deposit but any of `routing number` / `account type` / `account number` is blank while the others are populated (Form 1040 §1f rule 1). Backend checks all three fields after gating; if any is missing and DD was elected, emits advisory listing which field(s) are missing.

All three flags are non-blocking — the compute proceeds and the printed return is generated. The user sees the advisories on the Tax Return view and can decide whether to fix and re-save or accept the truncated/incomplete state.

## 11. Summary Rule

For tax year 2025:

```text
line35a = roundMoney(line34 − line36Capped)         when line34 > 0 (overpayment branch)
line35a = null                                        otherwise OR when subtraction ≤ 0

line35b = trim(directDepositData.routingNumber)     when DD elected && !Form 8888 split
line35c = trim(directDepositData.accountType)       same gating
line35d = truncate17(trim(directDepositData.accountNumber)) same gating
                                                     + 3 advisory validations (Gap D/E/F)
```

When Form 8888 is attached: `35a` shows the total refund; `35b/35c/35d` are blank on Form 1040 and Form 8888 carries the routing/account/amount triples.
