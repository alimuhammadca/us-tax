### Test 1 — `Line 12e itemized flow computes Schedule A totals from the Deductions section`

```

```

**Purpose.** Exercises the ITEMIZED election path: user explicitly chooses to itemize, enters Schedule A inputs across four categories (medical, taxes, interest, charity), saves the form, computes the return, and asserts (a) Schedule A line 17 totals match the IRS sum, (b) Form 1040 line 12e takes the Schedule A line 17 amount, (c) the `usedForItemizedDeduction` flag is set, and (d) the `usedForStandardDeductionIncrease` flag is not set (since no disaster loss is involved). Also verifies the user can navigate to the Schedule A preview after compute.

#### Inputs the test seeds

| Wages | none seeded → AGI = $0 | — |
| `deductionElection` | `ITEMIZED` | — |

| `hasCharitableContributions` (gate) | true | — |
| `charitableCashContributions` | $600 | 11 |

#### IRS rule that applies

Per 2025 Schedule A (Form 1040) instructions + IRC §63(d):

```
schedule_a_line_1   = medicalDentalExpensesPaid               = $500
schedule_a_line_2   = AGI                                     = $0
schedule_a_line_3   = line_2 * 0.075                          = $0
schedule_a_line_4   = max(0, line_1 - line_3)                 = $500     (deductible medical)

schedule_a_line_5a  = stateLocalIncomeTaxesPaid               = $7,000
schedule_a_line_5b  = realEstateTaxesPaid                     = $4,000
schedule_a_line_5c  = personalPropertyTaxesPaid               = $0
schedule_a_line_5d  = line_5a + line_5b + line_5c             = $11,000
salt_cap            = $40,000 (Single, no phasedown since MAGI = AGI = $0)
schedule_a_line_5e  = min(line_5d, salt_cap)                  = $11,000
schedule_a_line_6   = foreignTaxesPaid                        = $0
schedule_a_line_7   = line_5e + line_6                        = $11,000  (total taxes)

schedule_a_line_8a  = homeMortgageInterestPaid                = $3,000
schedule_a_line_8e  = line_8a + line_8b + line_8c + line_8d   = $3,000
schedule_a_line_9   = min(investmentInterestPaid, netInvestmentIncome) = $0
schedule_a_line_10  = line_8e + line_9                        = $3,000   (total interest)

schedule_a_line_11  = charitableCashContributions             = $600
schedule_a_line_12  = charitableNonCashContributions          = $200
schedule_a_line_13  = priorYearCharitableContributionCarryover = $0
schedule_a_line_14  = line_11 + line_12 + line_13             = $800     (total charity)

schedule_a_line_15  = personalCasualtyAndTheftLoss            = $0
schedule_a_line_16  = otherAllowedItemizedDeductions          = $0
schedule_a_line_17  = line_4 + line_7 + line_10 + line_14 + line_15 + line_16
                    = $500 + $11,000 + $3,000 + $800 + $0 + $0
                    = $15,300                                    (TOTAL ITEMIZED)
```

Per spec §8 (line 12e election rule): since the user explicitly elected `ITEMIZED`, `Form1040.line12e = Schedule A line 17 = $15,300` — even though the Single base standard deduction ($15,750) is larger. The Schedule A line 18 "elect to itemize although less than standard" flag should also fire.

- `scheduleA.usedForItemizedDeduction = true` (set when `deductionElection == "ITEMIZED"`).
- `scheduleA.usedForStandardDeductionIncrease = false` (set only when `electDisasterLossStandardDeductionIncrease == true` — which the test does not seed).
- `scheduleA.electsToItemizeAlthoughLessThanStandard = true` (set when itemized $15,300 < standard $15,750 AND ITEMIZED election AND both positive — see `computeLine12` ~line 5196).

### Test 2 — `Line 12e disaster-loss standard deduction increase creates Schedule A and Form 4684 output`

```
test('Line 12e disaster-loss standard deduction increase creates Schedule A and Form 4684 output')
```

**Purpose.** Exercises the increased-standard-deduction-for-net-qualified-disaster-loss path (spec §7): the user takes the standard deduction AND elects to increase it by a net qualified disaster loss. Verifies (a) Form 1040 line 12e equals `base_standard + disaster_loss`, (b) the disaster amount is echoed on Schedule A, (c) the `usedForStandardDeductionIncrease` flag is set, and (d) Form 4684 is generated as a required attachment.

#### Inputs the test seeds

| Field | Value |
|---|---|
| Filing status | Single |
| Wages | none seeded → AGI = $0 |
| `deductionElection` | `STANDARD` |
| `hasDisasterLoss` (gate) | true |
| `netQualifiedDisasterLoss` | $250 |
| `electDisasterLossStandardDeductionIncrease` | true |

No other Schedule A inputs are entered.

#### IRS rule that applies

Per 2025 Form 1040 instructions for line 12e + Schedule A instructions for the increased standard deduction path + Form 4684 instructions:

```
base_standard_deduction       = $15,750  (Single 2025)
net_qualified_disaster_loss   = $250
augmented_standard_deduction  = $15,750 + $250 = $16,000
Form1040.line12e              = $16,000  (combined Schedule A line 16 amount per spec §7)
```

Per spec §7: "do **not** enter an amount on any other line of Schedule A" — so Schedule A line 17 should equal the combined Schedule A line 16 ($16,000), and every other Schedule A line should be blank.

Form 4684 attachment required because `netQualifiedDisasterLoss > 0` and the disaster-election flag is true.

#### Code path

- `computeStandardDeduction()` returns the base $15,750 (Single, no dependent / dual-status / age boxes).
- `computeLine12()` (~line 5188) augments: `standardDeduction = base + scheduleA.netQualifiedDisasterLoss = $16,000`.
- 12e.md Gap #9 closure (2026-06-08): after augmentation, `computeLine12` writes the combined amount into `scheduleA.otherAllowedItemizedDeductions` and `scheduleA.totalItemizedDeductions`, then nulls every other Schedule A line.
- `chooseLine12e()` STANDARD branch returns `$16,000`.
- `buildLine12Form4684Attachment` returns a non-null required-attachment record when `netQualifiedDisasterLoss > 0` and (`usedForItemizedDeduction == true` OR `usedForStandardDeductionIncrease == true`).

#### Test assertions and per-assertion verdict

| # | Assertion | IRS-expected | Verdict |

| 2 | `deductions.standardDeductionComputed === 16000` | $16,000 (base $15,750 + disaster $250) | **OK** |
| 3 | `deductions.deductionAmount === 16000` | $16,000 | **OK** |
| 4 | `scheduleA.netQualifiedDisasterLoss === 250` | $250 (echoed unchanged by Gap #9 closure) | **OK** |
| 5 | `scheduleA.usedForStandardDeductionIncrease === true` | true | **OK** |
| 6 | `form4684 !== null` | non-null (required attachment) | **OK** |

All 6 compute assertions match IRS-expected values.
