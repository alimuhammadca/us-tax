# Line 16 — Tax — e2e test catalog (audit)

Source spec: `C:\us-tax\us-tax-be\e2e\tests\line16-tax.spec.ts`
Audit date: 2026-06-08
Latest revision: **2026-06-08 — 11 new tests added closing nearly all per-test and cross-cutting gaps (suite now 20 tests; was 9). See closure log at the bottom for the full diff.**
Method: For every `test(...)` in the spec — (1) read the inputs the test seeds, (2) compute by hand what the IRS rule requires from those inputs (Form 1040 line 16; 2025 Tax Table / Tax Computation Worksheet; QDCG Worksheet; Foreign Earned Income Tax Worksheet; Form 4972 lump-sum tax; Form 8814 Box 1; box-3 write-ins), (3) read the Java compute path that handles those inputs, (4) compare against the `expect(...).toBe(...)` assertions, (5) classify each assertion **OK** / **MISMATCH** / **GAP**.

Backend entry point: `TaxReturnComputeService.computeLine16(...)`. See `C:\us-tax\XLS\Computations\16.md` for the full IRS-canonical decision tree (7 branches: ZERO → FOREIGN_EARNED_INCOME → FORM_8615 → SCHEDULE_D_TAX_WORKSHEET → QDCG → SCHEDULE_J (out of scope) → TAX_TABLE/TCW) plus Box 1 (Form 8814) + Box 2 (Form 4972) + Box 3 (ECR + 962 + 1291TAX + Form 8978 + 965INC).

Shared 2025 reference data:
- Standard deduction Single 2025: **$15,750** (`STANDARD_DEDUCTION_BASE_SINGLE`).
- Single bracket subtraction constants: 12% bracket $48,475 ceiling, constant $238.50; 22% bracket $103,350 ceiling, constant $5,086; 24% bracket $197,300 ceiling, constant $7,153.
- Tax Table label boundary: line 15 < $100,000 → method label `TAX_TABLE`; ≥ $100,000 → `TAX_COMPUTATION_WORKSHEET` (math identical).
- QDCG 0% ceiling Single: $48,350; 20% floor Single: $533,400.
- Form 4972 Part II rate: 20% flat on the capital-gain portion of an eligible lump-sum distribution (born-before-1936 participant only).

## Test 1 — `Line 16 — no taxable income: tax is null or zero`

Command (run individually):
```powershell
cd C:\us-tax\us-tax-be\e2e
npx playwright test --workers=1 --output=test-results-16 -g "no taxable income: tax is null or zero" line16-tax.spec.ts
```

### Purpose
Smoke test for the ZERO branch of the decision tree — when no income / no deductions are seeded, line 15 is zero (or null) and line 16 must also be zero (or null). Confirms the compute layer doesn't return a phantom tax on an empty return.

### Inputs the test seeds
| Where | Field | Value |
|---|---|---|
| `seedRequiredDob` | DOB + SSN | default 1990-01-01, SSN `111-11-1111` |
| (nothing else) | — | — |

`computeReturnApi(page, { overrideFlags: true })` so the empty return doesn't 409 on the missing-AGI / missing-filing-status warnings.

### Outputs the test asserts
| Path | Asserted value |
|---|---|
| `taxAndCredits?.tax` | `null` OR `0` (either is acceptable) |

### Hand computation per IRS rule
With no income and no deductions:
- Line 1z (wages) = $0; line 11b (AGI) = $0; line 12e (standard deduction) = $15,750 (Single default).
- Line 15 = max(0, $0 − $15,750) = $0.
- Decision tree branch 1 (`line 15 ≤ 0`) fires → `regularTax = $0`.
- No Form 8814 / 4972 / Box 3 → all add-ons = $0.
- **Line 16 = $0.**

### Java compute path
- `computeLine16` reads `line15 = $0` and immediately hits the ZERO branch (`line15.compareTo(BigDecimal.ZERO) <= 0`).
- `regularTax = ZERO`, `computationMethod = "ZERO"`.
- Box 1/2/3 add-on paths all see no inputs → zero.
- `setTax(BigDecimal.ZERO)` on `TaxAndCredits`.

Actually — looking at the code more carefully: `line15` may be `null` (not zero) when the compute service decides line 15 was never finalized. In that case `computeLine16` returns at the early null-guard without setting `tax`, so `TaxAndCredits.tax` remains `null`. The assertion `tax === null || tax === 0` covers both possibilities.

### Per-assertion verdict
| # | Assertion | Verdict |
|---|---|---|
| 1 | `tax === null || tax === 0` | **OK** |

### Coverage gaps
| # | Gap | Why it matters |
|---|---|---|
| G1 | No assertion on `computationMethod`. | A regression that left a populated `computationMethod` (e.g., `"TAX_TABLE"` with zero tax) would pass this test but be misleading on the rendered PDF. |

Command:


### Purpose
Verifies that the presence of qualified dividends on a 1099-DIV routes line 16 through the QDCG Worksheet (Branch 5) rather than the ordinary Tax Table. Confirms `computationMethod = "QDCG"`.

### Inputs the test seeds
Same as Test 2 (Single, W-2 $50,000, employment-income opt-in), plus:

| Where | Field | Value |
|---|---|---|

### Outputs the test asserts
| Path | Asserted value |
|---|---|
| `taxAndCredits` | truthy |
| `computationMethod` | `"QDCG"` |
| `tax` | > 0 |

### Hand computation per IRS rule (QDCG Worksheet, Single)
```
Line 1z (wages)                = $50,000
Line 3a (qualified dividends)  = $4,000
Line 3b (ordinary dividends)   = $5,000
Line 11b (AGI)                 = $50,000 + $5,000 = $55,000
                                  (qualified dividends ride within ordinary on line 3b)
Line 12e (standard deduction)  = $15,750
Line 15                        = $55,000 − $15,750 = $39,250

Decision tree: qualified dividends > 0 AND no Schedule D worksheet path required
                  → Branch 5 (QDCG Worksheet).

QDCG allocation (Single):
  preferentialIncome = min($39,250, $4,000 + $0 net LTCG) = $4,000
  ordinaryIncome     = $39,250 − $4,000 = $35,250

  Single 0% ceiling = $48,350; 20% floor = $533,400.
  zeroRoom      = max(0, $48,350 − $35,250) = $13,100
  zeroPortion   = min($4,000, $13,100)      = $4,000   ← all $4,000 fits in the 0% bracket
  twentyRoom    = max(0, $39,250 − $533,400) = $0
  twentyPortion = min($0, $0) = $0
  fifteenPortion = $4,000 − $4,000 − $0      = $0

  Ordinary bracket on $35,250 (Single 12%):
    = $35,250 × 0.12 − $238.50 = $4,230 − $238.50 = $3,991.50

  Preferential tax = $0 × 0 + $0 × 0.15 + $0 × 0.20 = $0

  total = $3,991.50 + $0 = $3,991.50 → rounded $3,992
```

**Line 16 = $3,992** (entirely from ordinary brackets — qualified dividends fit in the 0% rate band for this filer).

### Java compute path
- `isQDCGWorksheetRequired(form1040, scheduleD)` returns true because `qualifiedDividends > 0`.
- Branch 5 fires.
- `computeQDCGWorksheet($39,250, "Single", $4,000, null, $0, null)`.
- Code allocates `preferentialIncome = $4,000`, `ordinaryIncome = $35,250`, all $4,000 lands in the 0% bucket.
- `tax = bracket($35,250) + 0×$4,000 = $3,992`.
- `regularTax = $3,992`, `computationMethod = "QDCG"`.

### Per-assertion verdict
| # | Assertion | Verdict |
|---|---|---|
| 1 | `taxAndCredits` truthy | **OK** |
| 2 | `computationMethod === "QDCG"` | **OK** |
| 3 | `tax > 0` | **OK** ($3,992) |
