# Line 1c — End-to-End Test Cases

Run each command from `C:\us-tax\us-tax-be\e2e` after Docker Desktop, the backend (`./run-dev.ps1`), and the UI (`npm start`) are running. Each test resets the local Quarkus Dev Services MSSQL container's user data first (via `clearUserData`) so prior runs don't pollute it.

Shared SSNs used throughout: **taxpayer = `111-11-1111`**, **spouse = `222-22-2222`** (when seeded).

Spec files:
- `C:\us-tax\us-tax-be\e2e\tests\line1c-tip-income.spec.ts` — 13 tests (main scenarios).
- `C:\us-tax\us-tax-be\e2e\tests\line1c-tip-income-overflow.spec.ts` — 2 tests (Form 4137 row-overflow / no-truncation guards).

The two spec files together cover the 12 scenarios listed in `lines/1c.md` §9. The scenario mapping at the top of `line1c-tip-income.spec.ts` is the canonical index.

A note on a recurring pattern in these tests: many seed scenarios omit a Form W-2 statement and rely on the per-employer `socialSecurityWagesW2Box3` form-input fallback (set to `0`). This intentionally exercises the "no matching W-2 in system" path. When the fallback is *not* set, the `MISSING_SS_W2_DATA_TAXPAYER` blocking flag fires and the test either accepts the auto-confirm dialog (UI path) or passes `overrideFlags: true` (API path) to obtain the computation. The corresponding Line 11 (Social Security tax) on Form 4137 will be `null` in those cases — Line 12 (Medicare tax) still computes and feeds Schedule 2 line 5.

## Test 1 — Unreported cash tips flow to line 1c with Medicare-only tax when SS W-2 data is missing

```powershell
npx playwright test --workers=1 --output=test-results-1c-124 line1c-tip-income.spec.ts:124
```

**Purpose.** The basic happy path for unreported cash and charge tips. Verifies (a) the per-employer formula `max(0, received - reported)` correctly flows to Line 1c, (b) Form 4137 line 13 routes to Schedule 2 line 5, and (c) when no Form W-2 is in the system the `MISSING_SS_W2_DATA_TAXPAYER` blocking flag fires and only the Medicare half of the Form 4137 tax is computed.

**Inputs (via UI).**
- Employment / Personal: taxpayer identification (DOB + address) seeded by `seedRequiredDob`.
- Tip Income (taxpayer): `hasUnreportedTips = Yes`; one employer "Downtown Diner" with `employerEIN = 12-3456789`, `totalTipsReceived = 500`, `totalTipsReportedToEmployer = 200`.
- No Form W-2 statement seeded; no `socialSecurityWagesW2Box3` fallback.

**Outputs.**
- Form 1040 line 1c (`form1040.income.tipIncome`) = **`300`** (= 500 − 200; no non-cash component).
- Schedule 2 line 5 (`schedule2.otherTaxes.unreportedTipIncomeTax`) = **`4`** (Medicare-only: 300 × 1.45% = 4.35 → 4; Social Security half suppressed because SS wage base is missing).
- Flags include `MISSING_SS_W2_DATA_TAXPAYER` (blocking, overrideable).
- The UI auto-confirms the "Compute anyway?" dialog → compute runs with `overrideFlags=true` → the computation is returned with the flag listed in `flags`.

# IRS-Compliance Audit (2026-05-30)

Desk-audit of all 15 tests across `line1c-tip-income.spec.ts` (13 tests) and `line1c-tip-income-overflow.spec.ts` (2 tests) against IRS source documents:

- **2025 Instructions for Form 1040 / 1040-SR** (`docs/books/i1040gi_2025.pdf`) — Line 1c definition (three categories of unreported tip income).
- **Publication 17 — Your Federal Income Tax** (`docs/books/p17.pdf`) — Chapter 5, Wages, Salaries, and Other Earnings.
- **Publication 525 — Taxable and Nontaxable Income** (`docs/books/p525.pdf`) — confirms all tip income is taxable.
- **Publication 531 — Reporting Tip Income** (`docs/books/p531.pdf`, downloaded 2026-05-30 from <https://www.irs.gov/pub/irs-pdf/p531.pdf>). Same rules are also restated in p17 and the Form 4137 instructions.
- **Form 4137 — Social Security and Medicare Tax on Unreported Tip Income** (`docs/books/f4137--2025.pdf`, downloaded 2026-05-30 from <https://www.irs.gov/pub/irs-prior/f4137--2025.pdf>). Source for the $20-per-employer-per-month tip-reporting threshold (Form 4137 line 5), the line-by-line computation, and the IRS-e-file EIN-required rule. Instructions embedded on the form.
- **2025 FICA rates** — Social Security 6.2 percent (employee share, on wages up to $176,100); Medicare 1.45 percent (no cap).

## Observations and recommended follow-ups

These are NOT verdict downgrades on the existing tests — they are forward-looking notes for future hardening:

1. **`ADEQUATE_RECORDS_MISSING_SUBSTANTIATED_AMOUNT_*` promoted to §17 non-overrideable on 2026-05-30.** The flag fires when the application *knows* the computed Line 1c is wrong — the user said the lower (substantiated) amount applies but with the field blank, compute would use the higher Box 8 allocated amount as a silent fallback. Now added to `NonOverrideableFlags.CODES` so `overrideFlags=true` no longer bypasses it. The §17 strict-criteria match here is "spirit of §17 applied to over-reporting": the IRS does NOT catch over-reporting (the user just over-pays tax) but the application's own contract says it will not facilitate computing a return that contradicts what the user explicitly told it. Flag message rewritten to §17 format (named consequence + concrete resolutions, no "proceed anyway"). Test 12 continues to pass.
2. **`MISSING_EMPLOYER_EIN_FORM_4137_*` is also a §17-adjacent candidate.** The arithmetic on Line 1c and Form 4137 is correct; the issue is e-file rejection on submission. Less obviously a §17 fit ("wrong return" vs. "unfileable return") but still worth considering.
3. **`MISSING_SS_W2_DATA_*` should remain overrideable.** It signals incomplete data (Line 11 SS tax suppressed) but the user may have a legitimate reason to file with only Medicare tax computed (e.g., paper filing with manual SS-tax calculation outside the application). Keep as is.
