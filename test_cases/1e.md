# Line 1e — End-to-End Test Cases

Run each command from `C:\us-tax\us-tax-be\e2e` after Docker Desktop, the backend (`./run-dev.ps1`), and the UI (`npm start`) are running. Each test resets the local Quarkus Dev Services MSSQL container's user data first (via `clearUserData`) so prior runs don't pollute it.

Shared SSNs used throughout: **taxpayer = `111-11-1111`**, **spouse = `222-22-2222`** (used in MFJ tests + Test 13 only).

Spec file: `C:\us-tax\us-tax-be\e2e\tests\line1e-dependent-care.spec.ts` — 16 tests. Tests 1–13 are computational and verified against IRS rules (Pub 503, Form 2441 instructions, IRC §129); tests 14–16 are pure UI behavior with no IRS computation to verify.

A note on the IRS source of truth (cross-referenced section by section to `C:\us-tax\XLS\Computations\1e.md` §1.5): Line 1e = Form 2441 Part III line 26 = `max(0, line15 − line25)` where `line25 = min(line20, line21)`, `line20 = min(line17, line18, line19)`, `line17 = min(line15, line16)`, and `line15 = max(0, line12 + line13 − line14)`. The §129(d) nondiscrimination failure branch sets `line25 = 0` outright (full benefit taxable).

A note on the two-pass Form 2441 computation: Part III (lines 12–31 → line 1e) runs in Phase 1 BEFORE AGI is known; Part II credit lines 7–11 are finalized in Phase 2 AFTER AGI + foreign tax credit. Tests 8 and 12 exercise Phase 2 explicitly.

A note on the 2026-05-21 form restructure: deemed-income (taxpayer + spouse), prior-year expenses, plan max, line 16 override, additional earned income, and the §129(d) question are all hidden behind the "Less common situations" toggle. Many tests call `expandLessCommon(page)` to reveal them.

### 2026-05-31 implementation gap closures (Gaps 1 / 2 / 3 from XLS/Computations/1e.md §3.4)

Three documented implementation gaps were addressed:

- **Gap 1 — Pub 503 Worksheet A current-year cap**: `TaxReturnComputeService.computeDependentCareBenefits` now caps Form 2441 line 9b at the IRC §21(c) per-year ceiling ($3,000 / $6,000).
