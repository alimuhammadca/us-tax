Run each command from `C:\us-tax\us-tax-be\e2e` after Docker Desktop, the backend (`./run-dev.ps1`), and the UI (`npm start`) are running. Each test resets the local Quarkus Dev Services MSSQL container's user data first (via `clearUserData`) so prior runs don't pollute it.

Tests 4, 5, 6, 9, 10 are pure save-time / UI behaviors with no IRS numerical computation to verify.

A note on the IRS source of truth (cross-referenced section by section to `C:\us-tax\XLS\Computations\1f.md` §1.5): Line 1f = Form 8839 Part III line 31 = `line23 − line30` where `line23 = ΣL22[i]` (current-year allocation per child), `line30 = ΣL29[i]` (excluded after phaseout per child), `L29[i] = L24[i] × (1 − phaseoutFraction)`, `L24[i]` is the smaller of remaining lifetime cap and current-year allocation EXCEPT the special-needs branch (`L24 = L21`) and the foreign-deferred branch (`L24 = min(L21, L22 + deferred)`). `L21[i] = max(0, $17,280 − L20[i])` where L20 is prior-year benefits already excluded. Phaseout runs $259,190 → $299,190 in 2025. Sources: Internal Revenue Code §137; Form 8839 Part III; Publication 17 (Adoption Credit chapter); Publication 525 (Adoption Assistance subsection); 2025 Instructions for Form 1040 (line 1f label).

A note on the IRC §1372 + §137(b)(1) + §137(d) gaps closed 2026-05-23 and the §137(e) + line 23 source + PYAB write-in + Modified Adjusted Gross Income add-backs closed 2026-05-24: Tests 12 (shared adoption), 13 (gross/reimbursement split), 14 (special-needs off-W-2), 15 (§1372 partition), and 16 (foreign-finality deferred) exercise each gap closure end-to-end. Tests 14 and 16 also verify the "PYAB" write-in renders on Form 1040 when line 1f is negative.

A note on `Less common situations`: the §1372 outer screening (`section1372AnyEmployer`) and the shared-adoption outer screening (`sharedAdoptionAnyChild`) live inside a collapsible "Less common situations" toggle on the adoption-expenses form. Tests that don't have either situation must still answer No on both to unlock Save. The `answerLessCommonNo(page)` helper handles this.

## Test 1 — Taxable adoption benefits flow to line 1f

```powershell
npx playwright test --workers=1 --output=test-results-1f-60 line1f-adoption-benefits.spec.ts:60
```

**Purpose.** The basic happy path: a Single filer with one W-2 carrying box 12 code T benefits AND a child with prior-year benefits that mostly consume the $17,280 lifetime cap. Verifies that the smaller-of-two-numbers logic (`line24 = min(line21, line22)`) routes the excess current-year benefit ($5,000 allocated, only $1,280 remaining cap) to Form 1040 line 1f as taxable wages ($3,720).

**Inputs.**
- Single (default from `seedRequiredDob`).
- W-2 (taxpayer): wages 50000, box 12 code T = $5,000.
- Adoption expenses: 1 child (Ava Stone); `child1PriorYearBenefits = 16000`; `child1Benefits2025 = 5000`. No special needs, no foreign, no shared adoption, no §1372.

**IRS hand-math (Form 8839 Part III + IRC §137).**
- line19 = **$17,280** (per-child cap; no shared adoption reduction)
- line20 = **$16,000** (prior-year benefits)
- line21 = max(0, 17280 − 16000) = **$1,280** (remaining lifetime cap)
- line22 = **$5,000** (current-year allocation)
- line24 = min(line21, line22) = min(1280, 5000) = **$1,280** (smaller-of branch)
- line23 = ΣL22[i] = **$5,000**
- Modified Adjusted Gross Income ≈ $50,000 (wages only); below $259,190 phaseout start → phaseoutFraction = **0**
- line28 = line24 × 0 = **$0**
- line29 = line24 − line28 = **$1,280**
- line30 = ΣL29[i] = **$1,280**
- line31 = line23 − line30 = 5000 − 1280 = **$3,720**
- Form 1040 line 1f = **$3,720**

**Verdict table.**

| Assertion | Asserted | IRS expected | Verdict |
|---|---|---|---|
| `form1040.income.adoptionBenefits` | 3720 | 3720 | **OK** |
