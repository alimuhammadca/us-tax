# Line 1g — End-to-End Test Cases

Run each command from `C:\us-tax\us-tax-be\e2e` after Docker Desktop, the backend (`./run-dev.ps1`), and the UI (`npm start`) are running. Each test resets the local Quarkus Dev Services MSSQL container's user data first (via `clearUserData`) so prior runs don't pollute it.

Shared SSNs used throughout: **taxpayer = `111-11-1111`**, **spouse = `222-22-2222`** (used in MFJ + MFS tests).

Spec file: `C:\us-tax\us-tax-be\e2e\tests\line1g-uncollected-ss-medicare.spec.ts` — 14 tests. All 14 tests are computational (numerical assertions on Form 1040 line 1g, Schedule 2 line 6, Form 8919 sub-lines, Form 8959 line 3, or specific flag emissions). There are no pure-UI tests in this spec.

A note on the IRS source of truth (cross-referenced section by section to `C:\us-tax\XLS\Computations\1g.md` §1.5): Line 1g = Form 8919 line 6 (sum across both spouses on MFJ). The companion Form 8919 lines 7–13 produce the uncollected Social Security + Medicare tax that flows to Schedule 2 line 6. `line7 = $176,100` (2025 Social Security wage base), `line8 = W-2 box 3 + W-2 box 7 + Railroad Retirement Tax Act tier 1 (W-2 box 14) + Form 4137 line 10 (unreported tips subject to Social Security)`, `line9 = max(0, line7 − line8)`, `line10 = min(line6, line9)`, `line11 = line10 × 6.2%`, `line12 = line6 × 1.45%` (no wage base cap), `line13 = line11 + line12`. Sources: Internal Revenue Code §3101 (employee FICA rates), §3121 (FICA wages and the employee-vs-contractor test), Form 8919 (the form itself), 2025 Instructions for Form 1040, Publication 17 (Wages chapter), and Publication 531 (Reporting Tip Income).

A note on the four IRS reason codes (Form 8919 column c): codes A, C, G, H. Codes A and C require an IRS determination / correspondence date in column (d); codes G and H do not. Code G means a separate Form SS-8 has been filed and is pending; code H means the worker received both a W-2 and a 1099-MISC/NEC from the same firm for the same services. Tests 6, 7, 10, 11 exercise each code's specific validation pattern.

A note on per-spouse multiplicity: Form 8919 is a per-spouse form. On MFJ, two Forms 8919 are possible (one per spouse), and Form 1040 line 1g aggregates both. On Married Filing Separately, only the taxpayer's Form 8919 is read (the spouse's stale data is correctly excluded per Internal Revenue Code §6013). Tests 8 and 9 cover the MFJ aggregation and the MFS exclusion respectively.

A note on execution order: Form 8919 line 8 includes `form4137.line10` (unreported tips subject to Social Security). `computeTips()` runs BEFORE `computeUncollectedSSTax()` so the line 10 value is available. Reversing the order would silently understate line 8, overstate line 9, and overcharge Social Security tax on line 11. Test 12 exercises this dependency explicitly.

### 2026-05-31 §17 promotion of `FORM8919_CODE_H_1099_*`

Separately from the test-coverage hardening above, the application also promoted the reason-code-H + no-1099 validator from non-blocking soft warning to **blocking + rules.md §17 non-overrideable** on 2026-05-31. The rationale: per the 2025 Instructions for Form 8919, code H applies ONLY when the worker received both a W-2 and a 1099-MISC/1099-NEC from the same firm for the same services — the code is structurally about reclassifying part of the 1099 amount as wages. Without a 1099 there is nothing to reclassify; filing as-is invites near-certain IRS document-matching correspondence (typically a CP-2057 letter). The promotion was made because no legitimate scenario produces code H + no 1099 — the worker always has a correct alternative (switch to code G/A/C or delete the firm row).

This is a *behavior change*, not just a test hardening — returns with code H + `received1099 = false` now return HTTP 409 (compute blocked) instead of HTTP 200 with a soft-warning flag. Test 11 was rewritten end-to-end to assert the 409-with-flag pattern with and without `overrideFlags=true`. The unit test `TaxReturnComputeServiceTest.form8919CodeHWithout1099RaisesBlockingFlag` (renamed from the prior `form8919CodeHWithout1099EmitsSoftWarning`) verifies `isBlocking() == true` AND `NonOverrideableFlags.isNonOverrideable(flag.getCode()) == true`. Backend suite: 593/593 unit tests pass.
