This document audits every test in `line1i-combat-pay.spec.ts` against the IRS rules in the 2025 Instructions for Form 1040 (file `i1040gi_2025.pdf`), Publication 17, Publication 590-A, and the project's `XLS/Computations/1i.md`. For each test the audit records:

1. **Command** — how to run the test in isolation from a normal terminal.
2. **Purpose** — what the test exists to verify.

4. **IRS rule that applies** — the hand-computation the IRS expects.

6. **Test assertions** — the `expect(...)` calls.
7. **Per-assertion verdict** — `OK`, `MISMATCH`, or `GAP` per assertion, with reasoning.

The `OK` / `MISMATCH` / `GAP` codes mean:

- **OK** — the assertion matches what the IRS rule would compute.
- **MISMATCH** — the assertion expects a value the IRS rule says is wrong; the test would pass against buggy code.
- **GAP** — the assertion is silent on an output the IRS rule says should also be populated, or the assertion is loose (e.g., `> 0` when a specific value is computable).

## Test 1 — Combat pay election for taxpayer flows to Form 1040 line 1i

**Command (isolated):**

**Purpose.** Verify that when a taxpayer has a Form W-2 with box 12 code Q (nontaxable combat pay) and elects to include it in earned income, the elected amount flows to Form 1040 line 1i (`Income.nontaxableCombatPayElection`).

**Inputs the test seeds.**

| Where | Field | Value |
|---|---|---|
| Identification | Taxpayer Social Security Number | `111-11-1111` |
| Identification | Taxpayer date of birth | `1990-01-01` (default — age 35) |
| W-2 statement | `employeeSSN` | `111-11-1111` (matches taxpayer) |
| W-2 statement | `wagesTipsOtherCompAmount` (box 1) | `50000` |
| W-2 statement | `socialSecurityWagesAmount` (box 3) | `50000` |
| W-2 statement | `box12Entries[0].code` | `Q` (Nontaxable combat pay) |
| W-2 statement | `box12Entries[0].box12Amount` | `2500` |
| Personal form `combat-pay-taxpayer` | `electCombatPay` | `true` (clicked "Yes") |

No spouse seeded — filing status defaults to Single.

**IRS rule that applies.**

- Per the 2025 Instructions for Form 1040, line 1i (page 108 of `i1040gi_2025.pdf`): *"If you elect to include nontaxable combat pay in figuring your EIC, enter the amount on Form 1040 or 1040-SR, line 1i."*
- Per `XLS/Computations/1i.md` §1.5:
  - `electedCombatPay_taxpayer = electCombatPay_taxpayer ? combatPayTotal_taxpayer : 0`
  - `Form1040.Line1i = electedCombatPay_taxpayer + electedCombatPay_spouse`

Hand computation:

```text
combatPayTotal_taxpayer = sum of W-2 box 12 code Q amounts attributed to taxpayer = $2,500
electCombatPay_taxpayer = true → electedCombatPay_taxpayer = $2,500
combatPayTotal_spouse   = 0 (no spouse on return)
electedCombatPay_spouse = 0
Form1040.Line1i         = $2,500 + 0 = $2,500
```

**Java compute path.**

`computeCombatPay` at `TaxReturnComputeService.java:14540` —

```text
taxpayerSsn   = normalize("111-11-1111") = "111111111"
spouseSsn     = null (no spouse seeded; Single filing status default)
taxpayerTotal = sumW2Box12ByCodes(w2s, {"Q"}, "111111111") = $2,500
spouseTotal   = null
electTaxpayer = true
electSpouse   = null
taxpayerElected = $2,500
spouseElected   = null
line1iRaw       = addNonNull($2,500, null) = $2,500
line1i          = roundMoney($2,500) = $2,500
```

Result: `Income.nontaxableCombatPayElection = $2,500`.

**Test assertions.**

```text
expect(computation?.form1040?.income?.nontaxableCombatPayElection).toBe(2500);
expect(computation?.form1040?.income?.totalWages).toBe(50000);
expect(computation?.form1040?.income?.totalIncome).toBe(50000);
expect(computation?.form1040?.adjustments?.adjustedGrossIncome).toBe(50000);
```

**Per-assertion verdict.**

| Assertion | Expected | IRS value | Verdict |
|---|---|---|---|
| `nontaxableCombatPayElection === 2500` | $2,500 | $2,500 | **OK** |
| `totalWages === 50000` (line 1z) | $50,000 | $50,000 (combat pay election excluded by IRS rule) | **OK** (added 2026-06-01, Round B) |
| `totalIncome === 50000` (line 9) | $50,000 | $50,000 (combat pay election excluded) | **OK** (added 2026-06-01, Round B) |
| `adjustedGrossIncome === 50000` (line 11) | $50,000 | $50,000 (combat pay election excluded) | **OK** (added 2026-06-01, Round B) |
