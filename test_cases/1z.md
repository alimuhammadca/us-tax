## Test 1 — Line 1z sums lines 1a-1h and excludes line 1i

**Command (isolated):**

```powershell
cd C:\us-tax\us-tax-be\e2e
npx playwright test --workers=1 --output=test-results-1z-t1 -g "Line 1z sums lines 1a-1h and excludes line 1i" line1z-total-wages.spec.ts
```

**Purpose.** Lock in the IRS rule that Line 1z is the sum of *only* lines 1a through 1h, and that Line 1i (Nontaxable combat pay election) is held separately for Earned Income Credit / Additional Child Tax Credit purposes and is **never** added to wages. A regression that summed line 1i into wages would silently inflate the taxpayer's taxable wages, total income, adjusted gross income, and tax liability.

### Inputs the test seeds

| Personal form `other-earned-income` | `manualStrikeBenefitsAmount` | `300` |
| Personal form `combat-pay-taxpayer` | `electCombatPay` | `true` (Yes) |

### IRS rule that applies

The controlling sources are the 2025 Instructions for Form 1040 (`i1040gi_2025.pdf`), Publication 17 chapter 5 (`p17.pdf`), and Publication 525 (`p525.pdf`).

- **Line 1a** (Total amount from Form(s) W-2, box 1) = sum of every eligible W-2 box 1 amount belonging to the taxpayer. Here a single W-2 with `$2,000` in box 1 and no carve-outs (no box 11 Nonqualified plans, no Statutory employee, no inmate wages). Line 1a = `$2,000`.
- **Line 1h** (Other earned income) = sum of catch-all earned-income items including strike-and-lockout benefits (per Publication 525, "Strike and lockout benefits" are earned income for Form 1040 purposes). Here the taxpayer reports `$300`. Line 1h = `$300`.
- **Line 1i** (Nontaxable combat pay election) = the elected portion of W-2 box 12 code Q amounts. Here box 12 code Q = `$500` and the taxpayer elects to include combat pay in earned income for the Earned Income Credit / Additional Child Tax Credit per Internal Revenue Code §32(c)(2)(B)(vi). Line 1i = `$500`.
- **Lines 1b, 1c, 1d, 1e, 1f, 1g** = no inputs were seeded; each is `$0` (or null, which propagates as zero in the sum).
- **Line 1z** = Line 1a + Line 1b + Line 1c + Line 1d + Line 1e + Line 1f + Line 1g + Line 1h
  - = `$2,000 + $0 + $0 + $0 + $0 + $0 + $0 + $300`
  - = **`$2,300`**

Per Internal Revenue Code §32(c)(2)(B)(vi) and the explicit IRS labelling on Form 1040, **Line 1i is excluded from Line 1z.** Combat pay flowing into Line 1z would constitute including a *nontaxable* item as wages — a substantive error.

### Code path that handles these inputs

- `TaxReturnComputeService.java:6065` calls `computeLine1aWages(...)` which reads `wagesTipsOtherCompAmount` and produces `line1a = $2,000`.
- `TaxReturnComputeService.java` `computeOtherEarnedIncome(...)` reads `manualStrikeBenefitsAmount` from the `other-earned-income` personal form and produces `line1h = $300`.
- `TaxReturnComputeService.java` `computeCombatPay(...)` reads box 12 code Q (`$500`) and `electCombatPay = true`, producing `combatPayElection = $500`.
- `TaxReturnComputeService.java:6215-6217` folds the eight sub-line values with `addNonNull(...)`: `$2,000 + $300 = $2,300`. Combat pay is stored separately at `:6347` via `income.setNontaxableCombatPayElection(...)` and is never added to `totalWagesRaw`.

## Test 3 — Negative line 1f from special-needs adoption reduces line 1z (gap 3)

**Command (isolated):**

```powershell
cd C:\us-tax\us-tax-be\e2e
npx playwright test --workers=1 --output=test-results-1z-t3 -g "Negative line 1f from special-needs adoption" line1z-total-wages.spec.ts
```

**Purpose.** Lock in the IRS rule that Line 1f (Employer-provided adoption benefits) can be **negative** when the taxpayer adopts a child whom a state agency has determined to be a child with special needs and the adoption is finalised in the tax year. Under Internal Revenue Code §137(a)(2)–(3), the excludable benefit amount equals the full per-child limit ($17,280 for tax year 2025) regardless of actual expenses, so the taxable benefit (Form 8839 Part III, line 29) becomes `box 12 code T amount − $17,280` and may go below zero. Line 1z must propagate that negative amount, since the floor at zero appears only on Form 1040 line 15 (Taxable income), not on Line 1z. A regression that clamped Line 1f or Line 1z at zero would over-state the taxpayer's adjusted gross income and tax liability.

### Inputs the test seeds

| Where | Field | Value |
|---|---|---|
| Identification | Taxpayer Social Security Number | `111-11-1111` |
| W-2 statement | `employeeSSN` | `111-11-1111` |
| W-2 statement | `wagesTipsOtherCompAmount` (W-2 box 1) | `50000` |
| W-2 statement | `socialSecurityWagesAmount` (W-2 box 3) | `50000` |
| W-2 statement | `box12Entries[0]` | `{ code: 'T', box12Amount: 2000 }` (employer-provided adoption benefits) |
| Personal form `adoption-expenses` | `claimsAdoptionExpensesOrBenefits` | `Yes` |
| Personal form `adoption-expenses` | child 1 `childFirstName` | `Leo` |
| Personal form `adoption-expenses` | child 1 `childLastName` | `Perez` |
| Personal form `adoption-expenses` | child 1 `hasSpecialNeeds` | `Yes` |
| Personal form `adoption-expenses` | child 1 `adoptionFinal2025OrEarlier` | `Yes` |
| Personal form `adoption-expenses` | child 1 `child1Benefits2025` (employer-provided portion in 2025) | `2000` |
| Personal form `adoption-expenses` | `section1372AnyEmployer` (Less common situations) | `No` |
| Personal form `adoption-expenses` | `sharedAdoptionAnyChild` (Less common situations) | `No` |

### IRS rule that applies

The controlling sources are the 2025 Instructions for Form 8839 (Qualified Adoption Expenses), Internal Revenue Code §137 (Adoption Assistance Programs), Internal Revenue Code §23(h) (Adjustments for Inflation — which §137(b)(2)(B) incorporates), and Publication 17 chapter 5.

**Step A — Reference data.** For tax year 2025, the per-child employer-provided adoption-benefits exclusion ceiling is `$17,280` (`MAX_EXCLUSION_PER_CHILD_2025`). This is the value Internal Revenue Code §137(b)(1) and §23(h) yield after the cost-of-living adjustment.

**Step B — Special-needs full-exclusion rule.** Under Internal Revenue Code §137(a)(2), an employer-provided-adoption-benefit exclusion is available; under §137(a)(3), when the adoption of a child whom a state has determined to be a child with special needs becomes final, the exclusion is the **full per-child limit** ($17,280 for 2025) regardless of actual qualified expenses incurred. The Form 8839 Part III worksheet implements this by setting line 22 (the excludable amount) to the full $17,280 for that child.

**Step C — Form 8839 Part III line 29 (Taxable benefits) for child 1.**
- Line 22 (excluded benefits, special-needs final ceiling) = `$17,280`
- Line 25 (employer-provided benefits received in the year) = `$2,000` (W-2 box 12 code T)
- Line 29 (Taxable benefits) = Line 25 − Line 22 = `$2,000 − $17,280` = **`−$15,280`**

A negative Line 29 reflects that the taxpayer is allowed to *exclude more than the employer paid*. The IRS Form 8839 instructions for Part III allow this carve-out specifically for the special-needs-final case. The amount flows to Form 1040 line 1f as a negative number.

**Step D — W-2 box 12 code T treatment.** W-2 box 12 code T is NOT included in W-2 box 1, per the IRS General Instructions for Forms W-2 and W-3. So Line 1a continues to be `$50,000` (the full box 1 amount) without any adjustment for the `$2,000` box 12 code T entry.

**Step E — Line 1z.**
- Line 1a = `$50,000`
- Line 1f = `−$15,280`
- Lines 1b, 1c, 1d, 1e, 1g, 1h = `$0` each
- Line 1z = `$50,000 + $0 + $0 + $0 + $0 + (−$15,280) + $0 + $0` = **`$34,720`**

This negative-pass-through is explicitly documented by `lines/1z.md` §6 ("Negative component handling") and `XLS\Computations\1z.md` §1.5 ("Negative amounts").

### Code path that handles these inputs

- `computeLine1aWages(...)` reads box 1 (`$50,000`) and does NOT subtract box 12 code T (Code T is already outside box 1). Result: `line1a = $50,000`.
- `computeAdoptionBenefits(...)` reads the `adoption-expenses` personal form plus the W-2 box 12 code T amount; applies Form 8839 Part III with the special-needs-final ceiling; produces `line1f = −$15,280`.
- `TaxReturnComputeService.java:6215-6217` folds: `$50,000 + (−$15,280) = $34,720`. Critically, `addNonNull` performs algebraic addition with **no** `max(0, …)` floor — see the inline comment at `:6218-6220`. Negative line 1f propagates through to line 1z.

### Test assertions


### What the three tests do NOT collectively lock in

The audit identifies four coverage gaps that span the spec rather than a single test:

1. **No test asserts downstream line 9 (Total income).** Every line-1z test would benefit from a paired `expect(computation?.form1040?.income?.totalIncome).toBe(<same value as totalWages>)` (when no other income lines are seeded). This catches regressions where Line 1z is right but the sum into Line 9 silently drops Line 1z as an operand.
2. **No test asserts downstream Adjusted gross income (line 11).** Same regression-target as the line-9 gap.
3. **No test asserts the absence of phantom sub-line values.** Each test could harden its lock-in by asserting that the un-seeded sub-lines (`householdEmployeeWages`, `tipIncome`, `medicaidWaiverPayments`, `dependentCareBenefits`, `adoptionBenefits` for tests 1/2, `uncollectedSocialSecurityMedicareWages` for tests 1/3, etc.) are `null` or `0`.
4. **No test asserts Form 8839 Part III line-by-line values for the special-needs case.** Test 3 asserts the final line 1f result but does not verify that lines 22, 25, and 29 of Form 8839 carry the right intermediate values. A buggy Form 8839 implementation could produce the right line 29 via wrong intermediate amounts.

None of these are correctness bugs in the spec; they are gaps in regression-target breadth.

## Glossary

- **Form 1040** — the standard United States individual federal income tax return.
- **Form W-2** (Wage and Tax Statement) — the year-end pay summary an employer sends.
- **Form 8839** (Qualified Adoption Expenses) — the form that figures the employer-provided-adoption-benefits exclusion and the adoption credit.
- **Form 8919** (Uncollected Social Security and Medicare Tax on Wages) — the form an employee files when work was misclassified as independent contracting.
- **Internal Revenue Code §32(c)(2)(B)(vi)** — the statute that lets a taxpayer elect to include nontaxable combat pay in earned income for the Earned Income Credit.
- **Internal Revenue Code §137(a)(3)** — the statute that grants the full per-child adoption-benefits exclusion for a special-needs-final adoption regardless of actual expenses.
- **Internal Revenue Code §23(h)** — the inflation-adjustment rule that produces the $17,280 per-child limit for tax year 2025.
- **Schedule 1** (Additional Income and Adjustments to Income) — supplementary schedule of income types and adjustments.
- **Schedule 8812** (Credits for Qualifying Children and Other Dependents) — schedule that computes the Child Tax Credit, the Credit for Other Dependents, and the refundable Additional Child Tax Credit.
- **Publication 17 — Your Federal Income Tax** — the IRS's master guide for individuals filing Form 1040.
- **Publication 525 — Taxable and Nontaxable Income** — the IRS publication that defines what kinds of income are taxable.
- **`addNonNull(a, b)`** — the application helper that returns `a + b` when both are non-null, the non-null operand when one is null, or `null` when both are null. Used inside `TaxReturnComputeService.java` to fold the eight Line 1z sub-line values without crashing on null inputs.
- **`roundMoney(value)`** — the application helper that rounds a BigDecimal to the nearest dollar (half-up) and returns null on a null input.
