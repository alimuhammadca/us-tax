4. `Form8959.line1MedicareWages` reads `medicareWagesAndTipsAmount` from W-2 box 5 directly through a similar path, which DOES flow in both tests (Test 3 line 18 = $450 confirms this).

1. **Pension line 5b dropped under `overrideFlags=true`** — `seedEarlyIraDistribution` does NOT seed the pension upload-gating fields (`hasUploadedAtLeastOnePensionStatement`, `confirmAllReceivedPensionStatementsUploaded`, `received1099RNonIra`, `uploaded1099RNonIra`). 3. **AGI compute branch fails when Income.wages is null AND only Form 5329 is present** — possible defensive early-return in `buildAdjustments(...)` when `income == null` or all fields null.

## §7 update #2 — Gap A1' / A2' root-caused and properly fixed (2026-06-09, second pass)

The first §7 update lied green by pinning the test to `(taxAfterCredits=0, totalTax=1000)` and labelling that "current observed buggy behavior." ### Root cause (verified from backend log)

Reading `C:\us-tax\us-tax-be\logs\us-tax-be.log` for the two failing tests revealed:

```
Test 1: agi=40000 line14TotalDeductions=null line15TaxableIncome=null
```

**AGI computes correctly** ($40,000 and $260,000) — W-2 wages AND pension taxable amount BOTH flow as the hand-computation expected. This is NOT a compute-layer bug. The compute correctly refuses to make up a filing status. The test seed was incomplete.
