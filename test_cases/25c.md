### 13h. Lesson captured from the third pass

The third-pass re-audit surfaced one previously-undocumented composer-path gap (Test 22's combined-branch) by **systematically enumerating each Boolean state of every composer conditional and checking which existing tests cover each state**. The composer has two contribution sources (W-2G and Form 8959 line 24) and each can be null-or-zero or positive. The 2×2 truth table:

| W-2G | Form 8959 line 24 | Test coverage |
|---|---|---|
| null/0 | null/0 | Tests 1, 2, 3, 6, 8 (line 25c stays null) |
| positive | null/0 | Test 4 (W-2G alone) |

Per `feedback_principled_diagnosis_over_test_tweaking`, the diagnostic shortcut is: **for every composer with multiple contribution sources, build the truth-table of conditional states and verify each cell has at least one test**. This pattern would have surfaced the gap during the §10 second pass had it been applied explicitly; logging it here as a generalizable lesson for future line audits.
