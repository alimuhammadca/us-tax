## Test 1 (line 37) — `electsNoActc opt-out zeroes ACTC regardless of eligibility`

```bash
npx playwright test --workers=1 --output=test-results-line28-1 \
  -g "electsNoActc opt-out" line28-actc-schedule8812.spec.ts
```

**Purpose.** Verifies the project-specific `electsNoActc` voluntary opt-out: when the user explicitly answers "yes, skip the refundable Additional Child Tax Credit," line 27 (and therefore Form 1040 line 28) is forced to zero — even when the IRS canonical computation would have produced a positive value. This is the G4 design decision documented in `XLS/Computations/28.md §10` (no IRS basis but kept as conservative opt-out preserving user agency).

**Inputs.**
- Single filer, $20,000 W-2 wages, 1 CTC-qualifying child.
- `ctc-actc-screening-taxpayer`: `{ electsNoActc: true }`.

**IRS hand calculation.** The IRS rule does not recognize an "opt-out" — a taxpayer who has earned-income-based Additional Child Tax Credit potential MUST claim it (subject only to the line 16b ceiling). The IRS canonical for these inputs (Single, $20k wages, 1 CTC child, no other credits) would produce:
- Line 14 (nonrefundable CTC) = $0 (standard deduction $15,750 brings line 15 to $4,250; tax bracket 10% = $425; minimal absorption).
- Line 16a = $2,200 − $0 = $2,200; line 16b = 1 × $1,700 = $1,700; line 17 = $1,700.
- Line 18a = $20,000; line 19 = $17,500; line 20 = $17,500 × 15% = $2,625.
- Line 27 = min($1,700, $2,625) = **$1,700** per IRS canonical.

**Project canonical (with electsNoActc).** The G4 design decision overrides this: when `electsNoActc=true`, line 27 = $0 unconditionally. This is documented in 28.md §10 Gap G4 as an accepted divergence from IRS canonical, preserving user agency to forgo the refund.
