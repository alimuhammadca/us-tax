**Purpose.** Verifies that a section 1341 claim-of-right repayment credit entered on the `31-other-payments` personal form reaches Schedule 3 line 13b, then line 14, then line 15, then Form 1040 line 31. The repayment amount of $8,000 is above the $3,000 statutory threshold per IRC §1341(a)(3), so the credit method is available and no advisory should fire.

**Inputs.**
- Personal form `31-other-payments`:
  - `hasSection1341Credit: true`
  - `hasOtherRefundableCredits: false`
  - `section1341RepaymentAmount: 8000` (informational; verifies threshold gate)
  - `section1341Credit: 2500`

**IRS hand calculation.**
- Per Internal Revenue Code section 1341(a), when a taxpayer repaid more than $3,000 of income that was reported in a prior year under a claim of right, they may choose between (a) deducting the repayment in the current year, OR (b) claiming a credit equal to the tax difference computed by recomputing the prior-year tax without the disputed income (the "credit method"). The user has chosen the credit method and computed $2,500 externally.
- $8,000 repayment > $3,000 threshold → credit method is available. ✓
- Schedule 3 line 13b = $2,500.
- Line 14 = 0 + $2,500 + 0 + 0 + 0 = $2,500.
- Line 15 = 0 + 0 + 0 + 0 + $2,500 = $2,500.
- Form 1040 line 31 = **$2,500**.

**Code path.** `applyOtherPaymentsFormToSchedule3` at `TaxReturnComputeService.java:25887-25920` (after the G-NEW-6 closure 2026-06-11). The repayment-amount-vs-credit-amount cross-check at lines 25901-25920 emits a `SECTION_1341_REPAYMENT_BELOW_THRESHOLD` advisory only when `section1341RepaymentAmount ≤ $3,000`. With $8,000 the gate does NOT fire. `finalizeSchedule3Totals` and `computeLine31ThroughLine38` aggregate as in Test 1. Code matches IRS rule.
