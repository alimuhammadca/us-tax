## Line 34 in one sentence

Form 1040 line 34 (the **overpaid** amount) is a conditional single subtraction `line 34 = line 33 − line 24` that fires only when `line 33 > line 24`. When it fires, the result is stored on `Refund.overpaid` via the lazy-creation convention documented in `XLS/Computations/34.md` gap A (the entire `Refund` object is created only when there is an overpayment, so `form1040.refund == null` means line 34 is blank, not zero). Line 34 has no IRS reference data of its own — all constants that move it from year to year move it via line 33 (see `XLS/Computations/33.md` section 5) or line 24.

## Critical project convention discovered during audit (Form 2210 stacking)

> **`AmountOwed.amountOwed = (IRS line 37 pure) + (Form 2210 line 38 penalty)`**

The project's `computeForm2210()` deliberately stacks the Form 2210 estimated-tax penalty INTO `AmountOwed.amountOwed` after the initial line-37 subtraction. The IRS Form 1040 keeps lines 37 and 38 in separate boxes; the project conflates them into one stored "total to pay" field. This is documented in `lines/33.md` section 4 and was empirically confirmed by the e2e re-run on 2026-06-12: for a Single $100k wages / $1k withholding fixture, `taxAndCredits.totalTax = $13,449`, `payments.totalPayments = $1,000`, `amountOwed.amountOwed = $12,965` (not $12,449 as the strict IRS invariant would predict), with the $516 difference accounted for by `amountOwed.estimatedTaxPenalty = $516` (Form 2210 line 38).

**Consequence for e2e tests:** any assertion of the strict IRS line-37 invariant must subtract `amountOwed.estimatedTaxPenalty` from `amountOwed.amountOwed` to recover the pure line 37 value:

```typescript
const lineThirtySevenPure = Number(amountOwed.amountOwed) - Number(amountOwed.estimatedTaxPenalty ?? 0);
expect(lineThirtySevenPure).toBe(totalTax - totalPayments);
```

This finding does **not** affect line 34 (the overpaid branch) — when line 34 fires, line 37 does not, and Form 2210 cannot fire (balance due is negative). Line 34 IS strictly `refund.overpaid = totalPayments − totalTax`, with no conflation.
