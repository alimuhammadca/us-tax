## Line 35a in one sentence

Form 1040 line 35a (the **refund amount**) is a conditional single subtraction `line 35a = line 34 − line 36` that fires only when line 34 fires (only when there is an overpayment). When it fires, the result is stored on `Refund.refundAmount` via the null-when-zero convention documented in `XLS/Computations/35a.md` gap A (when the taxpayer credits the entire overpayment to next year via line 36, line 35a equals zero — stored as `null` rather than `BigDecimal.ZERO` so the printed PDF box is blank, matching the IRS rule). Line 35a inherits the lazy-Refund-creation convention from `34.md` gap A — the entire `Refund` object stays null when there is no overpayment, so `refund.refundAmount` is unreachable in that case.

The structural partition invariant pinned by IRC §6402(b) and the IRS Form 1040 line-35a/36 labels:

```text
line 34 = line 35a + line 36   (every dollar of overpayment goes to either cash refund or carryforward)
```

This invariant is now pinned by the Java unit test `line35a_plus_line36_equals_line34_invariant` (added 2026-06-13 per `35a.md` gap F closure).
