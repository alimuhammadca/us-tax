## Line 35d in one sentence

Form 1040 line 35d (the **direct-deposit account number**) is a **conditional string passthrough** — not arithmetic. The taxpayer types the account number into the intake form; the backend copies it onto `Refund.accountNumber` only when the **triple gate of `XLS/Computations/35d.md` §1d** passes: (1) line 34 (overpaid) > 0, (2) intake `wantsDirectDeposit = true`, (3) Form 8888 split refund NOT elected. When any gate fails, line 35d is `null` on the model (and blank on the printed Form 1040). Line 35d inherits the lazy-Refund-creation convention from `34.md` gap A — the entire `Refund` object stays null when there is no overpayment, so `refund.accountNumber` is unreachable in that case.

Three IRS validation rules govern the string itself (now enforced by backend advisory flags as of 2026-06-13):

1. **17-character maximum** — `LINE_35D_ACCOUNT_NUMBER_TOO_LONG` flag + truncation when violated.
2. **Allowed character class** — digits 0-9, letters A-Z, hyphen "-". No spaces. `LINE_35D_ACCOUNT_NUMBER_INVALID_CHARACTERS` flag when violated.
3. **All-three-or-none triple rule** — lines 35b + 35c + 35d must all be populated or all blank when `wantsDirectDeposit=true`. `LINE_35BCD_INCOMPLETE_TRIPLE` flag when violated.

None of these three flag-emitting scenarios is exercised by the existing e2e suite (the existing tests all seed valid triples). This is documented as **Coverage Gap 1** in the audit findings below.

## Project convention — Form 8888 takes priority over single direct deposit

> **When `refund-allocation-taxpayer.wantsRefundAllocation = true` (Gate 3 fails), the backend SKIPS the entire direct-deposit triple block at `TaxReturnComputeService:27356`. Lines 35b/35c/35d are forced to `null` on `Refund` even when the legacy `35-direct-deposit` form has full routing/type/account-number data. The taxpayer-typed direct-deposit data does not appear anywhere on the computed return when Form 8888 is elected.**

This is the IRS Form 8888 mutual-exclusivity rule of `35d.md` §1d Gate 3. Verified by tests #6 (line33-refund-owed) and #19 (form8888-refund-allocation).
