## Line 36 in one sentence

Form 1040 line 36 (the **amount applied to 2026 estimated tax**) is a **voluntary taxpayer election** ÔÇö the dollar amount of the overpayment (line 34) the taxpayer wants the Internal Revenue Service to credit toward the first 2026 quarterly estimated tax installment (due April 15, 2026) instead of receiving as a cash refund. The amount is bounded by `0 ÔëĄ line 36 ÔëĄ line 34` (Internal Revenue Code ┬ž6402(b)). When the gate fails (no overpayment, OR taxpayer answered No to "Do you want to apply part or all of your overpayment to your 2026 estimated tax?", OR `amountToApply` rounded to zero), line 36 is stored as `null` on `Refund.amountAppliedToNextYear` per the null-when-zero convention (documented in `36.md` gap A as of 2026-06-13).

The structural partition invariant pinned by IRC ┬ž6402(b) and the IRS Form 1040 line-35a/36 labels:

```text
line 34 = line 35a + line 36
```

This invariant is pinned at the e2e layer by tests #4 and #10 (the two reduce-refund tests) and at the Java unit-test layer by `line35a_plus_line36_equals_line34_invariant` (added during the `35a.md` Gap F closure). The cap rule (`line 36 = min(amountToApply, line 34)`) is pinned at the e2e layer by tests #5 and #11 (the two cap-at-overpaid tests) and at the Java unit-test layer by `line36_cap_at_overpaid_when_amount_exceeds` (added today during `36.md` Gap F closure). The no-election gate is pinned at the Java unit-test layer by `line36_zero_when_election_not_made` (also added today). Together the four e2e tests and three Java unit tests cover the full line-36 surface: no election (line 36 = 0), within cap (line 36 = typed amount), over cap (line 36 = line 34).

## 2025 reference data (constants that affect line 36)

Line 36 has no rate, threshold, or cap constants of its own. The only constraints that limit line 36 are structural:

| Layer | Constant | 2025 value | Source | Enforcement |
|---|---|---|---|---|
| Line 36 upper bound | line 34 (the overpayment) | computed per return | Internal Revenue Code ┬ž6402(b) | Backend cap at Stage 2 (`line36.min(overpaid)`) |
| Line 36 lower bound | zero | constant | Internal Revenue Code ┬ž6402(b); IRS Form 1040 line 36 label | Backend initialization (`BigDecimal line36 = BigDecimal.ZERO;`) |
| Default crediting date | April 15, 2026 (first Q1 2026 installment) | constant | Publication 505 chapter 2 "Credit an Overpayment" | NOT modeled (no compute or display) |
| Override to later installment | Yes, via attached written statement | constant | Publication 505 chapter 2 | NOT modeled (rare in practice; per `36.md` Gap J) |
| Election irrevocability | True (after return filed) | statutory | Internal Revenue Code ┬ž6402(b); 2025 i1040gi_2025 line 36 paragraph | NOT enforced at code level; communicated to the user via the Tier-1 amber callout on the refund form |
| Rounding | Whole dollars | constant | IRS 2025 Form 1040 general instructions | Backend `roundMoney(raw)` at Stage 1 |

The upstream constants that affect Gate 1 (whether line 34 > 0 in the first place) are documented in `XLS/Computations/33.md` section 5 (line 33 components) and `XLS/Computations/34.md` section 5 (the `line34 = line33 Ôłĺ line24` subtraction).

## Project convention ÔÇö the two-stage compute pattern (Stage 1 unconditional read, Stage 2 capped store)

> **The line-36 election is read at Stage 1 (`TaxReturnComputeService.java:27317-27326`) BEFORE the overpayment branch is entered, into a local `BigDecimal line36` variable. The cap and the store happen at Stage 2 (`:27342-27346`) INSIDE the overpayment branch via `line36.min(overpaid)`. The local variable carries state across non-adjacent code regions; when the overpayment branch does not fire, the Stage-1 value goes out of scope at the end of the method and `Refund.amountAppliedToNextYear` is never set.**

This is the **TWO-STAGE CAPPED NUMERIC PASSTHROUGH** pattern documented in `lines/33.md` verification log row 4 (the original line-36 audit). It is distinct from a simple inline cap because the cap target (overpaid) is only available inside the overpayment branch.
