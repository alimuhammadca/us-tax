## Line 37 in one sentence

Form 1040 line 37 (the **amount you owe**) is a **conditional single subtraction** that fires when total tax (line 24) exceeds total payments (line 33). It is the dollar amount the taxpayer must remit to the Internal Revenue Service by April 15, 2026 per Internal Revenue Code §6151(a). The IRS canonical formula is `line 37 = line 24 − line 33` when `line 24 > line 33`; otherwise blank. Lines 34 (overpayment) and 37 (amount owed) are mutually exclusive — exactly one fires, or both are blank when `line 24 == line 33`.

The **project's Form 2210 stacking convention** (`lines/33.md` §4) carries the line 37 + line 38 penalty COMBINED in `AmountOwed.amountOwed` when Form 2210 fires. The pure IRS-canonical line-37 value is recovered via the new `getLineThirtySevenPure()` derived getter (added 2026-06-14 per `37.md` Gap D + G closure), computed as `amountOwed − estimatedTaxPenalty`. The frontend PDF emit at `form-tax-return-1040.component.ts:895` now writes the pure value to the IRS line-37 PDF box; the in-app summary continues to show the stacked total.

The strict-equality invariant pinned by Internal Revenue Code §6151(a) and the IRS Form 1040 line-37 label:

```text
pure line 37 = line 24 − line 33   (when line 24 > line 33; blank otherwise)
```

This invariant is pinned at the e2e layer by tests #11 (the primary `line37: amount owed when underpaid` test which subtracts `estimatedTaxPenalty` from `amountOwed.amountOwed` before asserting against `totalTax − totalPayments`) and at the Java unit-test layer by the two new `line37_pure_equals_total_tax_minus_total_payments_when_form2210_fires` and `line37_pure_equals_amountOwed_when_form2210_does_not_fire` tests added 2026-06-14.

## Project conventions — Form 2210 stacking + pure-vs-stacked emit divergence

> **Stage 1 storage** (`TaxReturnComputeService.computeLine31ThroughLine38:27440-27449`): when `totalTax > line33`, store `amountOwed.amountOwed = totalTax − line33` (the pure IRS-canonical line 37).
>
> **Stage 2 storage** (`TaxReturnComputeService:2073-2088`): when Form 2210 fires with a positive penalty AND is not waived, add the penalty on top: `amountOwed.amountOwed += form2210.totalPenalty` AND `amountOwed.estimatedTaxPenalty = form2210.totalPenalty`. The stored value of `amountOwed.amountOwed` is now `pure line 37 + line 38`.
>
> **Pure-value recovery** (added 2026-06-14): `AmountOwed.getLineThirtySevenPure()` returns `amountOwed − estimatedTaxPenalty` (handling nulls). This is the Stage-1 value.
>
> **PDF emit** (`form-tax-return-1040.component.ts:895`, updated 2026-06-14): writes the pure value to the IRS line-37 PDF box so the IRS strict-equality rule `line 37 == line 24 − line 33` holds on the printed return.
>
> **In-app summary** (unchanged): the refund-and-amount-owed banner reads from `amountOwed.amountOwed` (the stacked total) so the user sees what they actually must remit.

The existing e2e tests reads `amountOwed.amountOwed` directly from the JSON compute payload — which still carries the stacked total. The test #11 (the primary line-37 hand-pin) implements the subtract-penalty math locally; this manual implementation is now redundant given the new `getLineThirtySevenPure()` derived getter but continues to work correctly.

- `employment-income-taxpayer`: `hasEmploymentIncome=true`

**Hand-compute:** $100k Single → ~$13,449 total tax. $1k withholding → totalPayments = $1,000. pure line 37 = $13,449 − $1,000 = $12,449. Form 2210 fires (balance due ~$12,449 ≥ $1,000 AND neither safe harbor met) → penalty ≈ $500-700 (varies with quarterly timing). stored amountOwed = $12,449 + penalty.

**Code path:**
- Stage 1 (`computeLine31ThroughLine38:27442-27448`): `owed = roundMoney(totalTax − line33) = roundMoney(13449 − 1000) = 12449`. `amountOwed.setAmountOwed(12449)`.
- Stage 2 (`TaxReturnComputeService:2073-2088` after `computeForm2210`): `amountOwed.setEstimatedTaxPenalty(penalty)`. `amountOwed.setAmountOwed(12449 + penalty)`.
- Test subtracts: `pureLine37 = amountOwed − estimatedTaxPenalty = (12449 + penalty) − penalty = 12449`.
- Assert: `pureLine37 === totalTax − totalPayments === 12449`. ✓

**Assertions (line-37 lens):**

|---|---|---|
| 1 | `result` truthy | ✅ OK |
| 2 | `totalTax > 0` (dynamic read) | ✅ OK |
