# Test cases — Line 3a (Qualified dividends)

**Spec files audited:**
- `e2e/tests/line3ab-dividend-income.spec.ts` (8 tests, both for Line 3a and 3b)
- `e2e/tests/line8814-child-interest-dividends.spec.ts` (13 tests, Form 8814 line 9 → parent Line 3a/3b)

**Total tests: 21 (all passing per regression run 2026-06-03).**

**Audit method per test (IRS-first):**
1. Read inputs the test seeds.
2. Read the IRS rule that should apply (Form 1040 instructions, Pub. 17, Pub. 550, Form 8814 instructions for 2025; cited inline).
3. Hand-compute each expected output.
4. Read the Java compute path (`src/main/java/com/ustax/microservices/TaxReturnComputeService.java`) that processes the test's inputs and confirm whether the code matches the IRS rule.
5. Read the test's `expect(...)` assertions.
6. Per assertion: **OK** (matches IRS) / **MISMATCH** (assertion contradicts IRS rule) / **GAP** (assertion missing for an output the IRS rule says should be populated).

**Key IRS constants used in this audit (2025):**
- Schedule B Part II trigger: ordinary dividends > $1,500 (Schedule B 2025 instructions). Stable since 1990s.
- Form 8814 child gross-income ceiling: $13,500 (Form 8814 2025 instructions, "Parents' election" eligibility test).
- Form 8814 line 5 base amount (Part I unearned income exclusion): $2,700 (Form 8814 2025 instructions, line 5; 2024 was $2,600).
- Form 8814 line 13 base amount (Part II extra-tax base): $1,350 (Form 8814 2025 instructions, line 13; half of line 5).
- Form 8814 line 15 ceiling: $135 (10% of $1,350).
- Form 6251 line 2g (AMT specified Private Activity Bond dividends from 1099-DIV box 13): no IRS amount cap; flows through 1:1.

### Test 2 — Line 3a/3b flow computes and adds Schedule B when required

`line3ab-dividend-income.spec.ts:109`

- Schedule B 2025: filed when Line 3b > $1,500 OR when any nominee dividend present.

**Hand-compute:**
- Line 3b ordinary = $1,600 (statement) + $100 (manual) − $50 (nominee) = **$1,650**.
- Line 3a candidate qualified = $700 (statement box 1b) + $40 (manual) = $740.
- disallowedQualifiedTotal = $10 (nominee qualified) + $5 (failed common-stock holding period) = $15.
- Line 3a = $740 − $15 = **$725** (≤ Line 3b = $1,650 → cap not triggered).
- Schedule B required: Line 3b $1,650 > $1,500 → **TRUE** (also nominee triggers it).
- Schedule B Part II line 6 = Line 3b = **$1,650**.
- Form 6251: no box 13 PAB dividends → **null**.

**Code path:** `computeDividendForPerson` at `TaxReturnComputeService.java:7757-7895`. Line 3b at `:7874` (`subtractNonNegative(ordinary, nomineeOrdinary)`); Line 3a at `:7875` (`subtractNonNegative(candidateQualified, disallowedQualifiedTotal)`); Schedule B trigger at `:7662-7664`. Final whole-dollar rounding at `:7747-7748`. Matches IRS.

**Assertions:**
1. `qualifiedDividends === 725` → **OK** (matches hand-compute $725).
2. `ordinaryDividends === 1650` → **OK** (matches $1,650).
3. `scheduleB.line6TotalOrdinaryDividends === 1650` → **OK** (matches $1,650).
4. `form6251 === null` → **OK** (no box 13 → no Form 6251).
5. `scheduleBLink` visible → **OK** (Schedule B required, sidebar should show it).

**GAPs (additional outputs the IRS rule says should be populated):**
- **GAP 2-A** — No assertion that `scheduleB.dividendItems` contains the per-payer ordinary row (≈ $1,600) plus the new negative "Taxpayer nominee dividend distribution" row of -$50 added in 3a Gap 5 (closed 2026-06-03).
