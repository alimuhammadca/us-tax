## Test 1 — Taxpayer IRA form blocks save when no IRA 1099-R statements are uploaded

`line4abc-ira-income.spec.ts:111`

**Purpose (Line 4b lens):** **Not directly applicable to Line 4b.** This is a UI gate test that prevents the IRA Income form from being saved when the user claims IRA distributions but no IRA 1099-R statements exist in the user's `1099-r` collection. Per Internal Revenue Code §408(d)(1) and IRS Publication 590-B, the taxable amount of Individual Retirement Account distributions for Line 4b is computed from actual 1099-R box 2a values — there is no Line 4b value when no 1099-R exists.

The compute pipeline at `TaxReturnComputeService.computeIraDistributions` is therefore never invoked.

**Assertions touching Line 4b:** None. The test asserts only:
1. `saveResponse` is null (PUT did not fire) — proves the validation gate intercepted.
2. `refreshResponse` is not null (the GET `/api/statements/1099-r/entries` from `onSubmit`'s `refreshIraStatementCount()` did fire) — proves `onSubmit` ran past the silent `loading || saving` early-return.

Together these prove the `isValid()` early-return branch fired, not the silent early-return branch.

**Line 4b verdict:** **N/A** — Test 1 does not exercise Line 4b. No assertions on `taxableIraDistributions`. No IRS rule for Line 4b applies because no distribution is being reported.
