# Form 1040 Line 5a — Per-Test IRS Audit

**Status:** Tax Year 2025. Per-test review of every end-to-end and Java unit test that touches Form 1040 Line 5a (Pensions and annuities — gross amount, when required) or any code path that contributes to it. Each test is audited against the IRS source documents (`docs/books/i1040gi_2025.pdf`, IRS Publication 17, Publication 525, Publication 575, Publication 939, Internal Revenue Code §§61, 72, 402(c), 402(l)) and the Java compute path in `TaxReturnComputeService.java`.

Each assertion is graded:

- **OK** — assertion matches what the IRS rule would compute by hand AND the code implements that rule correctly.
- **MISMATCH** — assertion value disagrees with the IRS rule (either the test asserts the wrong value, or the code computes the wrong value and the test locks it in).
- **GAP** — test is silent on an output the IRS rule says should be populated (the assertion is missing, not wrong).

Companion spec: `XLS/Computations/5a.md` (line-level IRS rule + code implementation).

## Test 8 — Distribution code S generates Form 5329 with 10% early-distribution penalty

**Purpose:** Verifies that distribution code "S" triggers the Form 5329 early-distribution additional tax.

**Inputs:**
- 1099-R: `iraSepSimple=false`, gross=4000, taxable=4000, code='S'
- Pension form: `hasEarlyDistributionAdditionalTaxForForm5329=true`, `form5329ExceptionCodeOrReason='02'`

**IRS rule (Internal Revenue Code §72(t), IRS Form 1099-R instructions box 7):**
- **Code S** on a Form 1099-R = "Early distribution from a SIMPLE IRA in the first 2 years, no known exception" — per IRC §72(t)(6) the additional tax is **25%**, not 10%.
- **Code 1** = "Early distribution, no known exception" — additional tax is **10%** per IRC §72(t)(1).
- Code S inherently means SIMPLE IRA, so a real-world code-S 1099-R would have the IRA/SEP/SIMPLE checkbox marked (which would route the distribution to Form 1040 Lines 4a/4b, not 5a/5b).

**By-hand expected (strict IRS reading):** If `iraSepSimple=false` (pension routing), the IRS rule for code S is undefined because code S is an IRA-only code. The application currently treats code S identically to code 1 (10% rate at `:9269-9271`). The test asserts 400 (= 10% × 4000). For a true SIMPLE IRA code-S distribution (which would have iraSepSimple=true and route to Lines 4a/4b, not here), the IRS expects 25%, not 10%.

**Code path:** `:9106-9109` (`if codes.contains("1") || codes.contains("S") → earlyDistributionBase += taxable`); `:9268-9271` (`× 0.10`).

**Assertions:**

| # | Assertion | Expected (IRS) | Verdict | Notes |
|---|---|---|---|---|
| 1 | `pensionsAnnuities === null` | null | **OK** | Fully taxable. |
| 2 | `taxablePensionsAnnuities === 4000` | 4000 | **OK** | |
| 3 | `form5329.additionalTaxOnEarlyDistributions === 400` | **25% × 4000 = 1000 per IRC §72(t)(6) for a true SIMPLE IRA code-S** | **MISMATCH** | Code S on a SIMPLE IRA carries a 25% rate, not 10%. The application currently applies 10% to both code 1 and code S, and the test locks in the 10% behavior. The test scenario (`iraSepSimple=false` with code S) is itself IRS-inconsistent — a true code-S 1099-R would have `iraSepSimple=true` and route to the IRA path. |
| 4 | `form5329.exceptionCodeOrReason === "02"` | "02" | **OK** | Pass-through. |

**GAPs identified:**

- **GAP A8.1** — Test does not assert `line5cBox1/2/3` are all false.
- **GAP A8.2** — Test (and code) needs to be split: a code-S test on the IRA path (with `iraSepSimple=true` and computed via Line 4a/4b) with a 25%-rate assertion, plus a code-1 test on the pension path with a 10%-rate assertion. The current single test conflates the two.

**Recommended remediation (separate change):** Add `codes.contains("S")` to a 25% branch in the compute (Internal Revenue Code §72(t)(6)). Track in `outstanding.md`.
