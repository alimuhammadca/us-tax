# Form 1040 Line 5c — Per-Test IRS Audit

**Status:** Tax Year 2025. Per-test review of every end-to-end and Java unit test that touches Form 1040 Line 5c (pension/annuity disclosure checkboxes + write-in text) or any code path that contributes to it. Each test is audited against the IRS source documents (`docs/books/i1040gi_2025.pdf`, IRS Publication 17, Publication 525, Publication 575, Internal Revenue Code §§402(c), 402(e), 402(l)) and the Java compute path in `TaxReturnComputeService.java`.

Each assertion is graded:

- **OK** — assertion matches what the IRS rule would compute by hand AND the code implements that rule correctly.
- **MISMATCH** — assertion value disagrees with the IRS rule (either the test asserts the wrong value, or the code computes the wrong value and the test locks it in).
- **GAP** — test is silent on an output the IRS rule says should be populated.

- `XLS/Computations/5c.md` (line-level IRS rule + code implementation)
- `test_cases/5a.md` (Line 5a per-test audit — many tests assert all three of 5a/5b/5c; the 5a perspective is in 5a.md)
- `test_cases/5b.md` (Line 5b per-test audit)

**Code path:** Orchestrator at `:9008-9011`. Per-person `hasRollover/hasPsoElection/effective text` all set; orchestrator OR-aggregates.

| 2 | `line5cBox2Pso === true` | **OK** — PSO double-gate passes |
| 3 | `line5cBox3Other === true` | **OK** — write-in text "CODEX" present |
| 4 | `line5cBox3Text contains "CODEX"` | **OK** — joined text includes the token |

**GAPs identified:**
- **GAP C-T4.1** — Test asserts `line5cBox3Text contains "CODEX"` (substring match) but doesn't assert the EXACT text.
